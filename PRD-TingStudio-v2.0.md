## 1. 产品概述

### 1.1 产品定位

TingStudio 是面向食品配方行业的专业工作数据管理平台，当前版本已实现基础的客户管理、配方管理、原料管理等功能。v2.0 版本将重点拓展业务协作能力、数据追溯能力和生产对接能力。

### 1.2 版本目标

- **提升协作效率**：通过业务员角色打通销售端与研发端的协作链路
- **强化数据追溯**：建立配方版本管理机制，支持变更追溯和对比分析
- **扩展输出能力**：支持多格式导出和API对接，满足生产系统需求
- **增强数据价值**：集成营养成分计算，提供专业的营养分析报告

### 1.3 目标用户

| 用户角色 | 核心需求 | 使用场景 |
|---------|----------|---------|
| 业务员 | 管理客户、对接配方师、获取配方输出 | 销售拜访、客户沟通、订单跟进 |
| 配方师 | 管理配方版本、关联业务员、分析营养成分 | 配方研发、版本迭代、营养优化 |
| 生产人员 | 获取生产配方数据、查看营养信息 | 生产计划、配料计算 |
| 管理员 | 配置系统参数、管理用户权限 | 系统维护、数据管理 |

## 2. 用户角色

### 2.1 业务员

**角色定义**：负责客户关系维护和配方需求传达的销售人员

**核心职责**：
- 维护客户信息和关系
- 与配方师对接配方需求
- 接收配方输出（PDF/Excel/API）
- 记录客户反馈

**权限范围**：
- 查看和管理关联的客户
- 查看关联的配方师信息
- 导出和分享授权的配方
- 查看配方的基本信息和营养汇总

### 2.2 配方师

**角色定义**：负责配方研发和迭代的专业技术人员

**核心职责**：
- 创建和编辑配方
- 管理配方版本历史
- 分析和优化营养成分
- 与业务员沟通需求

**权限范围**：
- 全部的配方CRUD权限
- 版本管理和对比
- 营养成分编辑和计算
- 配方导出和API配置

### 2.3 生产人员

**角色定义**：负责配方生产执行的操作人员

**核心职责**：
- 获取生产配方数据
- 查看配料清单和营养信息
- 执行生产计划

**权限范围**：
- 查看已发布的配方
- 下载生产用的配方数据
- 查看营养成分汇总

### 2.4 管理员

**角色定义**：负责系统配置和权限管理

**核心职责**：
- 管理用户和角色
- 配置导出模板
- 管理营养标准数据
- 系统监控和维护

**权限范围**：
- 全部管理权限
- 系统配置权限

## 3. 核心功能设计

### 3.1 业务员数据管理体系

#### 3.1.1 功能概述

建立业务员、客户、配方师之间的关联关系，打通销售端与研发端的协作链路，提高配方需求传达和反馈的效率。

#### 3.1.2 功能模块

##### A. 业务员管理

**功能描述**：对业务员信息进行全生命周期管理

**核心功能**：

1. **业务员列表**：展示所有业务员的基础信息，支持搜索、筛选、分页，状态标识（活跃/停用）
2. **业务员创建**：填写基本信息（姓名、工号、部门等），关联初始客户，关联初始配方师
3. **业务员编辑**：修改基本信息，添加/移除客户关联，添加/移除配方师关联
4. **业务员详情**：展示完整业务员信息，关联客户列表（可跳转），关联配方师列表（可跳转），关系图谱可视化
5. **业务员删除/停用**：软删除机制（保留历史数据），解除所有关联关系

##### B. 业务员-客户关联

**功能描述**：建立业务员与客户的关联关系

**关联类型**：
- 主业务员：该客户的主要联系人，可编辑配方、导出数据
- 辅助业务员：协助主业务员，仅查看配方

**核心功能**：
1. 关联客户：选择客户（支持多选），设置关联类型，设置生效日期，添加备注
2. 关系管理：查看关联列表，修改关联类型，解除关联（保留历史记录）
3. 关系转移：当业务员离职时，将客户关系转移给其他业务员，保留历史沟通记录

##### C. 业务员-配方师对接

**功能描述**：建立业务员与配方师的协作关系

**对接模式**：
- 直接对接：业务员直接与配方师沟通，常规配方需求
- 间接对接：通过协调员或文档流转，跨部门协作

**核心功能**：
1. 对接关系建立：选择配方师（支持多选），设置对接模式，设置优先级（1-5），添加对接说明
2. 沟通记录：记录沟通内容，支持多种沟通类型（邮件、电话、会议、消息），关联配方或客户，添加附件
3. 对接管理：查看对接关系列表，查看沟通历史，修改对接模式，解除对接关系

#### 3.1.3 业务流程

```
┌─────────────┐
│ 业务员创建 │
└──────┬──────┘
       │
       ├─→ 关联客户（主/辅助）
       │
       └─→ 对接配方师（直接/间接）
             │
             ├─→ 配方需求沟通
             │
             └─→ 配方交付反馈
```

### 3.2 配方版本控制与对比

#### 3.2.1 功能概述

建立配方的多版本管理机制，记录每次修改的历史，提供版本间的可视化差异对比功能，确保配方变更的可追溯性。

#### 3.2.2 功能模块

##### A. 版本管理

**功能描述**：管理配方的所有版本

**核心功能**：

1. 自动版本创建：每次编辑配方时自动创建新版本，支持手动创建快照版本，版本号自动递增（v1.0, v1.1, v2.0）
2. 版本列表：展示配方的所有版本，显示版本号、版本名称、创建时间、创建人，标识当前版本，支持版本筛选（全部/草稿/已发布/已归档）
3. 版本状态管理：草稿（Draft）编辑中的版本，已发布（Published）可用于生产的版本，已归档（Archived）历史版本
4. 版本切换：可切换查看任意版本，支持将旧版本设为当前版本，切换时保存操作日志
5. 版本删除：删除草稿版本，归档历史版本，保留变更记录

##### B. 版本对比

**功能描述**：可视化对比两个版本间的差异

**核心功能**：

1. 版本选择：选择版本A和版本B进行对比，支持对比任意两个版本，默认对比当前版本与上一版本
2. 差异展示：客户变更（新增/修改/删除），原料变更（新增/修改/删除/数量调整），配方描述变更，营养成分变更
3. 可视化对比：并排视图（并排显示两个版本），统一视图（在一个视图中标注差异），表格对比（按字段逐行对比），高亮显示变更项
4. 差异筛选：全部差异，仅显示材料变更，仅显示描述变更，仅显示营养变更
5. 差异导出：导出对比报告（PDF/Excel），生成变更摘要，支持打印

##### C. 变更记录

**功能描述**：记录每次配方修改的详细变更

**核心功能**：

1. 自动记录变更：监控配方编辑操作，自动记录字段级变更，保存变更时间、变更人
2. 变更详情展示：字段名称，旧值和新值，变更类型（新增/修改/删除）
3. 变更追溯：查看某个字段的变更历史，时间轴展示，支持回滚到任意历史版本

#### 3.2.3 业务流程

```
创建配方（v1.0 草稿）
    ↓
编辑配方 → 自动创建 v1.1 草稿
    ↓
发布配方 → v1.1 设为已发布
    ↓
再次编辑 → 自动创建 v1.2 草稿
    ↓
对比 v1.1 和 v1.2
    ↓
确认变更 → v1.2 设为已发布
    ↓
需要回退 → 切换到 v1.1
```

### 3.3 多元化配方输出方案

#### 3.3.1 功能概述

扩展配方输出能力，支持PDF、Excel等通用格式导出，以及生成可供生产系统直接调用的API数据接口，满足不同场景的使用需求。

#### 3.3.2 功能模块

##### A. 导出管理

**功能描述**：提供多种导出格式和模板配置

**支持格式**：
- PDF：可打印的文档格式，客户确认、归档保存
- Excel：电子表格格式，数据分析、批量处理
- API：JSON/XML数据格式，系统对接、自动化
- Print：直接打印，快速打印、现场使用

**核心功能**：

1. PDF导出：配置页面格式（A4/A3、纵向/横向），自定义页眉页脚，选择包含的内容区块，添加水印，设置导出模板，批量导出
2. Excel导出：自定义列（选择导出哪些字段），设置数据格式，添加数据验证，支持多工作表，批量导出
3. API输出：生成标准JSON/XML格式，配置字段映射，设置访问认证，生成API文档，查看调用日志
4. 打印输出：快速打印功能，打印预览，打印设置（纸张、份数），打印模板选择

##### B. 导出任务管理

**功能描述**：管理导出任务的执行状态

**核心功能**：

1. 创建导出任务：选择配方和版本，选择导出格式和模板，设置输出参数
2. 任务队列：查看待处理任务，查看任务进度，任务优先级管理
3. 任务状态跟踪：待处理，处理中，已完成，失败
4. 任务历史：查看历史导出记录，重新下载已导出的文件，查看失败原因

##### C. API数据接口

**功能描述**：生成可供生产系统调用的标准API接口

**核心功能**：

1. 接口配置：配置接口名称和描述，设置请求方式（GET/POST/PUT/DELETE），配置数据格式（JSON/XML），设置字段映射规则
2. 认证配置：无需认证，Basic Auth，API Key，OAuth 2.0
3. 字段映射：配方字段到目标字段的映射，设置转换函数，设置默认值，必填字段标记
4. 接口测试：测试接口调用，查看请求/响应，验证字段映射
5. 接口文档：自动生成API文档，示例请求和响应，调用示例代码

##### D. 分享功能

**功能描述**：通过链接或邮件分享配方

**核心功能**：

1. 生成分享链接：设置链接有效期，设置访问密码，限制下载次数，限制访问邮箱
2. 邮件分享：选择收件人，添加邮件主题和正文，附件格式选择
3. 分享管理：查看分享记录，查看下载统计，取消分享

#### 3.3.3 业务流程

```
配方编辑完成
    ↓
选择导出格式（PDF/Excel/API/Print）
    ↓
选择/配置模板
    ↓
创建导出任务
    ↓
任务处理中...
    ↓
下载文件/获取API地址
    ↓
（可选）分享链接或邮件
```

### 3.4 营养成分集成模块

#### 3.4.1 功能概述

集成原料营养成分数据库，在配方编辑时自动计算并展示营养成分总值，提供专业的营养分析和合规性检查。

#### 3.4.2 功能模块

##### A. 原料营养成分库

**功能描述**：管理原料的营养成分数据

**核心功能**：

1. 营养成分录入：为每种原料录入完整的营养成分，支持批量导入（Excel/CSV），支持从公共数据库同步，数据来源标注
2. 营养成分字段：宏量营养素（能量、蛋白质、脂肪、碳水化合物、膳食纤维），矿物质（钠、钾、钙、铁、锌、镁、磷），维生素（维生素A、C、D、E、K、B族），其他（胆固醇、反式脂肪、饱和脂肪）
3. 数据版本管理：记录数据更新历史，支持回滚到旧版本，标注数据来源（国家标准/实验室检测）
4. 营养数据查询：按原料搜索，按成分范围筛选，对比不同原料的营养成分

##### B. 配方营养计算

**功能描述**：根据配方原料自动计算总营养成分

**核心功能**：

1. 自动计算：实时计算配方总营养成分，计算每100g营养成分，计算每份营养成分（可配置份量），显示各原料的营养贡献占比
2. 计算逻辑：原料营养成分 × 原料数量 = 原料营养贡献，所有原料营养贡献相加 = 配方总营养，总营养 ÷ 配方总重量 = 每100g营养
3. 实时更新：添加/修改原料时自动重算，调整原料数量时自动重算，删除原料时自动重算
4. 营养贡献分析：每种原料的营养贡献，原料重量占比，关键营养素的原料来源分析

##### C. 营养标准管理

**功能描述**：管理不同产品类别的营养标准

**核心功能**：

1. 标准配置：创建营养标准档案，设置目标值范围，设置容差范围，标注适用产品类型
2. 标准分类：婴幼儿配方标准，儿童配方标准，成人配方标准，老年人配方标准，孕妇配方标准，特殊用途配方标准
3. 容差配置：设置必填字段，设置最小值和最大值，设置警告等级，自定义容差说明

##### D. 营养分析报告

**功能描述**：生成配方营养分析报告

**核心功能**：

1. 合规性检查：对比配方营养值与标准范围，生成合规性报告，标注超限/不足项，计算偏差值
2. 营养平衡分析：宏量营养素比例分析，矿物质配比分析，维生素配比分析，生成优化建议
3. 可视化展示：营养成分雷达图，原料营养贡献饼图，与标准对比的柱状图，营养趋势折线图（多版本对比）
4. 报告导出：导出PDF报告，导出Excel数据，导出图表图片

#### 3.4.3 业务流程

```
配方编辑
    ↓
添加原料（带营养数据）
    ↓
自动计算营养总值
    ↓
实时显示营养汇总
    ↓
对比营养标准
    ↓
生成分析报告
    ↓
导出报告或优化配方
```

## 7. 实施计划

### 7.1 迭代规划

### A. 术语表

| 术语 | 说明 |
|------|------|
| 业务员 | 负责客户关系和配方需求传达的销售人员 |
| 配方师 | 负责配方研发和迭代的技术人员 |
| 版本 | 配方的特定快照，记录配方在某个时间点的状态 |
| 快照 | 配方完整数据的副本，用于版本回退 |
| 对比 | 比较两个版本之间差异的过程 |
| 营养成分 | 食物中的营养素含量（能量、蛋白质、脂肪等） |
| 合规性 | 配方营养成分是否符合相关标准的要求 |
| API接口 | 应用程序编程接口，用于系统间数据交换 |
