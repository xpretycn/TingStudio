# 第一章 功能模块划分

## 1.1 模块总览

| 模块编号 | 模块名称 | 功能描述 | 优先级 |
|---------|---------|---------|-------|
| FM1 | 文件列表页 | 文件列表展示、搜索筛选、分页排序 | P0 |
| FM2 | 文件上传 | 文件上传、格式校验、持久化存储 | P0 |
| FM3 | 文件详情 | 文件元数据展示、关联数据展示、解析结果查看 | P0 |
| FM4 | 文件-数据关联 | 文件与配方/原料的关联管理 | P0 |
| FM5 | 文件生命周期 | 归档标记、定期清理、版本管理 | P1 |
| FM6 | 审计追踪 | 上传/解析/关联/访问记录查询 | P1 |
| FM7 | 访问权限 | 文件查看/删除/重新解析权限控制 | P1 |
| FM8 | 文件预览 | 原文件内容预览、格式显示、内容导航、性能优化 | P0 |

## 1.3 FM2 文件上传

```
FM2 文件上传
├── FM2.1 上传入口（3 种方式）
│   │
│   ├── 方式1：AI 助手页面自动上传
│   │   ├── 触发条件：用户在 AI 助手页面确认提交配方或原料表单时同步触发
│   │   ├── 触发时机
│   │   │   ├── 智能填单 Tab：用户点击"确认并创建配方"→ 配方创建成功后 → 自动上传原始文件
│   │   │   └── 智能导入 Tab：用户点击"一键录入"或"逐条录入"→ 原料创建成功后 → 自动上传原始文件
│   │   ├── 交互流程
│   │   │   ├── 用户上传文件到 AI 助手 → 文件暂存于临时目录
│   │   │   ├── AI 解析完成 → 用户确认提交
│   │   │   ├── 数据创建成功 → 系统自动将暂存文件移至正式存储目录
│   │   │   ├── 自动写入 uploaded_files 表，关联 related_id 和 related_type
│   │   │   └── 用户无需额外操作，上传过程完全透明
│   │   ├── 技术实现
│   │   │   ├── 前端：AI 助手提交表单时，在 createFormula/createMaterial 请求中附带 fileId 参数
│   │   │   ├── 后端：数据创建成功后，调用 fileService.linkFile(fileId, newRecordId, type)
│   │   │   └── 若用户未确认提交（如清空结果），临时文件在 24 小时后自动清理
│   │   └── 权限：已认证用户，跟随 AI 助手操作权限
│   │
│   ├── 方式2：文件管理页面手动上传
│   │   ├── 触发条件：用户在文件管理页面主动上传文件
│   │   ├── 触发时机：用户点击 FileManagement.vue 工具栏右侧"上传文件"按钮
│   │   ├── 交互流程
│   │   │   ├── 点击"上传文件" → 弹出文件选择对话框
│   │   │   ├── 选择文件 → 选择文件类型（配方文件/原料文件）
│   │   │   ├── 文件上传 → 进度条展示 → 上传成功提示
│   │   │   ├── 文件记录写入 uploaded_files 表，状态为 uploaded
│   │   │   └── 文件列表自动刷新，新文件出现在列表顶部
│   │   ├── 技术实现
│   │   │   ├── 前端：fileApi.upload(formData) → 成功后 fetchFiles() 刷新列表
│   │   │   └── 后端：multer 处理文件存储 → INSERT INTO uploaded_files
│   │   └── 权限：已认证用户均可操作
│   │
│   └── 方式3：表单页提交时同步上传
│       ├── 触发条件：用户在配方表单页或原料表单页确认提交时同步触发
│       ├── 触发时机
│       │   ├── FormulaForm.vue：用户点击"保存配方"→ 配方创建/更新成功后 → 若当前表单数据来源于 AI 解析文件，自动上传该文件
│       │   └── MaterialForm.vue：用户点击"保存原料"→ 原料创建/更新成功后 → 若当前表单数据来源于 AI 解析文件，自动上传该文件
│       ├── 交互流程
│       │   ├── 用户通过 AI 助手解析文件 → 数据预填到表单页
│       │   ├── 用户在表单页修改/补充数据 → 点击"保存"
│       │   ├── 表单提交成功 → 系统检测到数据来源于 AI 解析（通过 route query 或 store 标记）
│       │   ├── 自动触发文件上传 → 将原始解析文件关联到新创建的记录
│       │   └── 用户无需额外操作，上传过程透明，成功后显示"文件已自动归档"提示
│       ├── 技术实现
│       │   ├── 前端：表单提交成功后，检查 sourceFileId 标记
│       │   │   ├── FormulaForm.vue：handleSubmit 成功后 → if (sourceFileId) fileApi.link(sourceFileId, newFormulaId, 'formula')
│       │   │   └── MaterialForm.vue：handleSubmit 成功后 → if (sourceFileId) fileApi.link(sourceFileId, newMaterialId, 'material')
│       │   ├── 数据来源标记传递
│       │   │   ├── AI 助手跳转表单时：router.push({ path: '/formulas/new', query: { sourceFileId: 'xxx' } })
│       │   │   └── 表单页接收：const sourceFileId = route.query.sourceFileId
│       │   └── 后端：linkFile API 更新 uploaded_files 表的 related_id 和 related_type
│       └── 权限：已认证用户，跟随表单提交权限
│
├── FM2.2 上传流程
│   ├── 选择文件 → 格式校验 → 大小校验 → 上传
│   ├── 上传中：进度条展示
│   ├── 上传成功：文件记录写入数据库 + 提示
│   └── 上传失败：错误提示 + 重试
│
├── FM2.3 格式校验
│   ├── 支持格式：.xlsx, .xls, .csv, .png, .jpg, .jpeg, .gif, .webp
│   ├── 大小限制：≤ 10MB
│   └── 文件名校验：不含特殊字符
│
└── FM2.4 存储策略
    ├── 存储路径：./data/uploads/{type}/{yyyy-MM}/{uuid}.{ext}
    ├── 命名规范：{原始文件名}_{uuid前8位}.{ext}
    ├── 目录自动创建：若路径不存在则递归创建
    └── 重复文件：同一文件重复上传，version 字段递增
```

const storage = multer.diskStorage({
  destination: (req, file, cb) => {
    const fileType = req.body.fileType || 'formula';

    const dir = path.join('./data/uploads', fileType, dateDir);
