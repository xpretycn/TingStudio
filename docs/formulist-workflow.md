## 3. 原料管理

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 3.1 | 进入原料列表 `/materials` | 显示所有原料（共享数据，非仅自己创建） |
| 3.2 | 创建新原料 `/materials/new` | 填写名称、类型(主料/辅料)、单位、单价、营养数据(每100g) |
| 3.3 | 提交新原料 | 状态为"已发布"，出现在所有配方的原料选择器中 |
| 3.4 | 编辑已发布原料 | 创建新版本(draft)，原版本保持不变 |
| 3.5 | 查看原料详情 `/materials/:id` | 显示营养数据、价格信息、引用该原料的配方列表 |
| 3.6 | 查看原料版本历史 `/materials/:id/versions` | 显示版本时间线，可对比最多3个版本 |

### 关键验证点

- **营养数据完整性**：能量/蛋白质/脂肪/碳水/钠 必须有值
- **单价精度**：`toFixed(2)`，空值显示"暂未录入"
- **版本隔离**：编辑已发布原料不会影响已关联的配方

## 4. 创建配方（标准流程）

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 4.1 | 点击"创建新配方" | 跳转 `/formulas/new`，空白表单 |
| 4.2 | 填写基础信息 | 配方名称(≥2字)、业务员(必选)、成品重量(>0) |
| 4.3 | 设置系数 | 主料系数(0.15-0.25，默认0.18)、辅料系数(0.5-1.5，默认1.0) |
| 4.4 | 添加原料 | 从原料库选择，输入用量(g)，显示单价和小计 |
| 4.5 | 查看含量比校验 | 实时计算 totalRatio，显示偏差百分比和校验级别 |
| 4.6 | 查看配方报价 | 原料成本 + 包材费用 + 其他费用 + 利润率 = 最终报价 |
| 4.7 | AI 生成描述/制法 | 调用 AI 服务，自动填充字段 |
| 4.8 | 提交配方 | 保存为"草稿"状态，跳转到配方详情或列表 |

### 提交前校验（任一失败阻止提交）

- 配方名称不能为空
- 成品重量必须 > 0
- 至少一种有效原料
- 所有原料已匹配（未匹配的阻止提交）
- 含量比校验不能为 error 级别
- 升版原因不能为空（编辑模式）

## 5. 快速录入配方

快速录入是一个独立的全屏工作台，提供比标准流程更高效的配方创建体验。

**入口**：配方列表点击"快速录入" → 跳转 `/formulas/quick`

### 5.1 界面布局

| 区域 | 组件 | 说明 |
|------|------|------|
| 左侧边栏 | `QuickFormulaSidebar` | 配方列表管理，可折叠 |
| 左面板 | `FormulaDashboard` | 实时营养指标和成本概览 |
| 中面板 | `FormulaEditor` | 参数设置 + 原料表 + 费用输入 |
| 右面板 | `MaterialPool` | 可搜索/筛选的原料库 |

### 5.2 操作流程

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 5.2.1 | 点击侧边栏"新建配方" | 输入配方名称，回车确认 |
| 5.2.2 | 设置参数 | 主料系数(0.15-0.25)、辅料系数(0.5-1.5)、成品重量 |
| 5.2.3 | 从原料池添加原料 | 悬停原料鱼 → 点击"添加"，原料进入配方表 |
| 5.2.4 | 输入原料用量 | 行内编辑用量(g)，实时更新含量比和报价 |
| 5.2.5 | 调整单价（可选） | 行内编辑单价，修改后显示黄色"价"标记 |
| 5.2.6 | 设置费用 | 包材费用、其他费用、利润率 |
| 5.2.7 | 保存配方 | 点击"保存配方"，数据存入 `quick_formulas` 表 |
| 5.2.8 | 发布配方 | 点击"发布配方" → 填写业务员和描述 → 转为正式配方 |

### 5.3 原料池特色功能

**两种视图模式：**

| 模式 | 说明 |
|------|------|
| 网格视图 | 标准卡片布局，按类型分组显示 |
| 鱼池视图 | 原料以"鱼"的形式在池中游动（Boid 物理模拟），悬停显示详情 |

**筛选系统：**
- 关键词搜索（按原料名称）
- 类型筛选：全部 / 主料 / 辅料
- 高级筛选：性状、口感、功效（多选，互斥规则）

### 5.4 草稿系统

| 功能 | 说明 |
|------|------|
| 自动保存 | 修改后 2 秒自动存入 localStorage |
| 草稿恢复 | 重新进入时检测到草稿，显示恢复/丢弃横幅 |
| 防误关 | 页面关闭前弹出浏览器确认框 |

### 5.5 模板系统

| 操作 | 说明 |
|------|------|
| 保存为模板 | 将当前配方参数和原料列表保存为模板 |
| 加载模板 | 从模板列表选择，自动填充参数和原料 |
| 删除模板 | 确认后删除模板 |

### 5.6 发布流程

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 5.6.1 | 点击"发布配方" | 校验通过后弹出发布抽屉 |
| 5.6.2 | 选择业务员 | 必填，从下拉列表选择 |
| 5.6.3 | 填写配方描述 | 必填，最多 2000 字 |
| 5.6.4 | 填写制备方法（可选） | 最多 5000 字 |
| 5.6.5 | 确认发布 | 快速配方状态变为"已发布"，不可再编辑 |
| 5.6.6 | 跳转配方列表 | 可在标准配方列表中查看新发布的配方 |

### 5.7 与标准流程的区别

| 维度 | 快速录入 | 标准流程 |
|------|---------|---------|
| 布局 | 三栏工作台 | 单列表单 |
| 原料选择 | 鱼池动画 + 高级筛选 | 表单内搜索 |
| 数据存储 | `quick_formulas` 表 | `formulas` 表 |
| 草稿系统 | localStorage 自动保存 | 无本地草稿 |
| 全屏模式 | 支持浏览器全屏 | 不支持 |
| 模板系统 | 完整的保存/加载/删除 | 不支持 |
| 发布流程 | 抽屉式发布（直接发布） | 提交审批流程 |
| 状态生命周期 | new → draft → published | new → draft → pending_review → published |
| 单价编辑 | 行内编辑 + 基价追踪 + 一键恢复 | 标准表单字段 |
| 含量比校验 | 可配置的服务端阈值 | 前端硬编码阈值 |

### 5.8 关键验证点

- **名称冲突处理**：自动重试，名称后追加 `(2)`、`(3)` 等，最多重试 10 次
- **单价恢复**：修改单价后显示黄色标记，点击恢复按钮回到基价，带闪烁动画
- **含量比实时更新**：Dashboard 面板实时显示校验级别（绿/黄/橙/红）
- **发布前校验**：名称非空、至少 1 种原料、成品重量 > 0
- **发布后不可编辑**：发布操作为终态，快速配方变为只读

## 7. 提交审批

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 7.1 | 在编辑页点击"提交审批" | 弹出确认框 |
| 7.2 | 确认提交 | 版本状态从 `draft` → `pending_review` |
| 7.3 | 在版本管理页点击"提交审批" | 同上效果 |
| 7.4 | 提交后查看版本列表 | 状态显示"待审批"，操作区显示"审批中"提示 |

### 审批流程

```
配方师创建配方 → draft
    ↓
配方师编辑配方 → saves (still draft)
    ↓
配方师点击"提交审批" → status becomes pending_review
    ↓
管理员在 Dashboard "审批中心" 看到待审批项
    ↓
管理员: "批准" (approve → published) 或 "驳回" (reject → draft with comment)
    ↓
若驳回: 配方师看到"草稿"状态，可重新编辑并提交
```

| 维度 | 配方对比 | 版本对比 |

## 23. 含量比校验规则

### 计算公式

```
对每个原料 i：
  ratio_i = (quantity_i / finishedWeight) × factor_i
    其中 factor_i = supplementRatioFactor  如果 materialType === 'supplement'
                     ratioFactor            否则（herb）

totalRatio = round( Σ(ratio_i) × 100000 ) / 100000
```

### 校验级别

| 级别 | 条件 | 效果 |
|------|------|------|
| **normal** | `0.98 ≤ totalRatio ≤ 1.02` | 通过，可保存 |
| **warning** | `(0.95 ≤ totalRatio < 0.98)` 或 `(1.02 < totalRatio ≤ 1.05)` | 预警，仍可继续 |
| **high_warning** | `(0.92 ≤ totalRatio < 0.95)` 或 `(1.05 < totalRatio ≤ 1.08)` | 警告，需人工确认 |
| **error** | `totalRatio < 0.92` 或 `totalRatio > 1.08` | 失败，阻止提交 |

### 边界 Case

| Case | 输入 | 预期结果 |
|------|------|---------|
| 空原料列表 | `materials = []` | level: "none"，不阻止 |
| 成品重量为0 | `finishedWeight = 0` | level: "none"，不阻止 |
| 单原料完美配比 | 1种草药，quantity=finishedWeight，ratioFactor=0.18 | totalRatio=1.0，level: "normal" |
| 偏差刚好在边界 | totalRatio=0.98 | level: "normal"（包含边界） |
| 偏差刚好在边界 | totalRatio=1.02 | level: "normal"（包含边界） |
| 偏差略超边界 | totalRatio=0.9799 | level: "warning" |
| 错误边界 | totalRatio=0.92 | level: "high_warning"（包含边界） |
| 错误边界 | totalRatio=0.9199 | level: "error"（阻止提交） |
| NaN 输入 | quantity=NaN | totalRatio=0，level: "none" |
| Infinity 输入 | quantity=Infinity | 被 isFinite 拦截，totalRatio=0 |

### 阈值定义位置

- **统一来源**：`frontend/src/utils/ratioValidation.ts` → `RATIO_THRESHOLDS`
- **后端独立**：`backend/src/config/nutritionConstants.ts` → `RATIO_VALIDATION_THRESHOLDS`

## 24. 营养成分计算规则

### 计算流程

```
对每个原料：
  per100g = 原料营养数据（每100g）
  ratio = (quantity / finishedWeight) × ratioFactor
  贡献 = per100g × ratio

汇总：
  Σ 蛋白质 → 蛋白质总量
  Σ 脂肪 → 脂肪总量
  Σ 碳水 → 碳水总量
  Σ 钠 → 钠总量
  能量(kJ) = 蛋白质×17 + 脂肪×37 + 碳水×17
```

### 0界限归零规则

| 营养素 | 归零阈值 |
|--------|---------|
| 能量 | ≤ 17 kJ |
| 蛋白质 | ≤ 0.5 g |
| 脂肪 | ≤ 0.5 g |
| 碳水化合物 | ≤ 0.5 g |
| 钠 | ≤ 5 mg |

**归零后必须重新计算能量。**

### 每100g换算

```
每100g营养成分 = 汇总值 / (总原料量 / 100)
NRV% = (每100g营养素 / NRV参考值) × 100
```

### NRV参考值

| 营养素 | NRV参考值 |
|--------|----------|
| 能量 | 8400 kJ |
| 蛋白质 | 60 g |
| 脂肪 | 60 g |
| 碳水化合物 | 300 g |
| 钠 | 2000 mg |

### 数据展示规则

| 场景 | 兜底文案 | 精度 |
|------|---------|------|
| 金额 | `'暂未录入'` | `.toFixed(2)` |
| 空值 | `'--'` | — |
| 偏差 | — | `.toFixed(1)`% |
| 营养占比 | — | `.toFixed(2)`% |
