# 权限管理功能 — 接口文档

## 1. 角色管理 API

### 1.1 获取角色列表

**GET** `/api/roles`

**权限**：authMiddleware（登录即可访问）

**请求参数**：无

**响应示例**：

```json
{
  "success": true,
  "data": [
    {
      "id": "r_abc123",
      "name": "管理员",
      "roleKey": "admin",
      "description": "拥有系统全部权限，不受权限配置约束",
      "isSystem": true,
      "permissionCount": 18,
      "userCount": 2,
      "createdAt": "2026-05-29T00:00:00.000Z",
      "updatedAt": "2026-05-29T00:00:00.000Z"
    },
    {
      "id": "r_def456",
      "name": "配方师",
      "roleKey": "formulist",
      "description": "普通用户角色，可管理自己创建的原料和配方",
      "isSystem": true,
      "permissionCount": 11,
      "userCount": 5,
      "createdAt": "2026-05-29T00:00:00.000Z",
      "updatedAt": "2026-05-29T00:00:00.000Z"
    }
  ]
}
```

---

### 1.2 获取角色详情（含权限列表）

**GET** `/api/roles/:id`

**权限**：authMiddleware

**路径参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| id | string | 角色 ID |

**响应示例**：

```json
{
  "success": true,
  "data": {
    "id": "r_def456",
    "name": "配方师",
    "roleKey": "formulist",
    "description": "普通用户角色，可管理自己创建的原料和配方",
    "isSystem": true,
    "permissionCount": 11,
    "userCount": 5,
    "permissions": [
      { "id": "p_001", "module": "material", "action": "read", "permissionKey": "material:read", "label": "原料管理-读取" },
      { "id": "p_002", "module": "material", "action": "write", "permissionKey": "material:write", "label": "原料管理-写入" }
    ],
    "createdAt": "2026-05-29T00:00:00.000Z",
    "updatedAt": "2026-05-29T00:00:00.000Z"
  }
}
```

**错误响应**：

| HTTP | code | 说明 |
|------|------|------|
| 404 | NOT_FOUND | 角色不存在 |

---

### 1.3 创建角色

**POST** `/api/roles`

**权限**：authMiddleware + requirePermission("permission:write")

**请求体**：

```json
{
  "name": "数据分析师",
  "roleKey": "data-analyst",
  "description": "可查看数据和导出报表，不可修改配方和原料"
}
```

| 字段 | 类型 | 必填 | 验证规则 |
|------|------|------|---------|
| name | string | 是 | 2-20 字符 |
| roleKey | string | 是 | 2-30 字符，仅小写字母/数字/连字符 |
| description | string | 否 | 最多 100 字符 |

**成功响应**（201）：

```json
{
  "success": true,
  "message": "角色创建成功",
  "data": {
    "id": "r_ghi789",
    "name": "数据分析师",
    "roleKey": "data-analyst",
    "description": "可查看数据和导出报表，不可修改配方和原料",
    "isSystem": false,
    "permissionCount": 0,
    "userCount": 0,
    "createdAt": "2026-05-29T10:00:00.000Z",
    "updatedAt": "2026-05-29T10:00:00.000Z"
  }
}
```

**错误响应**：

| HTTP | code | 说明 |
|------|------|------|
| 400 | VALIDATION_ERROR | 参数验证失败 |
| 409 | DUPLICATE_ENTRY | roleKey 已存在 |

---

### 1.4 更新角色

**PUT** `/api/roles/:id`

**权限**：authMiddleware + requirePermission("permission:write")

**路径参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| id | string | 角色 ID |

**请求体**：

```json
{
  "name": "高级分析师",
  "description": "可查看数据、导出报表和修改营养分析"
}
```

| 字段 | 类型 | 必填 | 验证规则 |
|------|------|------|---------|
| name | string | 是 | 2-20 字符 |
| description | string | 否 | 最多 100 字符 |

**成功响应**：

```json
{
  "success": true,
  "message": "角色更新成功",
  "data": {
    "id": "r_ghi789",
    "name": "高级分析师",
    "roleKey": "data-analyst",
    "description": "可查看数据、导出报表和修改营养分析",
    "isSystem": false,
    "permissionCount": 6,
    "userCount": 1,
    "createdAt": "2026-05-29T10:00:00.000Z",
    "updatedAt": "2026-05-29T12:00:00.000Z"
  }
}
```

**错误响应**：

| HTTP | code | 说明 |
|------|------|------|
| 400 | VALIDATION_ERROR | 参数验证失败 |
| 404 | NOT_FOUND | 角色不存在 |
| 403 | FORBIDDEN | 系统角色 admin 不允许编辑 |

---

### 1.5 删除角色

**DELETE** `/api/roles/:id`

**权限**：authMiddleware + requirePermission("permission:write")

**路径参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| id | string | 角色 ID |

**成功响应**：

```json
{
  "success": true,
  "message": "角色删除成功"
}
```

**错误响应**：

| HTTP | code | 说明 |
|------|------|------|
| 404 | NOT_FOUND | 角色不存在 |
| 403 | FORBIDDEN | 系统角色不可删除 |
| 409 | DUPLICATE_ENTRY | 角色下仍有用户，需先变更用户角色 |

---

### 1.6 获取角色权限

**GET** `/api/roles/:id/permissions`

**权限**：authMiddleware

**路径参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| id | string | 角色 ID |

**响应示例**：

```json
{
  "success": true,
  "data": [
    { "id": "p_001", "module": "material", "action": "read", "permissionKey": "material:read", "label": "原料管理-读取" },
    { "id": "p_002", "module": "material", "action": "write", "permissionKey": "material:write", "label": "原料管理-写入" }
  ]
}
```

---

### 1.7 更新角色权限

**PUT** `/api/roles/:id/permissions`

**权限**：authMiddleware + requirePermission("permission:write")

**路径参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| id | string | 角色 ID |

**请求体**：

```json
{
  "permissionIds": ["p_001", "p_002", "p_003", "p_005", "p_006", "p_011"]
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| permissionIds | string[] | 是 | 权限 ID 数组，全量替换 |

**成功响应**：

```json
{
  "success": true,
  "message": "权限配置更新成功"
}
```

**错误响应**：

| HTTP | code | 说明 |
|------|------|------|
| 404 | NOT_FOUND | 角色不存在 |
| 403 | FORBIDDEN | admin 角色权限不可修改 |

## 3. 用户管理 API

### 3.1 获取用户列表

**GET** `/api/users`

**权限**：authMiddleware + requirePermission("user:read")

**查询参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| keyword | string | 否 | 按用户名模糊搜索 |
| roleId | string | 否 | 按角色 ID 筛选 |
| isActive | boolean | 否 | 按状态筛选（true/false） |
| page | number | 否 | 页码，默认 1 |
| pageSize | number | 否 | 每页条数，默认 20，最大 100 |

**响应示例**：

```json
{
  "success": true,
  "data": {
    "list": [
      {
        "id": "u_001",
        "username": "admin",
        "displayName": "管理员",
        "role": "admin",
        "roleId": "r_abc123",
        "roleName": "管理员",
        "isActive": true,
        "createdAt": "2026-05-01T00:00:00.000Z"
      },
      {
        "id": "u_002",
        "username": "zhangsan",
        "displayName": "张三",
        "role": "formulist",
        "roleId": "r_def456",
        "roleName": "配方师",
        "isActive": true,
        "createdAt": "2026-05-10T00:00:00.000Z"
      }
    ],
    "pagination": {
      "page": 1,
      "pageSize": 20,
      "total": 7,
      "totalPages": 1
    }
  }
}
```

---

### 3.2 变更用户角色

**PUT** `/api/users/:id/role`

**权限**：authMiddleware + requirePermission("user:write")

**路径参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| id | string | 用户 ID |

**请求体**：

```json
{
  "roleId": "r_ghi789"
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| roleId | string | 是 | 目标角色 ID |

**成功响应**：

```json
{
  "success": true,
  "message": "用户角色变更成功"
}
```

**错误响应**：

| HTTP | code | 说明 |
|------|------|------|
| 404 | NOT_FOUND | 用户或角色不存在 |
| 403 | FORBIDDEN | 不可修改自己的角色 |

---

### 3.3 禁用/启用用户

**PUT** `/api/users/:id/status`

**权限**：authMiddleware + requirePermission("user:write")

**路径参数**：

| 参数 | 类型 | 说明 |
|------|------|------|
| id | string | 用户 ID |

**请求体**：

```json
{
  "isActive": false
}
```

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| isActive | boolean | 是 | true=启用，false=禁用 |

**成功响应**：

```json
{
  "success": true,
  "message": "用户状态更新成功"
}
```

**错误响应**：

| HTTP | code | 说明 |
|------|------|------|
| 404 | NOT_FOUND | 用户不存在 |
| 403 | FORBIDDEN | 不可禁用自己 / 不可禁用 admin 用户 |

---

## 4. 错误码汇总

| 错误码 | HTTP | 说明 |
|--------|------|------|
| UNAUTHORIZED | 401 | 未认证 |
| TOKEN_EXPIRED | 401 | Token 过期 |
| FORBIDDEN | 403 | 无权限 / 系统角色受保护 / 自我操作限制 |
| NOT_FOUND | 404 | 资源不存在 |
| VALIDATION_ERROR | 400 | 参数验证失败 |
| DUPLICATE_ENTRY | 409 | 重复数据（roleKey 重复 / 角色下有用户） |
| INTERNAL_ERROR | 500 | 服务器内部错误 |
