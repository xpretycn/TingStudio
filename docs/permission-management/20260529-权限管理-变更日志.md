## 数据库变更

### 新增表

| 表名 | 说明 | 初始数据 |
|------|------|---------|
| roles | 角色表 | 2 条系统角色（admin/formulist） |
| permissions | 权限表 | 18 条权限（9 模块 × 读写） |
| role_permissions | 角色-权限关联表 | 11 条（formulist 默认权限） |

### 扩展表

| 表名 | 新增字段 | 说明 |
|------|---------|------|
| users | role_id TEXT | 关联 roles.id |
| users | is_active INTEGER | 用户状态（1=启用，0=禁用） |

## API 端点

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | /api/permissions | 权限列表（按模块分组） | 登录 |
| GET | /api/roles | 角色列表 | 登录 |
| GET | /api/roles/:id | 角色详情 | 登录 |
| POST | /api/roles | 创建角色 | permission:write |
| PUT | /api/roles/:id | 更新角色 | permission:write |
| DELETE | /api/roles/:id | 删除角色 | permission:write |
| GET | /api/roles/:id/permissions | 角色权限列表 | 登录 |
| PUT | /api/roles/:id/permissions | 更新角色权限 | permission:write |
| GET | /api/users | 用户列表（分页/筛选） | user:read |
| PUT | /api/users/:id/role | 变更用户角色 | user:write |
| PUT | /api/users/:id/status | 禁用/启用用户 | user:write |

---

## 兼容性说明

- `users.role` 字段保留，不删除
- 现有 `role !== "admin"` 硬编码判断不受影响
- admin 角色始终拥有全部权限（JWT permissions: ["*"]）
- 迁移脚本幂等，可重复执行
- 变更用户角色时同时更新 role_id 和 role 字段（双向同步）
