# 权限管理功能 — 数据库设计文档

## 1. 新增表结构

### 1.1 roles — 角色表

```sql
CREATE TABLE IF NOT EXISTS roles (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  role_key TEXT NOT NULL UNIQUE,
  description TEXT NOT NULL DEFAULT '',
  is_system INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE INDEX IF NOT EXISTS idx_roles_role_key ON roles(role_key);
CREATE INDEX IF NOT EXISTS idx_roles_is_system ON roles(is_system);
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | TEXT | PRIMARY KEY | UUID，generateId() 生成 |
| name | TEXT | NOT NULL | 角色显示名称，如"管理员"、"配方师" |
| role_key | TEXT | NOT NULL, UNIQUE | 角色唯一标识，如"admin"、"formulist" |
| description | TEXT | NOT NULL DEFAULT '' | 角色描述 |
| is_system | INTEGER | NOT NULL DEFAULT 0 | 是否系统内置角色（0=否，1=是） |
| created_at | TEXT | NOT NULL | 创建时间 |
| updated_at | TEXT | NOT NULL | 更新时间 |

---

### 1.2 permissions — 权限表

```sql
CREATE TABLE IF NOT EXISTS permissions (
  id TEXT PRIMARY KEY,
  module TEXT NOT NULL,
  action TEXT NOT NULL,
  permission_key TEXT NOT NULL UNIQUE,
  label TEXT NOT NULL,
  description TEXT NOT NULL DEFAULT '',
  sort_order INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);

CREATE INDEX IF NOT EXISTS idx_permissions_module ON permissions(module);
CREATE INDEX IF NOT EXISTS idx_permissions_permission_key ON permissions(permission_key);
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| id | TEXT | PRIMARY KEY | UUID |
| module | TEXT | NOT NULL | 模块名，如"material"、"formula" |
| action | TEXT | NOT NULL | 操作类型，"read" 或 "write" |
| permission_key | TEXT | NOT NULL, UNIQUE | 权限唯一标识，如"material:read" |
| label | TEXT | NOT NULL | 显示名称，如"原料管理-读取" |
| description | TEXT | NOT NULL DEFAULT '' | 权限说明 |
| sort_order | INTEGER | NOT NULL DEFAULT 0 | 排序权重 |
| created_at | TEXT | NOT NULL | 创建时间 |

---

### 1.3 role_permissions — 角色-权限关联表

```sql
CREATE TABLE IF NOT EXISTS role_permissions (
  role_id TEXT NOT NULL,
  permission_id TEXT NOT NULL,
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  PRIMARY KEY (role_id, permission_id),
  FOREIGN KEY (role_id) REFERENCES roles(id) ON DELETE CASCADE,
  FOREIGN KEY (permission_id) REFERENCES permissions(id) ON DELETE CASCADE
);

CREATE INDEX IF NOT EXISTS idx_role_permissions_role_id ON role_permissions(role_id);
CREATE INDEX IF NOT EXISTS idx_role_permissions_permission_id ON role_permissions(permission_id);
```

| 字段 | 类型 | 约束 | 说明 |
|------|------|------|------|
| role_id | TEXT | NOT NULL, FK | 关联 roles.id |
| permission_id | TEXT | NOT NULL, FK | 关联 permissions.id |
| created_at | TEXT | NOT NULL | 创建时间 |

**联合主键**：(role_id, permission_id)，同一角色同一权限只能存在一条记录。

## 2. 现有表扩展

### 2.1 users 表新增字段

```sql
-- 新增 role_id 字段（如不存在）
ALTER TABLE users ADD COLUMN role_id TEXT;

-- 新增 is_active 字段（如不存在）
ALTER TABLE users ADD COLUMN is_active INTEGER NOT NULL DEFAULT 1;
```

| 新增字段 | 类型 | 约束 | 说明 |
|---------|------|------|------|
| role_id | TEXT | 可空 | 关联 roles.id，为空时使用 role 字段兼容 |
| is_active | INTEGER | NOT NULL DEFAULT 1 | 用户状态（1=启用，0=禁用） |

**兼容策略**：
- 保留 `users.role` 字段不删除
- `role_id` 优先，为空时 fallback 到 `role` 字段
- 变更用户角色时同时更新 `role_id` 和 `role` 字段（双向同步）

### 3.4 用户数据迁移

```sql
UPDATE users SET role_id = (SELECT id FROM roles WHERE role_key = users.role)
WHERE role_id IS NULL;
```

## 4. ER 关系图

```
┌──────────────┐       ┌──────────────────────┐       ┌──────────────┐
│    users     │       │   role_permissions   │       │ permissions  │
├──────────────┤       ├──────────────────────┤       ├──────────────┤
│ id (PK)      │       │ role_id (PK, FK) ────┼───────│ id (PK)      │
│ username     │       │ permission_id (PK,FK)│       │ module       │
│ password     │       │ created_at           │       │ action       │
│ role         │       └──────────┬───────────┘       │ permission_key│
│ role_id (FK) ┼──────┐            │                   │ label        │
│ is_active    │      │            │                   │ description  │
│ display_name │      │            │                   │ sort_order   │
│ ...          │      │            │                   │ created_at   │
└──────────────┘      │            │                   └──────────────┘
                      │            │
                      ▼            ▼
              ┌──────────────────────┐
              │        roles        │
              ├──────────────────────┤
              │ id (PK)             │
              │ name                │
              │ role_key (UNIQUE)   │
              │ description         │
              │ is_system           │
              │ created_at          │
              │ updated_at          │
              └──────────────────────┘
```

**关系说明**：
- `users.role_id` → `roles.id`：多对一，一个角色可分配给多个用户
- `role_permissions.role_id` → `roles.id`：多对多中间表
- `role_permissions.permission_id` → `permissions.id`：多对多中间表
- `roles` 与 `permissions` 通过 `role_permissions` 实现多对多关系

## 5. 查询性能优化

### 5.1 索引设计

| 表 | 索引 | 用途 |
|----|------|------|
| roles | idx_roles_role_key | role_key 唯一查找 |
| roles | idx_roles_is_system | 系统角色过滤 |
| permissions | idx_permissions_module | 按模块分组查询 |
| permissions | idx_permissions_permission_key | permission_key 唯一查找 |
| role_permissions | idx_role_permissions_role_id | 按角色查权限 |
| role_permissions | idx_role_permissions_permission_id | 按权限查角色 |

### 5.2 高频查询

| 查询 | 频率 | 优化 |
|------|------|------|
| 角色列表（含权限数、用户数） | 高 | 使用子查询 COUNT，避免 N+1 |
| 角色权限列表 | 高 | JOIN role_permissions + permissions |
| 用户登录时查权限 | 高 | 缓存到 JWT，7 天内不查库 |
| 用户列表（含角色名） | 中 | LEFT JOIN roles |

## 6. 数据完整性约束

| 约束 | 实现 |
|------|------|
| role_key 唯一 | UNIQUE 约束 |
| permission_key 唯一 | UNIQUE 约束 |
| 角色删除级联 | ON DELETE CASCADE（role_permissions 自动清理） |
| 用户 role_id 外键 | 应用层校验（SQLite 不强制 FK） |
| 系统角色保护 | 应用层校验（is_system = 1 不可删除/编辑权限） |
| admin 用户保护 | 应用层校验（不可禁用 admin 用户） |
