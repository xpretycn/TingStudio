## 2. 数据库设计

### 2.1 新增表

#### roles 表

```sql
CREATE TABLE IF NOT EXISTS roles (
  id TEXT PRIMARY KEY,
  name TEXT NOT NULL,
  role_key TEXT NOT NULL UNIQUE,
  description TEXT DEFAULT '',
  is_system INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  updated_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT | UUID，generateId() 生成 |
| name | TEXT | 角色显示名称，如"管理员"、"配方师" |
| role_key | TEXT | 角色唯一标识，如"admin"、"formulist"，UNIQUE |
| description | TEXT | 角色描述 |
| is_system | INTEGER | 是否系统内置角色（0/1），系统角色不可删除 |
| created_at | TEXT | 创建时间 |
| updated_at | TEXT | 更新时间 |

#### permissions 表

```sql
CREATE TABLE IF NOT EXISTS permissions (
  id TEXT PRIMARY KEY,
  module TEXT NOT NULL,
  action TEXT NOT NULL,
  permission_key TEXT NOT NULL UNIQUE,
  label TEXT NOT NULL,
  description TEXT DEFAULT '',
  sort_order INTEGER NOT NULL DEFAULT 0,
  created_at TEXT NOT NULL DEFAULT (datetime('now'))
);
```

| 字段 | 类型 | 说明 |
|------|------|------|
| id | TEXT | UUID |
| module | TEXT | 模块名，如"material"、"formula" |
| action | TEXT | 操作类型，"read" 或 "write" |
| permission_key | TEXT | 权限唯一标识，如"material:read"，UNIQUE |
| label | TEXT | 显示名称，如"原料管理-读取" |
| description | TEXT | 权限说明 |
| sort_order | INTEGER | 排序权重 |
| created_at | TEXT | 创建时间 |

#### role_permissions 表

```sql
CREATE TABLE IF NOT EXISTS role_permissions (
  role_id TEXT NOT NULL,
  permission_id TEXT NOT NULL,
  created_at TEXT NOT NULL DEFAULT (datetime('now')),
  PRIMARY KEY (role_id, permission_id),
  FOREIGN KEY (role_id) REFERENCES roles(id) ON DELETE CASCADE,
  FOREIGN KEY (permission_id) REFERENCES permissions(id) ON DELETE CASCADE
);
```

### 2.2 users 表扩展

新增两个字段：

```sql
ALTER TABLE users ADD COLUMN role_id TEXT;
ALTER TABLE users ADD COLUMN is_active INTEGER NOT NULL DEFAULT 1;
```

| 字段 | 类型 | 说明 |
|------|------|------|
| role_id | TEXT | 关联 roles.id，为空时使用 role 字段兼容 |
| is_active | INTEGER | 用户状态（1=启用，0=禁用） |

### 2.3 迁移策略

- 迁移脚本：`backend/src/scripts/migrations/addRbacTables.ts`
- 幂等执行：所有 `CREATE TABLE IF NOT EXISTS`，`ALTER TABLE` 前检查列是否存在
- 数据迁移：将现有 `users.role` 映射到 `roles` 表，填充 `role_id`
- 初始化数据：插入系统角色（admin/formulist）和 18 个权限记录
- 兼容策略：保留 `users.role` 字段不删除，`role_id` 优先，为空时 fallback 到 `role`

### 3.2 认证中间件改造

#### generateToken() 改造

```typescript
// 改造前
export function generateToken(payload: { userId: string; username: string; role: string }): string {
  return jwt.sign({
    sub: payload.userId,
    userId: payload.userId,
    username: payload.username,
    role: payload.role,
    permissions: payload.role === "admin" ? ["*"] : [],
  }, JWT_SECRET, { expiresIn: JWT_EXPIRES_IN });
}

// 改造后
export async function generateToken(payload: { userId: string; username: string; role: string; roleId?: string }): Promise<string> {
  let permissions: string[] = [];

  if (payload.role === "admin") {
    permissions = ["*"];
  } else if (payload.roleId) {
    permissions = await getPermissionsByRoleId(payload.roleId);
  }

  return jwt.sign({
    sub: payload.userId,
    userId: payload.userId,
    username: payload.username,
    role: payload.role,
    roleId: payload.roleId,
    permissions,
  }, JWT_SECRET, { expiresIn: JWT_EXPIRES_IN });
}
```

**关键变更**：
- `generateToken` 变为 `async` 函数
- 新增 `roleId` 参数
- 非 admin 用户从 `role_permissions` 表查询实际权限列表
- JWT payload 新增 `roleId` 字段

#### requirePermission() 改造

```typescript
// 改造后 — 逻辑不变，admin 仍然绕过
export function requirePermission(permission: string) {
  return (req: AuthenticatedRequest, res: Response, next: NextFunction): void => {
    if (!req.user) {
      res.status(401).json({ success: false, error: { message: "Authentication required", code: "UNAUTHORIZED" } });
      return;
    }
    if (!req.user.permissions.includes(permission) && req.user.role !== "admin") {
      res.status(403).json({ success: false, error: { message: `Insufficient permissions. Required: ${permission}`, code: "FORBIDDEN" } });
      return;
    }
    next();
  };
}
```

**注意**：`requirePermission` 逻辑保持不变。admin 角色通过 `role !== "admin"` 判断绕过，非 admin 角色通过 JWT 中的 `permissions` 数组判断。

#### AuthenticatedRequest 接口扩展

```typescript
interface AuthenticatedRequest extends Request {
  user?: {
    userId: string;
    role: string;
    roleId?: string;
    permissions: string[];
  };
}
```

### 3.4 Service 层设计

#### roleService.ts

| 方法 | 功能 | SQL |
|------|------|-----|
| `getAllRoles()` | 获取所有角色列表（含权限数和用户数） | SELECT roles + COUNT(permissions) + COUNT(users) |
| `getRoleById(id)` | 获取单个角色详情 | SELECT + JOIN role_permissions |
| `createRole(data)` | 创建新角色 | INSERT INTO roles |
| `updateRole(id, data)` | 更新角色名称/描述 | UPDATE roles |
| `deleteRole(id)` | 删除自定义角色 | DELETE FROM roles (CASCADE) |
| `getRolePermissions(roleId)` | 获取角色的权限列表 | SELECT FROM role_permissions JOIN permissions |
| `updateRolePermissions(roleId, permissionIds)` | 更新角色权限（全量替换） | DELETE + INSERT role_permissions |
| `getRoleUserCount(roleId)` | 获取角色关联用户数 | SELECT COUNT FROM users WHERE role_id = ? |

#### permissionService.ts

| 方法 | 功能 | SQL |
|------|------|-----|
| `getAllPermissions()` | 获取所有权限（按模块分组） | SELECT * FROM permissions ORDER BY sort_order |
| `getPermissionsByRoleId(roleId)` | 获取角色权限 key 列表 | SELECT permission_key FROM role_permissions JOIN permissions |

#### userService.ts

| 方法 | 功能 | SQL |
|------|------|-----|
| `getUserList(filters)` | 分页查询用户列表 | SELECT + WHERE + LIMIT/OFFSET |
| `updateUserRole(userId, roleId)` | 变更用户角色 | UPDATE users SET role_id, role |
| `toggleUserActive(userId, isActive)` | 禁用/启用用户 | UPDATE users SET is_active |
| `getUserCountByRoleId(roleId)` | 按角色统计用户数 | SELECT COUNT |

## 5. 数据迁移方案

### 5.1 迁移脚本

文件：`backend/src/scripts/migrations/addRbacTables.ts`

执行步骤：
1. 创建 `roles` 表
2. 创建 `permissions` 表
3. 创建 `role_permissions` 表
4. 插入系统角色（admin、formulist）
5. 插入 18 条权限记录
6. 为 formulist 角色分配默认权限
7. 为 `users` 表新增 `role_id` 列（如不存在）
8. 为 `users` 表新增 `is_active` 列（如不存在）
9. 迁移现有用户：`UPDATE users SET role_id = (SELECT id FROM roles WHERE role_key = users.role)`

### 5.2 兼容性保证

- `users.role` 字段保留，不删除
- 登录时优先使用 `role_id` 查询权限，`role_id` 为空时 fallback 到 `role` 字段
- 现有 `role !== "admin"` 判断不受影响
- 迁移脚本幂等，可重复执行

## 6. 安全设计

| 安全点 | 措施 |
|--------|------|
| API 鉴权 | 所有权限管理 API 需要 admin 角色或对应模块权限 |
| 自我保护 | 不可禁用自己、不可修改自己的角色 |
| admin 保护 | admin 角色不可删除、不可禁用 admin 用户、admin 权限不可编辑 |
| 角色删除保护 | 删除角色前必须先解除关联用户 |
| SQL 注入 | 全部参数化查询 |
| 输入验证 | validateBody 中间件校验所有输入 |
