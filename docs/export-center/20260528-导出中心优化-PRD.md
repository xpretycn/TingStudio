## 1. 执行摘要

### 1.1 问题陈述

当前导出中心存在以下核心问题：

**架构层面**：

- **流程断裂**：FormulaList.vue 的"批量导出"按钮通过 `router.push({ path: '/exports', query: { formulaIds } })` 跳转到导出中心，但导出中心未读取 URL 参数，导致跳转后丢失已选配方信息，用户需要重新选择
- **空实现**：MaterialList.vue 的"批量导出"按钮仅弹出 `MessagePlugin.success` 提示，无实际导出功能
- **多配方仅导出第1个**：导出中心创建任务时，即使选择多个配方，后端仅处理第一个 `formulaId`
- **职责混乱**：ExportCenter.vue 同时承担"导出操作入口"和"管理视图"两个角色，导致 formulist 用户被迫进入管理界面才能执行日常导出操作

**数据层面**：

- Dashboard 数据看板依赖前端内存中的分页数据计算统计值（非真实汇总）
- 配方筛选仅支持简单下拉选择
- 导出维度仅限单配方
- 模板管理缺少分类管理
- 系统配置缺失

**技术层面**：

- 存在 5 个已知 Bug（下载路径双前缀、控制器直连数据库、MySQL/SQLite 表结构不兼容、公开分享路由未实现、export_type CHECK 约束包含未实现的 `api` 类型）

### 1.2 解决方案

采用**双轨制架构**重构导出体系：

- **轨道一：业务页面就地导出** — FormulaList.vue 和 MaterialList.vue 成为导出操作的主入口，用户在业务页面直接发起导出，无需跳转到导出中心。周报/月报导出保留在导出中心（因为不属于特定业务页面）
- **轨道二：导出中心管理视图** — ExportCenter.vue 重新定位为 admin 管理模块，仅 admin 可访问，用于日常维护查看导出情况、管理模板和分享链接。**v2.1 变更**：导出中心移入系统管理模块，作为 SystemConfig.vue 的子 Tab（"导出中心"Tab），不再作为独立路由页面。导出配置从 SystemConfig.vue 的独立 Tab 移回 ExportCenter.vue，形成逻辑闭环——admin 在导出中心内既能管理导出任务/模板/分享，又能配置导出参数

同时进行五个模块的全面优化与增强：① 业务页面就地导出集成；② 导出中心管理视图重构；③ 数据来源真实化；④ 导出任务维度扩展；⑤ 模板管理系统化。修复全部已知 Bug。

### 1.3 成功指标

|------|--------|----------|

**现状分析**：

当前 [FormulaList.vue](file:///d:/Program%20Data/workspace-codebd/TingStudio/frontend/src/views/formulas/FormulaList.vue) 的批量导出功能存在严重缺陷：

```typescript
const handleBatchExport = () => {
  if (selectedRows.value.length === 0) return;
  const ids = selectedRows.value.map(f => f.id).join(',');
  router.push({ path: '/exports', query: { formulaIds: ids } });
  MessagePlugin.success(`已选择 ${selectedRows.value.length} 个配方进行导出`);
  clearSelection();

```

**问题**：
1. 跳转到导出中心后，URL 参数 `formulaIds` 未被读取，用户需要重新选择配方
2. 多配方仅导出第1个（后端 `createExportJob` 只处理单个 `formulaId`）
3. 用户体验断裂：选择 → 跳转 → 重新选择 → 导出

**需求规格**：

**工具栏改造**：

将现有批量操作栏中的"批量导出"按钮改为就地导出入口，不再跳转到导出中心。同时在工具栏右侧新增独立的"导出"按钮（支持未选择行时点击，弹出导出抽屉后选择配方）。

|------|------|----------|------|
| "导出"按钮 | 工具栏右侧（与"新增配方"并列） | 始终可见 | 打开导出抽屉，预填当前搜索结果 |
| "批量导出"按钮 | 批量操作栏（选中行后显示） | `selectedRows.length > 0` | 打开导出抽屉，预填已选配方 |

**导出抽屉/对话框 UI 规格**：


**交互流程**：

1. 用户在 FormulaList 中选择配方行 → 点击"批量导出" → 打开导出抽屉，已选配方自动填入
2. 或用户直接点击工具栏"导出"按钮 → 打开导出抽屉，配方列表为空，点击"+ 添加配方"弹出配方选择弹窗
3. 用户在导出抽屉中选择导出格式、模板（可选）、是否包含版本信息
4. 点击"开始导出" → 调用 `POST /api/exports/jobs` API
5. 导出完成后：
   - 自动下载文件（浏览器触发下载）
   - `MessagePlugin.success('导出完成：养生茶A.pdf')` 提示
   - 任务记录写入 `export_jobs` 表
6. 导出失败时：`MessagePlugin.error('导出失败：{errorMessage}')` 提示

```typescript
const result = await exportStore.createJob({
```

## 附录 C: 导出文件内容规范

### 配方导出 (formula)

| 区域 | 内容 |
|------|------|
| 表头 | 配方名称、编码、状态、业务员、创建人、创建日期 |
| 原料明细 | 原料名称、用量、单位、比例因子 |
| 营养汇总 | 每 100g 营养素含量、NRV% |
| 版本信息 | 版本号、变更日期、变更说明（includeVersionInfo = true 时） |

### 原料导出 (material)

| 区域 | 内容 |
|------|------|
| 表头 | 原料名称、编码、类别、供应商、创建日期 |
| 营养成分 | 每 100g 营养素含量 |
| 版本信息 | 版本号、变更日期 |

### 周报导出 (weekly-report)

| 区域 | 内容 |
|------|------|
| 报告标题 | XX周报 (YYYY-MM-DD ~ YYYY-MM-DD) |
| 配方汇总 | 新增配方数、发布配方数、审核中配方数 |
| 导出统计 | 本周导出次数、按维度分布 |
| 业务员排名 | 按配方数量/导出次数排序 |

### 月报导出 (monthly-report)

| 区域 | 内容 |
|------|------|
| 报告标题 | XX月报 (YYYY-MM) |
| 月度汇总 | 新增/发布/审核配方数、导出总量 |
| 趋势对比 | 与上月对比变化 |
| 业务员排名 | 按配方数量/导出次数排序 |
| 原料使用统计 | 高频原料 Top 10 |
