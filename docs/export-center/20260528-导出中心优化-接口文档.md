## 1. 概述

本次"导出中心优化"项目采用**双轨制架构**，对导出模块进行以下核心改造：

| 改造方向 | 说明 |
|---------|------|
| **双轨制架构** | 轨道一：业务页面就地导出（FormulaList/MaterialList 调用 `POST /api/exports/jobs`）；轨道二：导出中心管理视图（ExportCenter.vue，admin only）查看管理导出情况 |
| **移除 API 接口管理** | 移除 `POST /api/exports/api-interfaces` 和 `GET /api/exports/api-interfaces` 端点，统计响应移除 `apiCount` 字段，后端 `api_data_interfaces` 表保留但标记废弃 |
| **导出中心定位为管理模块** | ExportCenter.vue 仅 admin 可访问；导出任务创建从业务页面发起（FormulaList/MaterialList）；周报/月报导出从导出中心发起（admin） |
| **导出中心移入系统管理模块** | v2.1 变更：ExportCenter.vue 从独立页面移入系统管理模块（SystemConfig 内嵌 Tab），导出配置 API 调用方由 SystemConfig.vue 变为 ExportCenter.vue，统计 API 调用方由独立页面变为 SystemConfig 内嵌 Tab；移除前端 `/exports` 独立路由 |
| **数据分类导出** | 新增 `dataCategory` 字段，支持配方、原料、周报、月报四类数据导出 |
| **批量导出** | 配方/原料导出由单条改为批量（`formulaIds` / `materialIds` 数组） |
| **版本信息** | 新增 `includeVersionInfo` 开关，控制是否在导出中包含版本数据 |
| **报表导出** | 新增周报/月报导出能力，需传入 `periodStart` / `periodEnd` |
| **模板分类** | 模板新增 `category` 字段，按业务分类管理 |
| **权限收紧** | 模板 CRUD 权限收紧为 admin only |
| **公开分享** | 新增无需认证的公开分享访问端点 |
| **数据看板** | 新增统计接口和配置接口，支撑导出中心仪表盘 |
| **Bug 修复** | 修复前端下载路径双重 `/api` 前缀问题 |

## 2. 通用约定

### 2.1 基础路径

所有接口挂载在 `/api/exports` 前缀下，由 `backend/src/routes/exports.ts` 定义，在 `backend/src/routes/index.ts` 的 `createAppRouter()` 中注册。

### 2.2 认证方式

| 认证方式 | 说明 |
|---------|------|
| **需认证（任意角色）** | 请求头携带 `Authorization: Bearer <JWT_TOKEN>` |
| **需认证（仅 admin）** | 同上，且 `req.user.role` 必须为 `admin` |
| **无需认证** | 公开端点，不校验 Token |

### 2.3 响应格式

**成功响应**：

```json
{
  "success": true,
  "data": { ... }
}
```

**分页响应**：

```json
{
  "success": true,
  "data": {
    "list": [...],
    "pagination": {
      "page": 1,
      "pageSize": 20,
      "total": 100,
      "totalPages": 5
    }
  }
}
```

**错误响应**：

```json
{
  "success": false,
  "error": {
    "message": "Error description",
    "code": "ERROR_CODE",
    "timestamp": "2026-05-28T00:00:00.000Z"
  }
}
```

### 2.4 请求验证

使用 `validateBody()` 中间件验证请求体，验证规则定义在路由层。

### 2.5 字段命名

- 数据库字段：`snake_case`（如 `data_category`、`target_ids_json`）
- API 响应字段：`camelCase`（如 `dataCategory`、`targetIds`）
- 转换工具：`rowToCamelCase()` / `rowsToCamelCase()`

## 4. 新增接口

### 4.1 获取导出中心统计

获取导出中心仪表盘统计数据，包括任务统计、分享统计、模板统计和最近活动。

| 属性 | 值 |
|------|-----|
| **方法** | `GET` |
| **路径** | `/api/exports/statistics` |
| **认证** | 需认证（任意角色） |
| **权限** | admin 可见全部数据，formulist 仅见自己创建的数据 |

#### 请求参数

无。

#### 响应格式

```json
{
  "success": true,
  "data": {
    "totalJobs": 128,
    "completedJobs": 102,
    "failedJobs": 8,
    "processingJobs": 3,
    "activeShares": 15,
    "templateCount": 6,
    "recentActivities": [
      {
        "type": "export",
        "title": "配方导出",
        "desc": "EXCEL 导出完成",
        "time": "2026-05-28T10:30:00.000Z"
      },
      {
        "type": "share",
        "title": "分享链接",
        "desc": "创建公开分享",
        "time": "2026-05-28T09:15:00.000Z"
      },
      {
        "type": "export",
        "title": "原料导出",
        "desc": "PDF 导出完成",
        "time": "2026-05-27T16:45:00.000Z"
      }
    ]
  }
}
```

#### 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `totalJobs` | number | 导出任务总数 |
| `completedJobs` | number | 已完成任务数 |
| `failedJobs` | number | 失败任务数 |
| `processingJobs` | number | 处理中任务数 |

| `recentActivities[].desc` | string | 活动描述 |
| `recentActivities[].time` | string | 活动时间（ISO 8601） |

#### 与旧版差异

| 字段 | 旧版 | 新版 | 说明 |
|------|------|------|------|
| `apiCount` | 存在（number） | ❌ 已移除 | API 接口管理功能已废弃，不再统计 |
| `recentActivities` | 存在 | ✅ 保留 | 活动类型仅保留 `export` 和 `share`，移除 `template` 和 `api` |

#### 错误响应

| HTTP 状态码 | 错误码 | 说明 |
|------------|--------|------|
| 401 | `UNAUTHORIZED` | 未认证 |
| 500 | `INTERNAL_ERROR` | 服务器内部错误 |

### 4.3 更新导出中心配置

批量更新导出中心配置项，仅管理员可操作。

> **v2.1 调用方变更**：本接口原由 SystemConfig.vue 调用，现改为由 ExportCenter.vue（系统管理模块内嵌）调用。

| 属性 | 值 |
|------|-----|
| **方法** | `PUT` |
| **路径** | `/api/exports/config` |
| **认证** | 需认证（admin only） |

#### 请求参数

**Body（JSON）**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `configs` | array | 是 | 配置项列表 |
| `configs[].configKey` | string | 是 | 配置项键名 |
| `configs[].configValue` | string | 是 | 配置项值 |

**验证规则**（`validateBody`）：

- `configs`：必填，类型 `array`

#### 请求示例

```json
{
  "configs": [
    {
      "configKey": "export_rate_limit",
      "configValue": "20"
    },
    {
      "configKey": "auto_delete_days",
      "configValue": "60"
    }
  ]
}
```

#### 响应格式

```json
{
  "success": true,
  "data": {
    "updatedCount": 2
  }
}
```

#### 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `updatedCount` | number | 成功更新的配置项数量 |

#### 业务规则

- 仅更新 `export_center_config` 表中已存在的配置项，不存在的 `configKey` 会被跳过
- `configType` 为 `number` 时，`configValue` 必须可转为数字，否则跳过
- `configType` 为 `boolean` 时，`configValue` 必须为 `"true"` 或 `"false"`，否则跳过

#### 错误响应

| HTTP 状态码 | 错误码 | 说明 |
|------------|--------|------|
| 400 | `VALIDATION_ERROR` | 参数验证失败（缺少 configs） |
| 401 | `UNAUTHORIZED` | 未认证 |
| 403 | `FORBIDDEN` | 非 admin 角色 |
| 500 | `INTERNAL_ERROR` | 服务器内部错误 |

### 4.4 公开分享访问

无需认证的公开分享访问端点，用于外部用户通过分享链接访问导出内容。

| 属性 | 值 |
|------|-----|
| **方法** | `GET` |
| **路径** | `/api/exports/public/share/:shareId` |
| **认证** | 无需认证 |

> ⚠️ **重要**：该端点必须在 `authMiddleware` 之外注册，否则会因全局认证中间件拦截而无法访问。当前路由文件已将此端点注册在 `authMiddleware` 之前。

#### 请求参数

**路径参数**：

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `shareId` | string | 是 | 分享 ID |

#### 响应格式

**情况一：分享有效，返回分享详情和关联配方数据**

```json
{
  "success": true,
  "data": {
    "shareId": "share_abc123",
    "shareUrl": "/share/abc123",
    "formulaId": "f_001",
    "versionId": null,
    "shareType": "link",
    "password": null,
    "expireDate": "2026-06-04",
    "allowedEmails": [],
    "downloadLimit": null,
    "downloadCount": 1,
    "formula": {
      "id": "f_001",
      "name": "枸杞养生茶",
      "code": "GQ001"
    }
  }
}
```

**情况二：分享已过期或达到下载上限**

```json
{
  "success": false,
  "error": {
    "message": "分享链接已过期",
    "code": "GONE",
    "timestamp": "2026-05-28T00:00:00.000Z"
  }
}
```

#### 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `shareId` | string | 分享 ID |
| `shareUrl` | string | 分享访问路径 |
| `formulaId` | string | 关联配方 ID |
| `versionId` | string \| null | 版本 ID |
| `shareType` | string | 分享类型：`link` / `email` |
| `password` | string \| null | 访问密码 |
| `expireDate` | string \| null | 过期日期 |
| `allowedEmails` | string[] | 允许访问的邮箱列表 |
| `downloadLimit` | number \| null | 下载次数上限 |
| `downloadCount` | number | 已下载次数 |
| `formula` | object \| null | 关联配方数据（camelCase 字段） |

#### 业务规则

- 每次访问自动将 `download_count` 加 1
- `expireDate` 不为空且已过期时，返回 410
- `downloadLimit` 不为空且 `downloadCount >= downloadLimit` 时，返回 410

#### 错误响应

| HTTP 状态码 | 错误码 | 说明 |
|------------|--------|------|
| 404 | `NOT_FOUND` | 分享不存在 |
| 410 | — | 分享已过期或下载次数已达上限 |

| 属性 | 值 |

#### 调用方说明

#### 变更说明

| 变更项 | 旧版 | 新版 | 影响 |
|--------|------|------|------|
| 数据分类 | 无（仅支持配方） | 新增 `dataCategory` 必填字段 | **破坏性**：旧版不传此字段将返回 400 |
| 配方 ID | `formulaId`（单条 string） | `formulaIds`（数组 string[]） | **破坏性**：字段名和类型均变更 |
| 版本信息 | `versionId`（单条） | `includeVersionInfo`（布尔开关） | **破坏性**：语义变更 |

| 报表导出 | 不支持 | 新增 `periodStart` / `periodEnd` | 新增功能 |

#### 请求参数

**Body（JSON）**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| `dataCategory` | string | 是 | 数据分类：`formula` / `material` / `weekly-report` / `monthly-report` |
| `exportType` | string | 是 | 导出类型：`excel` / `pdf` |
| `formulaIds` | string[] | 条件必填 | 配方 ID 列表（`dataCategory=formula` 时必填） |
| `materialIds` | string[] | 条件必填 | 原料 ID 列表（`dataCategory=material` 时必填） |
| `includeVersionInfo` | boolean | 否 | 是否包含版本数据（默认 `false`） |
| `periodStart` | string | 条件必填 | 周期起始日期（`dataCategory` 为 `weekly-report` 或 `monthly-report` 时必填，ISO 8601 日期格式） |
| `periodEnd` | string | 条件必填 | 周期结束日期（`dataCategory` 为 `weekly-report` 或 `monthly-report` 时必填，ISO 8601 日期格式） |
| `templateId` | string | 否 | 导出模板 ID |

**验证规则**（`validateBody`）：

| 字段 | 规则 |
|------|------|
| `dataCategory` | 必填，枚举值：`formula` / `material` / `weekly-report` / `monthly-report` |
| `exportType` | 必填，枚举值：`excel` / `pdf` |
| `formulaIds` | 当 `dataCategory=formula` 时必填，类型 `array`，最少 1 项 |
| `materialIds` | 当 `dataCategory=material` 时必填，类型 `array`，最少 1 项 |
| `periodStart` | 当 `dataCategory` 为 `weekly-report` 或 `monthly-report` 时必填 |
| `periodEnd` | 当 `dataCategory` 为 `weekly-report` 或 `monthly-report` 时必填 |

#### 请求示例

**配方导出（由 FormulaList.vue 发起）**：

```json
{
  "dataCategory": "formula",
  "exportType": "excel",
  "formulaIds": ["f_001", "f_002", "f_003"],

  "templateId": "tpl_standard_excel"
}
```

**原料导出（由 MaterialList.vue 发起）**：

```json
{
  "dataCategory": "material",
  "exportType": "pdf",
  "materialIds": ["mat_001", "mat_005"],
  "includeVersionInfo": false
}
```

**周报导出（由 ExportCenter.vue admin 发起）**：

```json
{
  "dataCategory": "weekly-report",
  "exportType": "excel",
  "periodStart": "2026-05-19",
  "periodEnd": "2026-05-25"
}
```

**月报导出（由 ExportCenter.vue admin 发起）**：

```json
{
  "dataCategory": "monthly-report",
  "exportType": "pdf",
  "periodStart": "2026-05-01",
  "periodEnd": "2026-05-31"
}
```

#### 响应格式

**导出成功**：

```json
{
  "success": true,
  "data": {
    "jobId": "job_abc123",
    "status": "completed",
    "fileName": "枸杞养生茶_红枣桂圆茶_20260528.xlsx"
  }
}
```

**导出失败**：

```json
{
  "success": true,
  "data": {
    "jobId": "job_abc123",
    "status": "failed",
    "errorMessage": "配方数据不存在，请检查配方是否已被删除"
  }
}
```

#### 响应字段说明

| 字段 | 类型 | 说明 |
|------|------|------|
| `jobId` | string | 导出任务 ID |
| `status` | string | 任务状态：`completed` / `failed` |
| `fileName` | string | 导出文件名（status 为 completed 时返回） |
| `errorMessage` | string | 错误信息（status 为 failed 时返回） |

#### 业务规则

- 任务创建后状态为 `processing`，同步执行导出逻辑
- 导出完成后状态更新为 `completed`，文件写入 `exports/` 目录
- 导出失败时状态更新为 `failed`，`error_message` 记录用户友好的错误描述
- 错误消息自动转换：数据库异常 → "系统数据库连接异常"，数据不存在 → "XX数据不存在，请检查XX是否已被删除"
- 仅支持 `excel` 和 `pdf` 两种导出格式，其他值直接返回 failed

#### 错误响应

| HTTP 状态码 | 错误码 | 说明 |
|------------|--------|------|
| 400 | `VALIDATION_ERROR` | 参数验证失败（缺少 dataCategory、exportType 等） |
| 401 | `UNAUTHORIZED` | 未认证 |
| 500 | `INTERNAL_ERROR` | 服务器内部错误 |

#### 旧版兼容说明

> ⚠️ **破坏性变更**：旧版请求体 `{ formulaId, versionId, exportType }` 将不再兼容。前端需同步更新为新的请求格式。建议在过渡期对 `dataCategory` 缺失的请求自动推断为 `formula`，对 `formulaId` 自动转为 `formulaIds: [formulaId]`，实现向后兼容。

## 7. 缺陷修复

### 7.1 前端下载路径双重 `/api` 前缀

**问题描述**：

前端 `frontend/src/api/export.ts` 中 `downloadFile` 方法的请求路径为：

```typescript
// ❌ 错误：双重 /api 前缀
downloadFile(jobId: string) {
  return http.get(`/api/exports/jobs/${jobId}/download`, {
    responseType: 'blob',
    headers: { Authorization: `Bearer ${localStorage.getItem(TOKEN_KEY) || ''}` },
  })
}
```

由于 `http.ts` 中的 axios 实例 `baseURL` 已配置为 `/api`，实际请求路径变为 `/api/api/exports/jobs/:jobId/download`，导致 404。

**修复方案**：

移除路径中多余的 `/api` 前缀：

```typescript
// ✅ 正确
downloadFile(jobId: string) {
  return http.get(`/exports/jobs/${jobId}/download`, {
    responseType: 'blob',
    headers: { Authorization: `Bearer ${localStorage.getItem(TOKEN_KEY) || ''}` },
  })
}
```

**影响范围**：仅前端代码，后端接口无变更。
