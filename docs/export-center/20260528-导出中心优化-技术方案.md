## 1. 项目背景与目标

### 1.1 背景：双轨制架构的必要性分析

当前导出中心存在严重的**架构错位**问题：

| 问题 | 现状 | 影响 |
|------|------|------|
| 导出操作入口错位 | 用户在 FormulaList/MaterialList 选好数据后，需跳转到导出中心才能执行导出 | 操作路径断裂，跳转参数丢失 |
| 角色定位混乱 | admin 和 formulist 共用同一个导出中心，但管理需求完全不同 | formulist 不需要看到全局任务、分享管理 |
| API 接口 Tab 无用 | `api_data_interfaces` 功能从未被业务使用，增加维护负担 | 前后端代码冗余 |
| 业务页面导出空实现 | MaterialList.handleBatchExport 仅弹提示，无实际导出 | 功能缺失 |
| 配方导出参数丢失 | FormulaList.handleBatchExport 通过 query 传 formulaIds，但导出中心未读取 | Bug |
| createJob 仅取首项 | `formulaIds?.[0]` 只导出第一个配方，多选无效 | Bug |
| 导出中心独立路由割裂 | 导出中心作为独立页面 `/exports`，与系统管理脱节 | 管理功能分散，admin 需记忆多个入口 |

**双轨制架构**的核心思想：

- **轨道一**：业务页面（FormulaList / MaterialList）成为导出操作的**主入口**，用户在数据上下文中直接完成导出
- **轨道二**：导出中心（ExportCenter）重新定位为 **admin 管理模块**，聚焦全局任务监控、模板管理、分享管理、导出配置

**v2.1 架构优化**：导出中心从独立路由页面移入系统管理模块（SystemConfig），作为其子 Tab，形成管理功能聚合。导出配置从 SystemConfig 移回 ExportCenter，形成逻辑闭环——admin 在导出中心内既能管理又能配置。

### 1.2 目标：双轨制架构目标表

| 目标 | 说明 | 优先级 |
|------|------|--------|
| 业务页面就地导出 | FormulaList / MaterialList 内置导出弹窗，无需跳转 | P0 |
| 修复 P0 Bug | handleBatchExport 参数丢失、空实现、createJob 仅取首项 | P0 |
| 导出中心管理化 | ExportCenter 仅 admin 可访问，移除创建表单和 API 接口 Tab | P1 |
| 导出中心移入系统管理 | ExportCenter 作为 SystemConfig 子 Tab，统一管理入口 | P1 |
| 导出配置闭环 | 导出配置从 SystemConfig 移入 ExportCenter，管理+配置一体 | P1 |
| 移除 API 接口模块 | 前端移除 Tab，后端移除路由，数据库表标记废弃 | P1 |
| 仪表盘改造 | 移除 apiCount 统计卡片，新增数据来源维度 | P1 |
| 统一导出 API | 业务页面和导出中心共用 `POST /api/exports/jobs` | P0 |
| 周报/月报导出保留 | 周报/月报导出保留在导出中心（admin 管理视图内） | P2 |

### 2.5 业务页面导出流程断裂分析

#### FormulaList.vue：handleBatchExport 跳转参数丢失

```typescript
// 当前实现（FormulaList.vue L748）
const handleBatchExport = () => {
  if (selectedRows.value.length === 0) return;
  const ids = selectedRows.value.map(f => f.id).join(',');
  router.push({ path: '/exports', query: { formulaIds: ids } });
  MessagePlugin.success(`已选择 ${selectedRows.value.length} 个配方进行导出`);
  clearSelection();
};
```

**问题**：
1. 跳转到 ExportCenter 后，ExportCenter 未读取 `route.query.formulaIds`
2. 用户选中的配方上下文在跳转后丢失
3. 需要在导出中心重新选择配方，体验割裂

#### MaterialList.vue：handleBatchExport 空实现

```typescript
// 当前实现（MaterialList.vue L777）
const handleBatchExport = () => {
  if (selectedRows.value.length === 0) return;

```

**问题**：
1. 仅弹出成功提示，无任何实际导出逻辑
2. 用户点击后以为导出成功，但实际没有文件生成

```typescript
export async function getStatistics(userId: string, role: string): Promise<ExportStatistics> {
  const ownerFilter = role === "admin" ? "" : "WHERE created_by = ?";
  const ownerParams = role === "admin" ? [] : [userId];

  const [jobStats] = query(
    `SELECT
      COUNT(*) as total_jobs,
      SUM(CASE WHEN status = 'completed' THEN 1 ELSE 0 END) as completed_jobs,
      SUM(CASE WHEN status = 'failed' THEN 1 ELSE 0 END) as failed_jobs,
      SUM(CASE WHEN status IN ('pending', 'processing') THEN 1 ELSE 0 END) as processing_jobs
     FROM export_jobs ${ownerFilter}`,
    ownerParams,
  ) as [Record<string, unknown>[]];

  const shareFilter = role === "admin" ? "" : "WHERE created_by = ?";
  const shareParams = role === "admin" ? [] : [userId];

  const [shareStats] = query(
    `SELECT COUNT(*) as active_shares FROM share_configs ${shareFilter}`,
    shareParams,
  ) as [Record<string, unknown>[]];

  const [templateStats] = query(
    "SELECT COUNT(*) as template_count FROM export_templates",
  ) as [Record<string, unknown>[]];

  const [recentJobs] = query(
    `SELECT job_id, data_category, export_type, status, created_at, error_message
     FROM export_jobs ${ownerFilter}
     ORDER BY created_at DESC LIMIT 5`,
    ownerParams,
  ) as [Record<string, unknown>[]];

  const [recentShares] = query(
    `SELECT share_id, share_type, created_at FROM share_configs ${shareFilter} ORDER BY created_at DESC LIMIT 5`,
    shareParams,
  ) as [Record<string, unknown>[]];

  const recentActivities: Array<{ type: string; title: string; desc: string; time: string }> = [];

  for (const job of recentJobs) {
    const category = (job.data_category as string) || "formula";
    const categoryLabel = category === "formula"
      ? "配方"
      : category === "material"
        ? "原料"
        : category === "weekly-report"
          ? "周报"
          : "月报";
    const statusLabel = job.status === "completed" ? "完成" : job.status === "failed" ? "失败" : "处理中";
    recentActivities.push({
      type: "export",
      title: `${categoryLabel}导出`,
      desc: `${(job.export_type as string)?.toUpperCase() || "EXCEL"} 导出${statusLabel}`,
      time: job.created_at as string,
    });
  }

  for (const share of recentShares) {
    recentActivities.push({
      type: "share",
      title: "分享链接",
      desc: `创建${share.share_type === "link" ? "公开" : share.share_type === "email" ? "邮件" : "API"}分享`,
      time: share.created_at as string,
    });
  }

  recentActivities.sort((a, b) => new Date(b.time).getTime() - new Date(a.time).getTime());

  return {
    totalJobs: Number(jobStats[0]?.total_jobs ?? 0),
    completedJobs: Number(jobStats[0]?.completed_jobs ?? 0),
    failedJobs: Number(jobStats[0]?.failed_jobs ?? 0),
    processingJobs: Number(jobStats[0]?.processing_jobs ?? 0),
    activeShares: Number(shareStats[0]?.active_shares ?? 0),
    templateCount: Number(templateStats[0]?.template_count ?? 0),
    recentActivities: recentActivities.slice(0, 10),
  };
}
```

```typescript
import { useExportStore } from '@/stores/export'

const exportStore = useExportStore()

const showExportDrawer = ref(false)
const exporting = ref(false)
const showFormulaPicker = ref(false)

const exportForm = reactive({
  exportType: 'excel' as 'excel' | 'pdf',
  templateId: '',
  includeVersionInfo: false,
})

const exportTargetFormulas = ref<Formula[]>([])

const formulaTemplates = computed(() =>
  exportStore.templates.filter(t => t.category === 'formula')
)

function openExportDrawer(formulas: Formula[]) {
  exportTargetFormulas.value = [...formulas]
  exportForm.exportType = 'excel'
  exportForm.templateId = ''
  exportForm.includeVersionInfo = false
  showExportDrawer.value = true
  exportStore.fetchTemplates({ category: 'formula', page: 1, pageSize: 100 })
}

function removeFromExportSelection(id: string) {
  exportTargetFormulas.value = exportTargetFormulas.value.filter(f => f.id !== id)
}

async function handleExport() {
  if (exportTargetFormulas.value.length === 0) {
    MessagePlugin.warning('请至少选择一个配方')
    return
  }
  exporting.value = true
  const result = await exportStore.createJob({
    dataCategory: 'formula',
    exportType: exportForm.exportType,
    formulaIds: exportTargetFormulas.value.map(f => f.id),
    templateId: exportForm.templateId || undefined,
    includeVersionInfo: exportForm.includeVersionInfo,
  })
  exporting.value = false
  if (result.success) {
    const resultData = result.data as { jobId: string; status: string; fileName?: string; errorMessage?: string } | undefined
    if (resultData?.status === 'completed') {
      MessagePlugin.success('导出成功，正在下载...')
      const ext = exportForm.exportType === 'pdf' ? 'pdf' : 'xlsx'
      await exportStore.downloadFile(
        resultData.jobId,
        resultData.fileName || `配方导出.${ext}`,
        exportForm.exportType,
      )
    } else if (resultData?.status === 'failed') {
      MessagePlugin.error(`导出失败: ${resultData.errorMessage || '未知错误'}`)
    }
    showExportDrawer.value = false
    clearSelection()
  } else {
    MessagePlugin.error(result.message || '导出失败')
  }
}
```

```typescript
const exportConfig = reactive({
  defaultExportType: 'excel',
  formulaDefaultTemplate: '',
  materialDefaultTemplate: '',
  weeklyReportDefaultTemplate: '',
  monthlyReportDefaultTemplate: '',
  exportRateLimit: 100,
  fileNamingPattern: '{type}_{date}_{id}',
  fileRetentionDays: 30,
  maxExportRecords: 500,
  autoCleanupEnabled: false,
})
const savingConfig = ref(false)

const formulaTemplates = computed(() =>
  exportStore.templates.filter(t => t.category === 'formula')
)
const materialTemplates = computed(() =>
  exportStore.templates.filter(t => t.category === 'material')
)
const weeklyReportTemplates = computed(() =>
  exportStore.templates.filter(t => t.category === 'weekly-report')
)
const monthlyReportTemplates = computed(() =>
  exportStore.templates.filter(t => t.category === 'monthly-report')
)

async function loadExportConfig() {
  const result = await exportStore.fetchConfig()
  if (result.success && result.data) {
    const configs = result.data as Record<string, string>
    if (configs.default_export_type) exportConfig.defaultExportType = configs.default_export_type
    if (configs.formula_default_template) exportConfig.formulaDefaultTemplate = configs.formula_default_template
    if (configs.material_default_template) exportConfig.materialDefaultTemplate = configs.material_default_template
    if (configs.weekly_report_default_template) exportConfig.weeklyReportDefaultTemplate = configs.weekly_report_default_template
    if (configs.monthly_report_default_template) exportConfig.monthlyReportDefaultTemplate = configs.monthly_report_default_template
    if (configs.export_rate_limit) exportConfig.exportRateLimit = Number(configs.export_rate_limit)
    if (configs.file_naming_pattern) exportConfig.fileNamingPattern = configs.file_naming_pattern
    if (configs.file_retention_days) exportConfig.fileRetentionDays = Number(configs.file_retention_days)
    if (configs.max_export_records) exportConfig.maxExportRecords = Number(configs.max_export_records)
    if (configs.auto_cleanup_enabled) exportConfig.autoCleanupEnabled = configs.auto_cleanup_enabled === 'true'
  }
}

async function handleSaveConfig() {
  savingConfig.value = true
  const configs = [
    { configKey: 'default_export_type', configValue: exportConfig.defaultExportType, configType: 'string' },
    { configKey: 'formula_default_template', configValue: exportConfig.formulaDefaultTemplate, configType: 'string' },
    { configKey: 'material_default_template', configValue: exportConfig.materialDefaultTemplate, configType: 'string' },
    { configKey: 'weekly_report_default_template', configValue: exportConfig.weeklyReportDefaultTemplate, configType: 'string' },
    { configKey: 'monthly_report_default_template', configValue: exportConfig.monthlyReportDefaultTemplate, configType: 'string' },
    { configKey: 'export_rate_limit', configValue: String(exportConfig.exportRateLimit), configType: 'number' },
    { configKey: 'file_naming_pattern', configValue: exportConfig.fileNamingPattern, configType: 'string' },
    { configKey: 'file_retention_days', configValue: String(exportConfig.fileRetentionDays), configType: 'number' },
    { configKey: 'max_export_records', configValue: String(exportConfig.maxExportRecords), configType: 'number' },
    { configKey: 'auto_cleanup_enabled', configValue: String(exportConfig.autoCleanupEnabled), configType: 'boolean' },
  ]
  const result = await exportStore.updateConfig(configs)
  savingConfig.value = false
  if (result.success) {
    MessagePlugin.success('导出配置已保存')
  }
}

function handleResetConfig() {
  exportConfig.defaultExportType = 'excel'
  exportConfig.formulaDefaultTemplate = ''
  exportConfig.materialDefaultTemplate = ''
  exportConfig.weeklyReportDefaultTemplate = ''
  exportConfig.monthlyReportDefaultTemplate = ''
  exportConfig.exportRateLimit = 100
  exportConfig.fileNamingPattern = '{type}_{date}_{id}'
  exportConfig.fileRetentionDays = 30
  exportConfig.maxExportRecords = 500
  exportConfig.autoCleanupEnabled = false
}
```

## 8. 关键技术决策

### 8.1 双轨制架构决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 导出入口位置 | 业务页面就地导出 | 用户在数据上下文中完成导出，无需跳转 |
| 导出中心定位 | admin 管理模块 | admin 需要全局视角，formulist 不需要 |
| 导出中心归属 | SystemConfig 子 Tab（v2.1） | 管理功能聚合，统一入口，减少路由碎片化 |
| 导出配置归属 | ExportCenter 内部 Tab（v2.1） | 管理与配置逻辑闭环，admin 在导出中心内既能管理又能配置 |
| 周报/月报导出位置 | 导出中心内 | 周报/月报是管理视角数据，非业务操作 |
| 共用 API | `POST /api/exports/jobs` | 统一入口，避免重复代码 |

### 8.2 导出维度字段设计

| 字段 | 值域 | 说明 |
|------|------|------|
| `data_category` | `formula` / `material` / `weekly-report` / `monthly-report` | 导出数据类别 |
| `target_ids_json` | JSON 数组 | 目标 ID 列表（配方/原料） |
| `export_type` | `excel` / `pdf` | 导出格式 |

### 8.3 配置存储方案

使用 `export_center_config` 表存储键值对配置：

- 优势：无需 ALTER TABLE 即可新增配置项
- 类型校验：`config_type` 字段标记类型（number / boolean / string），更新时校验
- 审计：`updated_by` + `updated_at` 记录变更

### 8.4 权限控制方案

| 资源 | 权限 | 实现 |
|------|------|------|
| 业务页面导出 | 所有认证用户 | `authMiddleware` |
| 导出中心 | admin only | SystemConfig `requiresAdmin: true`（v2.1 继承） |
| 导出配置 | admin only | ExportCenter 继承 SystemConfig 权限 + Controller 层 `role !== 'admin'` 检查 |
| 系统配置 | admin only | Controller 层 `role !== 'admin'` 检查 |
| 公开分享 | 无需认证 | `/public/share/:shareId` 路由在 `authMiddleware` 之前 |

### 8.5 文件命名模式

| 模式 | 示例 | 说明 |
|------|------|------|
| `{type}_{date}_{id}` | `formula_20260528_abc123.xlsx` | 默认模式 |
| `{type}_{names}_{date}` | `formula_金银花菊花_20260528.pdf` | 含名称模式 |

文件命名通过 `export_center_config` 中的 `file_naming_pattern` 配置控制。

### 8.6 导出中心移入系统管理模块决策

| 决策 | 选择 | 理由 |
|------|------|------|
| 归属方式 | SystemConfig 子 Tab | 管理功能聚合，减少独立路由 |
| 权限方式 | 继承 SystemConfig | 无需独立路由守卫，简化权限逻辑 |
| 组件位置 | `views/system/ExportCenter.vue` | 与 SystemConfig 同目录，语义清晰 |
| 导出配置归属 | ExportCenter 第5个 Tab | 管理与配置逻辑闭环，避免跨组件配置 |
