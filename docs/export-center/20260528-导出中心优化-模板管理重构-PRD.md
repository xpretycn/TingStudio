## 1. 执行摘要

### 1.1 问题陈述

当前导出中心模板管理 Tab 存在以下问题：

**交互层面**：
- 扁平表格列出所有模板，无法区分业务维度
- 创建/编辑对话框仅有 name/type/category/description/isDefault 五个字段，无法配置导出内容
- 无字段选择能力，用户不知道会导出哪些数据
- 无模板预览，配置后无法验证效果

**数据层面**：
- `format_config_json` 字段形同虚设，仅存储 `{ columns, orientation, fontSize }` 三项
- 所有分类（配方/原料/周报/月报）使用相同的配置结构，实际各分类字段完全不同
- 种子数据中有 2 条已废弃模板（MES对接API模板、质检报告打印模板）

### 1.2 解决方案

参考系统管理 → 模型管理 → 模型应用 Tab 的卡片网格交互模式，重构模板管理为四个业务模块卡片 + 右侧 Drawer 配置 + 右侧预览面板的形式。

***

## 2. 用户故事

### 模块：模板管理重构

* **US-T1**: 作为管理员，我希望在模板管理 Tab 看到按业务分类（配方/原料/周报/月报）的模块卡片，以便快速定位各业务的模板配置。

* **US-T2**: 作为管理员，我希望能为每个业务模块选择要导出的数据字段（必选/可选），以便定制导出内容。

* **US-T3**: 作为管理员，我希望能配置导出格式（PDF/Excel）、设置默认模板、调整排版参数，以便标准化导出格式。

* **US-T4**: 作为管理员，我希望在配置模板后能实时预览导出效果（使用真实数据填充），以便确认配置正确。

## 4. 数据模型变更

### 4.1 format_config_json 结构升级

当前结构：
```json
{
  "columns": ["配方名称", "业务员名称", "原料列表", "创建时间"],
  "orientation": "portrait",
  "fontSize": 12
}
```

升级为：
```json
{
  "selectedFields": ["name", "code", "createdAt", "salesmanName", "..."],
  "requiredFields": ["name", "code", "createdAt"],
  "exportFormat": "pdf",
  "orientation": "portrait",
  "pageSize": "A4",
  "fontSize": 12,
  "includeHeader": true,
  "includeFooter": true
}
```

### 4.2 默认模板标识

每个 category + type 组合下只能有一个 `is_default = 1` 的模板。

### 4.3 字段配置表（前端常量）

```typescript
interface FieldGroup {
  groupName: string
  required: boolean
  fields: FieldConfig[]
}

interface FieldConfig {
  key: string
  label: string
  defaultSelected: boolean
}
```

四个 category 各有独立的字段分组配置。

### 4.4 种子数据清理

| 操作 | 模板名称 | 理由 |
|------|---------|------|
| 删除 | MES对接API模板 | type=api 已废弃 |
| 删除 | 质检报告打印模板 | type=print 无业务场景 |
| 保留改造 | 标准配方PDF模板 → formula | 升级 formatConfig 为字段配置 |
| 保留改造 | 生产配方Excel模板 → formula | 升级 formatConfig |
| 保留改造 | 营养标签PDF模板 → formula | 升级 formatConfig |
| 保留改造 | 原料清单Excel模板 → material | 升级 formatConfig |
| 新建 | 周报导出Excel模板 → weekly-report | 含全部周报字段 |
| 新建 | 月报导出Excel模板 → monthly-report | 含全部月报字段 |

## 5. 交互规范

参考 [ModelManagement.vue](file:///d:/ProgramData/workspace-codeby/ting-studio/frontend/src/views/system/ModelManagement.vue#L149-L253) 模型应用 Tab 的交互模式：

| 设计元素 | 参考来源 |
|---------|---------|
| 模块卡片网格布局 | 模型应用 Tab `.applications-grid` |
| 卡片 header / body / footer 结构 | `.application-card-header` / `.application-card-body` / `.application-card-footer` |
| 图标 + 名称 + 状态徽章 | `.app-module-icon` + `.app-module-name` + `.app-status-badge` |
| 字段列表展示 | `.app-field` / `.field-label` / `.field-value` |
| Drawer 表单布局 | Prompt 编辑 dialog 表单结构 |

### 5.2 与现有模板 CRUD 的关系

- 模板列表（表格视图）→ 改为模块卡片视图
- 模板创建/编辑 Dialog → 改为右侧 Drawer（含字段配置 + 格式设置）
- 模板删除 → 保留 popconfirm，在 Drawer 模板选择器中提供删除
