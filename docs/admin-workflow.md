## 1. 角色概述

| 维度 | 管理员 (Admin) | 配方师 (Formulist) |
|------|---------------|-------------------|
| 数据范围 | 全部数据 | 仅自己创建的数据 |
| 审批权限 | 可审批/驳回 | 只能提交审批 |
| 系统配置 | 完整访问 | 不可见 |
| 模型管理 | 完整访问 | 不可见 |
| AI 助手 | 完整访问 | 不可见 |

### 路由权限

以下路由仅管理员可访问（`requiresAdmin: true`），非管理员访问会被重定向到 `/`：

| 路由 | 页面 |
|------|------|
| `/system/config` | 系统管理 |
| `/model-management` | 模型管理 |
| `/tools/ai-assistant` | AI 助手工作台 |

## 2. 工作台总览

### 2.1 审批中心面板

管理员在工作台看到 **AdminReviewPanel**（审批中心），包含三个标签页：

| 标签页 | 内容 | 操作 |
|--------|------|------|
| 配方待审 | 配方师提交的待审批配方列表 | 点击"审核"跳转版本管理 |
| 原料待审 | 待审批的原料版本列表 | 点击"审核"跳转原料版本历史 |
| 已审核 | 历史审批记录（批准/驳回） | 查看状态标签 |

**高级功能：**
- 可折叠筛选面板：时间范围（今天/3天/7天/30天）、提交人筛选、审批结果筛选
- 三态排序：点击切换 升序 → 降序 → 清除
- 自动刷新：每 30 秒轮询更新
- 分页浏览

### 2.2 统计卡片

| 卡片 | 数据范围 |
|------|---------|
| 配方总数 | 全部配方 |
| 原料总数 | 全部原料 |
| 本月营收 | 全部销量 |
| 销量配方 | 配方数量 |

### 2.3 销量趋势图

- ECharts 折线图，支持周/月/年切换
- 渐变填充区域，悬浮提示显示时间段和销量

### 2.4 近期动态时间线

- 按时间倒序显示配方/原料变更记录
- 分页浏览，点击可跳转详情

### 2.5 精选配方

- 显示 Top 3 配方
- 管理员看到全部配方，配方师仅看到自己的

### 2.6 快捷操作

管理员的快捷操作（6 个）：
- 快速录入 → `/formulas/quick`
- 创建配方 → `/formulas/new`
- 录入原料 → `/materials/new`
- 系统管理 → `/system`
- 模型管理 → `/model-management`
- 用户管理 → `/users`

## 4. 配方审批流程

**入口**：版本管理 `/versions/formula/:id`

### 4.1 直接发布（草稿版本）

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 4.1.1 | 选择草稿版本 | 显示"发布"按钮 |
| 4.1.2 | 点击"发布" | 弹出确认框："确定要直接发布该版本吗？发布后将替换当前配方。" |
| 4.1.3 | 确认发布 | 版本状态 `draft` → `published`，配方数据同步 |

### 4.2 批准审批（待审批版本）

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 4.2.1 | 选择待审批版本 | 显示"批准"和"驳回"按钮 |
| 4.2.2 | 点击"批准" | 弹出确认框："确定要批准该版本吗？批准后将替换当前配方。" |
| 4.2.3 | 确认批准 | 版本状态 `pending_review` → `published`，配方数据同步 |

### 4.3 驳回审批

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 4.3.1 | 选择待审批版本 | 显示"驳回"按钮 |
| 4.3.2 | 点击"驳回" | 弹出驳回对话框 |
| 4.3.3 | 填写驳回意见 | 必填，说明驳回原因 |
| 4.3.4 | 确认驳回 | 版本状态 `pending_review` → `draft`，配方师可重新编辑 |

### 4.4 审批后验证点

- 批准后：配方详情页显示最新已发布版本
- 驳回后：配方师在"我的审批"中看到驳回状态和意见
- 版本时间线：正确记录审批操作和时间

## 7. 含量比阈值配置

**入口**：系统管理 → 含量比配置

| 步骤 | 操作 | 预期结果 |
|------|------|---------|
| 7.1 | 查看当前阈值 | 显示 normal/warning/highWarning 范围 |
| 7.2 | 编辑阈值 | 修改各级别边界值 |
| 7.3 | 保存配置 | 阈值立即生效，影响所有配方的含量比校验 |

### 阈值配置范围

| 参数 | 最小值 | 最大值 | 说明 |
|------|--------|--------|------|
| normalLow | 0.5 | 1.0 | 正常范围下限 |
| normalHigh | 1.0 | 1.5 | 正常范围上限 |
| warningLow | 0.3 | 1.0 | 预警范围下限 |
| warningHigh | 1.0 | 1.7 | 预警范围上限 |
| highWarningLow | 0.1 | 1.0 | 警告范围下限 |
| highWarningHigh | 1.0 | 2.0 | 警告范围上限 |

### 关键验证点

- 阈值修改后，前端含量比校验使用新阈值
- 阈值必须满足：highWarningLow < warningLow < normalLow < normalHigh < warningHigh < highWarningHigh
- 保存后刷新页面验证配置持久化

| 步骤 | 操作 | 预期结果 |

| 13.1.4 | 快捷操作 | "全选读取" / "全选写入" / "清除全部" |
| 13.1.5 | 编辑角色 | 修改角色信息和权限 |

**权限粒度：** 每个模块有读取和写入两项权限，共 9 模块 × 2 = 18 项权限。
角色卡片显示权限数（如 "12/18 项权限"）和用户数。

| 步骤 | 操作 | 预期结果 |

- 新用户首次登录需强制修改密码
