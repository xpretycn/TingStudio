# 产品需求文档（PRD）：数据库可视化管理模块

## 1. 执行摘要

### 1.1 问题陈述

TingStudio 平台当前拥有 **30+ 张数据表**、**60+ 个后端脚本**，但所有数据库操作均需通过命令行手动执行。管理员无法通过 Web 界面直观地查看表结构、浏览数据、管理备份或执行脚本，运维效率低下且存在误操作风险。

### 1.2 解决方案

在 **Toolbox.vue（工具箱页面）** 中新增独立的 **「数据库管理」** 区域，提供四大核心能力：

* **数据库概览**：全局查看表结构、行数、索引等元信息

* **表数据浏览**：可视化浏览、搜索、分页查看任意表数据

* **备份与恢复**：一键导出/恢复数据库，管理备份历史

* **脚本执行**：可视化执行现有维护脚本并查看执行日志

### 1.3 成功标准

| 指标                  | 目标值     |
| ------------------- | ------- |
| 管理员可通过 UI 完成核心数据库操作 | 100% 覆盖 |
| 危险操作（恢复/清空）有二次确认保护  | 100% 覆盖 |
| 表数据查询响应时间（<1000 行）  | ≤ 500ms |
| 非管理员角色无法访问该功能       | 100% 阻断 |

## 2. 用户体验与功能详情

### 2.1 用户画像

| 角色                 | 权限范围               | 使用场景                |
| ------------------ | ------------------ | ------------------- |
| **admin（管理员）**     | 全部功能（读写+执行脚本+备份恢复） | 日常运维、数据检查、故障排查、备份管理 |

### 2.3 功能一：数据库概览（Tab: 概览）

#### 2.3.1 页面交互流程

```
用户点击「概览」Tab
    ↓
页面加载 → GET /api/db/info（获取数据库基本信息）
    ↓
展示信息卡片区 + 表列表区
```

#### 2.3.2 信息卡片区（顶部 4 列网格）

| 卡片           | 显示内容           | 数据来源                               |
| ------------ | -------------- | ---------------------------------- |
| 📦 **数据库类型** | SQLite / MySQL | `GET /api/db/info` → `dbType`      |
| 📊 **总表数**   | 数字（如 32）       | `GET /api/db/info` → `tableCount`  |
| 💾 **文件大小**  | 如 12.5 MB      | `GET /api/db/info` → `dbSize`      |
| 🕐 **最后更新**  | 时间格式化          | `GET /api/db/info` → `lastUpdated` |

#### 2.3.3 表列表区（TDesign Table）

| 列名   | 字段            | 说明                          |
| ---- | ------------- | --------------------------- |
| 表名   | `name`        | 可点击跳转到「数据浏览」Tab 并自动选中该表     |
| 行数   | `rowCount`    | 数字格式化（如 1,234）              |
| 字段数  | `columnCount` | 数字                          |
| 索引数  | `indexCount`  | 数字                          |
| 创建时间 | `createdAt`   | 使用 `timeFormat.ts` 格式化      |
| 操作   | -             | 「查看结构」按钮 → 弹出 Drawer 展示字段详情 |

#### 2.3.4 查看结构交互

点击「查看结构」→ 右侧滑出 **t-drawer**：

```
┌─ 表结构：materials ──────────────────┐
│                                       │
│  字段名         │ 类型      │ 是否必填 │
│  ─────────────────────────────────── │
│  id            │ TEXT      │ ✓ 主键   │
│  name          │ TEXT      │ ✓       │
│  material_code │ TEXT      │ ✗       │
│  created_at    │ DATETIME  │ ✓       │
│  ...           │ ...      │ ...     │
│                                       │
│  索引：                                │
│  · idx_materials_name (name)          │
│  · idx_materials_code (material_code) │
│                                       │
└───────────────────────────────────────┘
```

### 2.4 功能二：表数据浏览（Tab: 数据浏览）

#### 2.4.1 页面交互流程

```
用户点击「数据浏览」Tab
    ↓
加载表列表下拉框 → GET /api/db/tables
    ↓
用户选择一张表（或从概览页跳转带参数）
    ↓
加载表数据 → GET /api/db/tables/:tableName?page=1&pageSize=20&search=&sort=
    ↓
展示数据表格 + 工具栏
```

#### 2.4.2 工具栏布局

```
┌────────────────────────────────────────────────────┐
│ [▼ 选择表 ▼]  [🔍 搜索...]  [排序: ▼]  [每页: 20▼] │
└────────────────────────────────────────────────────┘
```

**控件说明**：

| 控件   | 类型         | 说明                                |
| ---- | ---------- | --------------------------------- |
| 选择表  | `t-select` | 下拉选择，选项来自 `/api/db/tables`，支持搜索过滤 |
| 搜索   | `t-input`  | 300ms 防抖，模糊匹配所有文本字段               |
| 排序   | `t-select` | 下拉选择排序列 + ASC/DESC                |
| 每页条数 | `t-select` | 10 / 20 / 50 / 100                |

#### 2.4.3 数据表格（TDesign Table）

* **列自动生成**：根据表结构动态渲染列

* **长文本截断**：超过 50 字符的文本显示省略号，hover 显示完整内容（tooltip）

* **空值显示**：`null` / 空字符串显示为 `--`

* **时间格式化**：检测 `datetime`/`timestamp` 类型字段，使用 `timeFormat.ts`

* **高亮搜索词**：搜索时匹配文字高亮显示

* **分页**：底部 TDesign 分页器，支持跳页

#### 2.4.4 特殊处理规则

| 场景                         | 处理方式                                |
| -------------------------- | ----------------------------------- |
| JSON 字段（如 `per_100g_json`） | 显示 `[JSON]` 标签，点击展开 JSON 格式化查看器（弹窗） |
| 大表（>10000 行）               | 强制分页，最大每页 100 条，提示"数据量较大，已启用分页模式"   |
| 无数据表                       | 显示 TDesign EmptyState 空状态组件         |
| 加载状态                       | 表格骨架屏（Skeleton）                     |

### 2.5 功能三：备份与恢复（Tab: 备份）

#### 2.5.1 页面交互流程

```
用户点击「备份」Tab
    ↓
加载备份列表 → GET /api/db/backups
    ↓
展示备份列表 + 操作按钮
```

#### 2.5.2 操作按钮区

```
┌──────────────────────────────────────────────┐
│  [📥 创建新备份]  [📤 上传备份恢复]           │
└──────────────────────────────────────────────┘
```

**创建新备份交互**：

1. 点击「创建新备份」→ 弹出确认对话框（t-dialog）
2. 确认后调用 `POST /api/db/backups`
3. 显示进度条（loading 状态）
4. 成功后刷新备份列表，顶部出现成功提示

**上传恢复交互**：

1. 点击「上传备份恢复」→ 弹出文件选择对话框（t-upload）
2. 选择 `.json` 备份文件
3. **二次确认弹窗**（t-popconfirm）：

   ```
   ⚠️ 危险操作确认

   即将从备份文件恢复数据库，当前数据将被覆盖！

   备份文件：tingstudio_backup_20260529.json
   文件大小：8.5 MB

   此操作不可撤销，请确认是否继续？
   [取消] [确认恢复]
   ```
4. 确认后调用 `POST /api/db/backups/restore`
5. 显示进度条，完成后提示重启服务

#### 2.5.3 备份列表（TDesign Table）

| 列名   | 字段           | 说明                                         |
| ---- | ------------ | ------------------------------------------ |
| 文件名  | `fileName`   | 如 `tingstudio_backup_20260529_143022.json` |
| 大小   | `fileSize`   | 格式化为 KB/MB                                 |
| 创建时间 | `createdAt`  | `timeFormat.ts` 格式化                        |
| 表数量  | `tableCount` | 备份中包含的表数                                   |
| 校验码  | `hash`       | SHA256 前 16 位...                           |
| 操作   | -            | 「下载」「恢复」（均需二次确认）                           |

**操作交互**：

| 操作 | 交互方式              | API                                      |
| -- | ----------------- | ---------------------------------------- |
| 下载 | 直接浏览器下载           | `GET /api/db/backups/:fileName/download` |
| 恢复 | t-popconfirm 二次确认 | `POST /api/db/backups/:fileName/restore` |
| 删除 | t-popconfirm 二次确认 | `DELETE /api/db/backups/:fileName`       |

### 2.6 功能四：脚本执行（Tab: 脚本）

#### 2.6.1 页面交互流程

```
用户点击「脚本」Tab
    ↓
加载可用脚本列表 → GET /api/db/scripts
    ↓
展示脚本分类列表
    ↓
用户选择脚本 → 查看详情 → 执行
```

#### 2.6.2 脚本分类与列表

脚本按类别分组展示（TDesign Collapse 折叠面板）：

| 分类          | 包含脚本                                           | 危险等级 |
| ----------- | ---------------------------------------------- | ---- |
| 🔧 **数据修复** | fixMaterials, fixMaterialCodes, fixCreatedBy 等 | ⚠️ 中 |
| 🌱 **种子数据** | seedData, seedEnhancedData, seedEnumOptions 等  | ℹ️ 低 |
| 📥 **数据导入** | importMaterialsFromExcel, importRealData 等     | ⚠️ 中 |
| 🧹 **数据清理** | cleanInvalidMaterials, cleanOldMaterials 等     | 🔴 高 |
| ✅ **数据验证**  | verifyData, checkUsers, check-reports 等        | ℹ️ 低 |
| 🔄 **迁移**   | migrate-ratio-factor, syncMaterialTypes 等      | ⚠️ 中 |

每个脚本项展示：

```
┌─ fixMaterials — 修复原料数据 ─────────────────┐
│  描述：修复原料数据中的异常值和缺失字段          │
│  危险等级：⚠️ 中（可能修改数据）                 │
│  预计耗时：< 5 秒                               │
│                                                 │
│  [▶ 执行]  [📋 查看详情]                        │
└─────────────────────────────────────────────────┘
```

#### 2.6.3 执行脚本交互

1. 点击「执行」→ 根据**危险等级**决定确认方式：

| 危险等级 | 确认方式                | 示例         |
| ---- | ------------------- | ---------- |
| ℹ️ 低 | 直接执行                | 种子数据、验证脚本  |
| ⚠️ 中 | t-dialog 确认         | 数据修复、导入、迁移 |
| 🔴 高 | t-popconfirm + 输入确认 | 数据清理类脚本    |

1. **中/高危确认文案示例**：

   ```
   ⚠️ 脚本执行确认

   即将执行脚本：cleanInvalidMaterials

   该脚本将清理无效的原料数据记录。
   请确认已做好数据备份！

   [取消] [确认执行]
   ```

2. **执行中状态**：

   * 脚本卡片显示 loading 动画 + 进度文字

   * 底部实时日志区域滚动输出（模拟终端效果）

   * 日志颜色：成功绿色、错误红色、信息灰色

3. **执行完成**：

   * 显示执行结果统计：✅ 成功 / ❌ 失败 / ⏱️ 耗时

   * 错误时展开显示详细错误堆栈

   * 提供「复制日志」按钮

## 4. 技术规格

### 4.1 架构概述

```
┌─────────────────────────────────────────┐
│         Toolbox.vue（前端）               │
│  ┌─────────┬─────────┬────────┬──────┐  │
│  │ 概览Tab  │ 浏览Tab  │ 备份Tab│脚本Tab│  │
│  └────┬────┴────┬────┴───┬───┴──┬───┘  │
│       │         │        │       │      │
│  api/db.ts（前端 API 层）                │
│       │         │        │       │      │
├───────┼─────────┼────────┼───────┼──────┤
│  /api/db/*  （后端路由）                  │
│       │         │        │       │      │
│  dbController.ts → dbService.ts          │
│       │         │        │       │      │
│  database-adapter.ts (query/execute)     │
└─────────────────────────────────────────┘
```

### 4.2 技术栈

| 层级    | 技术                         | 说明                                                                                                                                                                                                                           |
| ----- | -------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| 前端框架  | Vue 3 + `<script setup>`   | Composition API                                                                                                                                                                                                              |
| UI 组件 | TDesign Vue Next           | `t-tabs`, `t-table`, `t-drawer`, `t-dialog`, `t-popconfirm`, `t-collapse`, `t-select`, `t-input`, `t-button`, `t-empty`, `t-loading`, `t-alert`, `t-message`, `t-tooltip`, `t-tag`, `t-upload`, `t-skeleton`, `t-pagination` |
| HTTP  | axios（@/api/http.ts）       | 已有封装，拦截器自动解包                                                                                                                                                                                                                 |
| 后端框架  | Express + TypeScript (ESM) | REST API                                                                                                                                                                                                                     |
| 数据库适配 | database-adapter.ts        | 统一 query() 接口                                                                                                                                                                                                                |
| 认证    | JWT + authMiddleware       | admin 角色校验                                                                                                                                                                                                                   |

### 4.3 安全要求

| 要求     | 实现                                   |
| ------ | ------------------------------------ |
| 认证     | 所有 `/api/db/*` 路由使用 `authMiddleware` |
| 授权     | 通过 `req.user.role === 'admin'` 校验    |
| 危险操作   | 恢复/清空/高危脚本必须二次确认（t-popconfirm）       |
| SQL 注入 | 仅使用参数化查询，禁止拼接 SQL                    |
| 敏感信息   | 备份文件不含密码/token 等敏感配置                 |

## 6. 风险与路线图

### 6.1 分阶段交付

| 阶段           | 内容            | 优先级   |
| ------------ | ------------- | ----- |
| **MVP v1.0** | 数据库概览 + 表数据浏览 | P0 必须 |
| **v1.1**     | 备份与恢复         | P0 必须 |
| **v1.2**     | 脚本执行          | P1 重要 |

### 6.2 技术风险

| 风险         | 影响           | 缓解措施                         |
| ---------- | ------------ | ---------------------------- |
| 大表查询性能     | 浏览大表时响应慢     | 强制分页 + 最大 100 条/页 + LIMIT 优化 |
| 脚本执行阻塞     | 同步脚本可能阻塞事件循环 | 使用异步包装 + 超时机制                |
| 备份文件过大     | 内存溢出         | 流式写入 + 文件大小限制告警              |
| SQLite 并发锁 | 备份期间其他请求等待   | 备份操作使用只读连接副本                 |

### 6.3 依赖关系

```
后端路由注册 (/api/db/*)
```
