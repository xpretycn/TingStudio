# 模型管理功能 — 产品需求文档（PRD）

> 版本：v1.0 | 日期：2026-05-05 | 作者：AI 助手

---

## 1. 产品概述

### 1.1 背景

TingStudio 当前 AI 模型配置硬编码在环境变量中，无法动态管理。随着业务扩展，需要支持多模型切换、用量监控、预警通知和健康检测，以提升系统可观测性和运维效率。

### 1.2 目标

- 提供独立的模型管理页面，支持模型的增删改查和版本切换
- 实时监控各模型 Token 用量，可视化展示趋势和分布
- 支持用量预警配置，在接近阈值时自动通知管理员
- 定时检测模型健康状态，异常时自动切换到备用模型

### 1.3 用户角色

| 角色 | 权限 |
|------|------|
| 管理员（admin） | 模型增删改、预警配置、健康检测、查看全部数据 |
| 配方师（user） | 选择使用哪个模型、查看用量统计（只读） |

## 2. 功能需求

### P0 — 核心功能

#### 2.1 模型管理

- 查看所有已配置模型列表（名称、Provider、版本、API Key 状态、健康状态）
- 新增模型：填写 Provider、名称、Base URL、API Key、默认版本
- 编辑模型：修改名称、API Key、版本、描述等
- 移除模型：确认后删除
- 连通性测试：一键检测模型是否可用，返回延迟

#### 2.2 模型版本管理

- 每个模型支持下拉切换不同版本（如 Qwen Plus / Turbo / Max）
- 版本列表预置主流模型版本，切换后即时生效
- 配方师可自由选择使用哪个模型

#### 2.3 用量监控

- 顶部统计卡片：已配置模型数、今日调用次数、今日 Token、本月 Token
- Token 消耗趋势折线图（按天，支持日期范围筛选）
- 模型用量占比饼图
- 各模型用量明细表格

#### 2.4 用量预警

- 每个模型可独立配置：日调用上限、月 Token 上限、预警阈值(%)、严重阈值(%)
- 支持启用/禁用预警
- 达到阈值时生成预警记录
- 预警记录列表展示

### P1 — 选做功能（已集成）

#### 2.5 调用日志

- 按模型、调用类型、状态筛选
- 分页查看每次调用的详细信息（Token 数、延迟、状态、摘要）
- 支持查看自动切换（fallback）记录

#### 2.6 模型健康检测

- 定时检测各模型连通性（间隔以天为单位，可配置，默认 1 天）
- 健康状态标记：healthy / degraded / unhealthy / unknown
- 检测结果记录历史

#### 2.7 自动配额切换

- 每个模型可配置备用模型（fallback）
- 主模型调用失败时自动切换到备用模型
- 切换记录在调用日志中标记

## 3. 页面结构

### 3.1 入口

- 侧边栏新增「模型管理」菜单项（仅 admin 角色可见）
- 路由路径：`/model-management`

### 3.2 页面布局

```
┌─────────────────────────────────────────┐
│  预警横幅（有未读预警时显示）              │
├──────┬──────┬──────┬──────┐            │
│ 已配置 │ 今日  │ 今日  │ 本月  │            │
│ 模型数 │ 调用  │ Token │ Token │            │
├──────┴──────┴──────┴──────┘            │
│  [模型管理] [用量监控] [预警设置] [调用日志] │
├─────────────────────────────────────────┤
│  Tab 内容区域                             │
└─────────────────────────────────────────┘
```

### 3.3 Tab 说明

| Tab | 内容 |
|-----|------|
| 模型管理 | 模型卡片网格，每个卡片展示模型信息+操作按钮 |
| 用量监控 | 趋势图 + 饼图 + 明细表 |
| 预警设置 | 预警配置卡片 + 预警记录表 |
| 调用日志 | 筛选器 + 分页日志表 |

## 4. 用户流程

### 4.1 管理员新增模型

1. 点击「新增模型」按钮
2. 填写 Provider、名称、Base URL、API Key、默认版本
3. 点击确认，模型出现在列表中
4. 点击「测试」验证连通性

### 4.2 配方师切换模型版本

1. 在模型卡片中找到目标模型
2. 点击版本下拉框，选择新版本
3. 版本切换即时生效

### 4.3 管理员配置预警

1. 切换到「预警设置」Tab
2. 找到目标模型的预警配置卡片
3. 设置日调用上限、月 Token 上限、阈值
4. 开启预警开关

### 4.4 模型自动切换

1. 用户调用 AI 功能时，主模型请求失败
2. 系统自动尝试备用模型
3. 调用日志中标记 fallback 记录

## 5. 验收标准

| # | 验收项 | 标准 |
|---|--------|------|
| 1 | 模型列表 | 正确展示所有模型及实时用量统计 |
| 2 | 新增模型 | 填写完整信息后可成功添加，连通测试通过 |
| 3 | 版本切换 | 下拉选择后即时生效，后续 AI 调用使用新版本 |
| 4 | 用量图表 | 趋势图和饼图数据准确，支持日期筛选 |
| 5 | 预警触发 | 达到阈值时自动生成预警记录 |
| 6 | 健康检测 | 按配置间隔执行，状态标记正确 |
| 7 | 自动切换 | 主模型失败后自动使用备用模型 |
| 8 | 权限控制 | 非 admin 用户侧边栏无入口，页面操作只读 |
| 9 | 调用日志 | 可筛选、分页，fallback 记录标记清晰 |
