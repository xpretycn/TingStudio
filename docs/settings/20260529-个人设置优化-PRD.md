## 1. 执行摘要

### 1.1 问题陈述

当前个人设置页面（AccountSettings.vue）仅有"个人资料"和"账号安全"两个 Tab，缺少用户偏好设置功能。已有的主题切换（亮色/暗色/跟随系统 + 4 种品牌色）仅通过 Home.vue 用户下拉菜单操作，偏好数据仅存储在 localStorage，换设备或清缓存后丢失，无法跨端同步。

### 1.2 解决方案

在 AccountSettings.vue 新增**偏好设置 Tab**，包含三大设置模块：

1. **主题模式切换** — 将已有的 themeStore 功能集成到设置页面，提供可视化配置入口
2. **布局与显示偏好** — 侧边栏默认折叠、默认首页、表格每页条数
3. **业务偏好** — 配方列表默认视图、默认导出格式、默认配方排序方式

偏好数据持久化到后端数据库，登录后自动同步，换设备不丢失。

### 1.3 成功指标

| 指标        | 目标值            | 衡量方式   |
| --------- | -------------- | ------ |
| 偏好设置保存成功率 | ≥ 99%          | E2E 测试 |
| 主题切换即时生效  | ≤ 100ms        | 性能测试   |
| 跨设备偏好同步   | 登录后自动恢复        | 功能测试   |
| 偏好数据后端持久化 | 100% 偏好项存储到数据库 | 代码审查   |

## 2. 用户画像与用户故事

### 2.1 用户画像

| 角色                  | 核心诉求              |
| ------------------- | ----------------- |
| **管理员 (admin)**     | 统一管理个人偏好，跨设备同步设置  |
| **配方师 (formulist)** | 快速切换主题和布局，个性化工作界面 |

### 2.2 用户故事

#### 主题模式切换

* **US-1.1**: 作为用户，我希望在设置页面中切换主题模式（亮色/暗色/跟随系统），以便个性化界面外观。

* **US-1.2**: 作为用户，我希望在设置页面中选择品牌色（粉/黄/蓝/绿），以便定制品牌视觉风格。

* **US-1.3**: 作为用户，我希望主题切换即时生效，无需刷新页面。

#### 布局与显示偏好

* **US-2.1**: 作为用户，我希望设置侧边栏默认折叠/展开状态，以便每次登录后自动恢复我习惯的布局。

* **US-2.2**: 作为用户，我希望设置登录后的默认首页，以便快速进入常用页面。

* **US-2.3**: 作为用户，我希望设置表格默认每页条数，以便列表页按我的习惯展示数据量。

#### 业务偏好

* **US-3.1**: 作为用户，我希望设置配方列表默认视图（卡片/表格），以便打开配方页时自动使用我偏好的展示方式。

* **US-3.2**: 作为用户，我希望设置默认导出格式（Excel/PDF），以便导出操作时自动选中我常用的格式。

* **US-3.3**: 作为用户，我希望设置配方默认排序方式（按时间/按名称/按业务员），以便列表按我的习惯排序。

#### 跨设备同步

* **US-4.1**: 作为用户，我希望偏好设置保存到服务器，以便换设备登录后自动恢复我的偏好。

* **US-4.2**: 作为用户，我希望偏好设置同时保存在本地，以便离线时仍可使用上次保存的偏好。

### 3.5 偏好数据持久化

**存储策略**：双写（localStorage + 后端数据库）

| 场景     | 行为                                |
| ------ | --------------------------------- |
| 用户修改偏好 | 先写 localStorage（即时生效），再异步写后端（持久化） |
| 用户登录   | 从后端拉取偏好，覆盖 localStorage           |
| 后端写入失败 | 仅 localStorage 生效，不阻塞用户操作         |
| 离线使用   | 读取 localStorage 缓存                |

**后端 API**：

| 方法  | 路径                      | 权限             | 说明       |
| --- | ----------------------- | -------------- | -------- |
| GET | `/api/auth/preferences` | authMiddleware | 获取当前用户偏好 |
| PUT | `/api/auth/preferences` | authMiddleware | 更新当前用户偏好 |

**数据库**：

新增 `user_preferences` 表：

```sql
CREATE TABLE IF NOT EXISTS user_preferences (
  user_id TEXT PRIMARY KEY,
  preferences_json TEXT NOT NULL DEFAULT '{}',
  updated_at TEXT NOT NULL DEFAULT (datetime('now')),
  FOREIGN KEY (user_id) REFERENCES users(id)
);
```

`preferences_json` 存储所有偏好项的 JSON 对象：

```json
{
  "themeMode": "auto",
  "brandColor": "pink",
  "sidebarDefaultCollapsed": false,
  "defaultHomePage": "/formulas",
  "defaultPageSize": 8,
  "formulaDefaultView": "card",
  "defaultExportFormat": "excel",
  "formulaDefaultSort": "updatedAt"
}
```
