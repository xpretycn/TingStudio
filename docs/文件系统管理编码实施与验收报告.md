## 一、项目概述

### 1.1 模块定位

文件系统管理模块是 TingStudio 配方管理系统的核心子模块，负责实现上传文件的持久化存储、生命周期管理、数据关联、内容预览及审计追踪等完整功能链路。该模块与配方管理、原料管理、AI 助手等模块深度集成，确保文件可用于配方准确性验证、原料溯源及审计追踪。

### 1.2 设计依据

本模块严格依据 `docs/文件系统管理设计实施文档.md` 进行编码实施，所有功能点、接口规范、UI 布局、性能指标均与设计文档保持一致。

### 1.3 技术栈

| 层级 | 技术选型 |
|------|----------|
| 前端框架 | Vue 3 + Composition API + TypeScript |
| UI 组件库 | TDesign Vue Next |
| 状态管理 | Pinia |
| HTTP 客户端 | Axios |
| 后端框架 | Express.js + TypeScript |
| 数据库 | SQLite (better-sqlite3) |
| 文件上传 | Multer (磁盘存储) |
| Excel 解析 | xlsx 库 |
| 图片缩略图 | sharp (动态导入，降级兼容) |

### 2.4 API 接口规范

| 方法 | 路径 | 说明 | 权限 |
|------|------|------|------|
| GET | /api/files | 文件列表（搜索、筛选、分页） | 登录用户 |
| GET | /api/files/stats | 文件统计 | 登录用户 |
| POST | /api/files/upload | 文件上传 | 登录用户 |
| POST | /api/files/batch-delete | 批量删除 | 管理员 |
| POST | /api/files/batch-archive | 批量归档 | 登录用户 |
| GET | /api/files/:fileId | 文件详情 | 登录用户 |
| GET | /api/files/:fileId/preview | 文件预览 | 登录用户 |
| GET | /api/files/:fileId/thumbnail | 缩略图 | 登录用户 |
| GET | /api/files/:fileId/download | 文件下载 | 登录用户 |
| GET | /api/files/:fileId/audit | 审计日志 | 登录用户 |
| POST | /api/files/:fileId/link | 关联数据 | 登录用户 |
| POST | /api/files/:fileId/unlink | 取消关联 | 登录用户 |
| POST | /api/files/:fileId/reparse | 重新解析 | 登录用户 |
| DELETE | /api/files/:fileId | 删除文件 | 管理员 |

### 2.5 文件上传规范

- **存储路径**：`./data/uploads/{fileType}/{yyyy-MM}/{originalName}_{uuid8}.{ext}`
- **文件大小限制**：10MB
- **支持格式**：.xlsx, .xls, .csv, .png, .jpg, .jpeg, .gif, .webp
- **上传入口**：
  1. AI 助手页面自动上传（确认提交配方/原料表单时同步触发）
  2. 文件管理页面手动上传（工具栏"上传文件"按钮）
  3. FormulaForm.vue / MaterialForm.vue 提交时同步上传
