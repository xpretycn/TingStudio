# AI 表单辅助 Agent 需求说明

## 一、整体架构

### 1. 核心定位

Agent 不负责表单渲染、校验、接口调用，仅做：**自然语言意图理解 → 结构化数据解析 → 回填原有表单**。完全复用现有系统的表单、接口、校验、权限逻辑。

### 2. 系统分层

- **前端层**：悬浮气泡聊天组件 + 表单回填适配器
- **Agent 服务层**：意图解析、字段映射、上下文管理
- **业务系统层**：原有表单页面、后端接口、数据库

### 3. 数据流

1. 用户在悬浮聊天框自然语言描述需求
2. 前端发送文本 + 当前页面标识到 Agent 服务
3. Agent 解析为 **结构化表单 JSON**（字段名 + 值）
4. 前端接收 JSON，**自动回填到当前页面原生表单**
5. 用户校验、微调后，按原有表单流程提交

### 4. 关键优势

- 不重复开发表单，**零冗余**
- 兼容所有复杂表单（下拉、级联、动态显隐）
- 体验接近原生表单，**无学习成本**
- 维护成本低，**一次开发全表单复用**

## 二、前端呈现：悬浮气泡 Agent（品牌logo、右下角）

### 1. 视觉样式

- **悬浮按钮**：页面右下角固定圆形按钮（品牌logo + 闪烁提示）
- **聊天窗口**：点击弹出侧边抽屉（宽 380px–420px，高 80vh）
- **窗口控制**：可拖动、最小化、关闭、全屏切换
- **样式规范**：贴合系统主题色，圆角 12px，阴影柔和

### 2. 聊天界面结构

- **头部**：品牌logo、关闭/最小化/全屏按钮
- **消息区，可以复用现有AIDashboard的聊天界面样式**：
  - 用户消息：右侧气泡，支持用户图像设置
  - AI 消息：左侧气泡（含文本、回填成功提示、错误提示）
  - 加载状态：AI 回复中显示加载动画
- **输入区**：文本框 + 发送按钮（支持 Enter 发送）
- **快捷操作**：清空对话、重新填写、一键回填

### 3. 交互流程

1. 打开表单页 → 右下角显示悬浮球
2. 点击悬浮球 → 展开聊天窗口
3. 用户输入："新增配方：新增配方1，手机号 13800138000，类型 VIP"
4. AI 解析 → 左侧显示："已解析：姓名=张三，手机=13800138000，客户类型=VIP"
5. 前端自动回填 → 表单对应字段填充数据
6. 用户确认 → 手动提交（沿用原有提交逻辑）

### 4. 核心功能点

- **多轮对话补全**：字段缺失时 AI 追问（如"请问客户所属行业是？"）
- **回填反馈**：回填成功高亮字段，失败提示错误
- **上下文保留**：切换表单页清空上下文，同页保留
- **防重复回填**：已填字段可选择覆盖/保留
- **历史记录**：同页面会话短期留存（刷新清空）

## 三、开发范围（明确不做/必做）

### ✅ 必须开发

1. 悬浮球 + 聊天窗口 UI 组件
2. 聊天消息发送/接收/渲染
3. **表单回填适配器**：根据字段名映射到 DOM 输入框/下拉/复选框
4. Agent 接口调用（请求/响应/异常）
5. 回填成功/失败/加载状态交互

### ❌ 禁止开发（模式三废弃）

- 不开发 Agent 内动态表单渲染
- 不重复定义表单结构、下拉字典、校验规则
- 不接管原有表单提交、接口调用、权限校验

---

## 四、接口定义（极简）

### 1. 解析接口

- **URL**：`POST /api/agent/parse-form`
- **请求**

```json
{
  "pageId": "customer-add",
  "utterance": "新增客户张三，手机号13800138000，类型VIP",
  "context": []
}
```

- **响应**

```json
{
  "code": 0,
  "data": {
    "fields": {
      "customerName": "张三",
      "phone": "13800138000",
      "customerType": "VIP"
    },
    "missingFields": ["industry"],
    "message": "已解析3个字段，缺少行业"
  }
}
```

### 2. 回填事件

- 前端监听 `agent:form-data` 事件，执行 `autoFillForm(fields)`

---

## 五、交付要求

1. **组件化**：独立 `AiAssistantFloat` 组件，全局引入
2. **无侵入**：不修改原有页面布局、路由、业务逻辑
3. **可配置**：位置（右下/左下）、宽度、主题色、是否开启
4. **兼容性**：支持 Vue2/Vue3/React，适配现有 UI 库（Element/AntD）
5. **性能**：聊天窗口懒加载，不影响首屏速度

---

## 六、一句话总结

**Agent = 智能翻译官**：把用户说的话，翻译成表单能懂的数据，自动填进原有表单；**UI = 右下角悬浮球**：不挡页面、随时唤起、边聊边填。
