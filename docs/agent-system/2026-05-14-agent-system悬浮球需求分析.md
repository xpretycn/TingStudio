# 表单录入：悬浮Agent（纯文字）非替代价值（vs 现有文件智能填单/导入）

悬浮Agent（纯文字）补上的，是**文件导入无法覆盖、表单侧高频刚需缺口**：

## 1. 零散字段快速补全（无文件、少量信息）

文件导入必须完整素材；实际录入常只有**3-5个零散字段**：

- 配方名：健脾养胃膏
- 业务员：张三
- 成品：700g
- 利润率：22%

悬浮Agent：**文字输入零散字段 → 自动回填**。
文件导入做不到：**没文件就没法填**。

## 2. 字段含义/规则/合规即时咨询

用户填表最卡：看不懂字段、不清楚规则、担心合规：

- “主料系数0.19合规吗？范围是啥？”
- “原料单位是g还是kg？”
- “利润率20%怎么算？”
- “配方命名规范是什么？”

悬浮Agent：**文字提问 → 实时查规则/库 → 直接回答/修正回填**。
文件导入做不到：**不会解释、不会校验、不会提醒**。

## 3. 字段自动生成（描述/制法/编码）

文件里**通常没有配方描述、制法、备注**，只能手动写：

- 生成配方专业描述
- 生成标准制法步骤
- 自动生成配方编码

悬浮Agent：**文字指令 → 生成文本回填**。
文件导入做不到：**解析不出文本类内容**。

## 4. 实时关联计算/校验（成本、报价、配比）

表单侧高频：填完算成本、校验配比、核对报价：

- 原料+包装+其他 → 自动算报价
- 主料系数是否在0.15–0.25
- 原料配比总和是否合规

悬浮Agent：**文字指令 → 实时计算/校验 → 回填结果**。
文件导入做不到：**只解析字段，不做实时运算校验**。

## 5. 漏字段识别+文字补问

文件解析常缺字段：没业务员、没成品重量、漏原料。
悬浮Agent：**自动识别缺失 → 文字追问 → 用户回复 → 回填**。
文件导入做不到：**不会追问、不会补全**。

## 6. 多轮文字微调（反复改参数）

表单常反复改：利润率、原料用量、配方名。
悬浮Agent：**文字指令修改 → 直接更新表单**。

文件导入做不到：**改一次重传，效率极低**。
