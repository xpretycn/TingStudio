# PRD：TingStudio Agent 智能对话系统

## 1. 产品概述

### 1.1 产品目标

将 TingStudio AI 助手从"简易聊天工具"升级为"Agent 驱动的智能业务执行器"，实现：
- 对话式业务操作（查询/新增/修改/删除）
- 多轮对话 + 意图识别 + 参数追问 + 操作确认
- 会话持久化 + 历史恢复
- 智能检索集成到 smart-tools 页面

### 1.2 目标用户

| 角色 | 使用场景 | 权限 |
|------|---------|------|
| 管理员 | 全部业务操作 + 数据分析 | 全部 Agent 工具 + 写操作 |
| 业务员 | 查询自己数据 + 受限操作 | 查询类工具 + 仅操作自己数据 |

### 1.3 核心价值

| 维度 | 现状 | 目标 |
|------|------|------|
| 对话能力 | 纯聊天，无工具调用 | Agent 驱动，支持 Tool Calling |
| 意图处理 | 无 | 显式意图识别 + 参数提取 + 追问/确认 |
| 业务操作 | 文件上传 + 手动表单 | 对话式交互（与现有方式并存） |
| 会话管理 | 无持久化 | SQLite 持久化 + 历史恢复 |
| 智能检索 | 内联在 AiAssistant | 独立 Tab 在 smart-tools 页面 |

### 2.2 P1：核心体验

#### F-P1-1：AI 意图处理层

**描述**：新增显式的意图识别模块和对话管理模块，替代当前隐式的 LLM Function Calling 意图识别。

**意图识别模块**：
- 输入：用户 query + 上下文（最近 N 轮对话）
- 输出：意图标签 + 关键参数
- 意图标签枚举：

| 意图标签 | 说明 | 示例 |
|---------|------|------|
| `query_data` | 查询数据 | "查看含有黄芪的配方" |
| `create_data` | 新增数据 | "帮我创建一个新配方" |
| `update_data` | 修改数据 | "把配方A的成品重量改为200g" |
| `delete_data` | 删除数据 | "删除业务员张三" |
| `calculate` | 计算/分析 | "计算这个配方的营养成分" |
| `chat` | 普通聊天 | "你好" / "什么是含量比" |
| `unclear` | 意图不明确 | 需要追问 |

**对话管理模块**：
- 参数不完整 → 生成追问话术（如"请提供配方名称"）
- 参数完整 + 写操作 → 生成确认话术（如"确认删除ID为123的记录？"）
- 参数完整 + 读操作 → 直接执行

**验收标准**：
- [ ] 意图识别准确率 ≥ 90%（针对业务场景测试集）
- [ ] 参数缺失时正确追问
- [ ] 写操作前必须确认
- [ ] 确认后正确执行工具

#### F-P1-2：前端 Agent 对话界面

**描述**：在 AIDashboard.vue 中支持 Agent 特有的交互模式。

**工具调用确认弹窗**：
- 当 Agent 返回 `tool_calls` 且工具标记为 `requiresConfirmation` 时
- 弹窗显示：工具名称 + 参数列表 + 确认/取消按钮
- 用户确认 → 发送确认消息继续执行
- 用户取消 → 发送取消消息，Agent 生成替代回复

**操作结果可视化**：
- 查询结果 → 表格展示（动态列）
- 计算结果 → 卡片展示（营养值/成本等）
- 创建/修改/删除 → 成功提示 + 操作摘要
- 分析结果 → 图表展示（趋势/对比）

**验收标准**：
- [ ] 工具调用确认弹窗正确显示
- [ ] 确认/取消交互正常
- [ ] 查询结果以表格展示
- [ ] 计算结果以卡片展示
- [ ] 写操作成功/失败有明确反馈

#### F-P1-3：会话持久化

**描述**：Agent 会话从内存 Map 迁移到 SQLite 数据库。

**数据库表**：
- `agent_sessions`：会话元数据
- `agent_messages`：消息记录（含角色、内容、工具调用信息）

**功能**：
- 新建对话 → 创建 session 记录
- 每轮对话 → 写入 message 记录
- 刷新页面 → 从数据库恢复最近会话
- 历史列表 → 展示所有会话
- 切换会话 → 加载对应消息历史
- 删除会话 → 清除 session + messages

**验收标准**：
- [ ] 刷新页面后会话不丢失
- [ ] 历史列表正确展示
- [ ] 可切换到历史会话继续对话
- [ ] 可删除历史会话

#### F-P1-4：扩展 NL2SQL

**描述**：增强自然语言检索能力。

**新增能力**：
- 跨表 JOIN 查询（如"查看配方及其原料信息"）
- 聚合分析（如"按业务员统计配方数量"、"按月统计销量"）
- 数据导出（CSV/Excel）

**验收标准**：
- [ ] 跨表查询正确生成 JOIN SQL
- [ ] 聚合查询正确生成 GROUP BY SQL
- [ ] 查询结果可导出为 CSV
- [ ] SQL 安全校验仍有效

#### F-P1-5：智能检索集成到 smart-tools

**描述**：在现有 SmartTools.vue 页面新增"智能检索"Tab（第三个 Tab）。

**Tab 结构**：
| Tab | 组件 | 状态 |
|-----|------|------|
| 智能填单 | SmartFormTab | 已有，不变 |
| 智能导入 | SmartImportTab | 已有，不变 |
| 智能检索 | SmartSearchTab（新） | 新增 |

**SmartSearchTab 功能**：
- 自然语言输入框 + 快捷标签
- SQL 预览卡片
- 结果表格展示（动态列）
- 高级筛选面板（表名/字段/条件）
- 结果导出（CSV）
- 搜索历史

**验收标准**：
- [ ] SmartTools 页面显示三个 Tab
- [ ] 智能检索 Tab 功能完整
- [ ] 搜索结果正确展示
- [ ] 可导出 CSV
- [ ] 智能填单/智能导入 Tab 不受影响

## 4. 用户流程

### 4.1 Agent 对话流程

```
用户输入 → 前端发送 /api/agent/chat
    ↓
后端意图识别 → 返回意图标签 + 参数
    ↓
┌─────────────────────────────────────┐
│ 参数完整？                           │
│   否 → 生成追问 → 前端展示追问       │
│   是 → 写操作？                      │
│     是 → 生成确认 → 前端弹窗确认     │
│       确认 → 执行工具 → 返回结果     │
│       取消 → Agent 生成替代回复      │
│     否 → 直接执行工具 → 返回结果     │
└─────────────────────────────────────┘
    ↓
前端结果可视化（表格/卡片/图表/提示）
```

### 4.2 智能检索流程

```
用户输入自然语言 → 前端发送 /api/ai/natural-search
    ↓
后端 NL2SQL → 生成 SQL → 安全校验 → 执行查询
    ↓
返回 { sql, rows, rowCount } → 前端展示
    ↓
用户可导出 CSV
```

## 5. 约束与边界

### 5.1 不修改的功能

- 智能填单 Tab（SmartFormTab）— 保持不变
- 智能导入 Tab（SmartImportTab）— 保持不变
- 配方管理页面 — 保持不变
- 原料管理页面 — 保持不变
- 文件上传功能 — 保持不变

### 5.2 技术约束

- 沿用 Vue 3 + TDesign + Express + SQLite
- 沿用自研 Agent 框架（toolRegistry + llmService + promptEngine）
- 不引入新的 UI 框架
- 不引入新的数据库
