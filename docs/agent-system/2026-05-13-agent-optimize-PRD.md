# PRD — AI助手Agent功能优化（查询专注模式）

> 版本: 1.0.0 | 日期: 2026-05-13 | 范围: AIDashboard页面Agent

---

## 1. 背景与目标

### 1.1 现状问题

| 问题 | 影响 | 严重度 |
|------|------|--------|
| Agent同时承担查询+写入+表单生成+确认流程，ReAct循环复杂度高 | 响应延迟大、容易超时 | P0 |
| 写入操作需二次确认/表单弹窗，SSE流中断后状态丢失 | 用户卡在"确认中"无法继续 | P0 |
| LLM调用无重试/超时/熔断，fallback链仅遍历一次 | 单provider故障导致整条请求失败 | P0 |
| SSE心跳15s间隔过长，断开无恢复机制 | 网络波动时连接静默断开 | P1 |
| 会话无过期清理，SQLite无限增长 | 长期运行后磁盘/查询性能下降 | P1 |
| agentController.ts 1548行，职责混杂 | 维护困难、bug难定位 | P2 |

### 1.2 优化目标

**核心定位转变**: Agent从"全能执行器"变为"查询理解+数据展示助手"

| 维度 | 优化前 | 优化后 |
|------|--------|--------|
| 查询 | 可用但不稳定 | 稳定可靠，Markdown清晰展示 |
| 写入 | 实际执行（需确认/表单） | 理解意图→返回导航指引 |
| 稳定性 | 无重试/超时/熔断 | 3次重试+30s超时+熔断保护 |
| 会话管理 | 无过期清理 | 7天自动过期+定时清理 |

## 2. 用户角色

| 角色 | 查询权限 | 写入指引 |
|------|----------|----------|
| 普通用户 | ✅ 全部查询Tool | ✅ 返回操作指引+导航链接 |

## 3. 功能需求

### 3.1 F1-1 查询类意图精准执行

**用户故事**: 作为用户，我输入"查看含有黄芪的配方"，Agent调用query_formulas Tool，以Markdown表格展示结果。

**查询场景矩阵**:

| 场景 | 用户输入示例 | 调用Tool | 展示格式 |
|------|-------------|----------|----------|
| 配方搜索 | "查看黄芪配方" | query_formulas | Markdown表格 |
| 配方详情 | "配方ID xxx的详情" | query_formulas(keyword) | Markdown卡片 |
| 原料搜索 | "有哪些药材类原料" | query_materials | Markdown表格 |
| 原料库存 | "库存低于10的原料" | nl2sql_query | Markdown表格 |
| 业务员查询 | "北京区域业务员" | query_salespersons | Markdown表格 |
| 销售分析 | "本月销售趋势" | analyze_sales | Markdown摘要+表格 |
| 营养计算 | "计算这个配方的营养" | calculate_nutrition | Markdown卡片 |
| 成本计算 | "这个配方成本多少" | calculate_cost | Markdown卡片 |
| 含量比校验 | "校验含量比" | validate_ratio_factor | Markdown卡片 |
| 高级查询 | "各区域配方数量统计" | nl2sql_query | Markdown表格 |

**Markdown展示规范**:
- 列表数据: 使用Markdown表格，列名中文映射
- 单条详情: 使用`### 标题` + `key: value`列表
- 计算结果: 使用结构化卡片格式
- 空结果: "数据库中暂无相关数据" + 引导语

### 3.2 F1-2 写入类意图理解+指引

**用户故事**: 作为用户，我输入"创建一个配方"，Agent理解意图后回复操作指引和导航链接。

**写入意图处理规则**:

| 意图 | Agent回复模板 |
|------|--------------|
| 创建配方 | "📋 创建配方请前往**配方管理**页面操作。\n\n👉 [前往创建配方](/formula/add)" |
| 编辑配方(有ID) | "✏️ 编辑配方请前往**配方管理**页面。\n\n👉 [前往编辑配方](/formula/edit/{id})" |
| 编辑配方(无ID) | "✏️ 请先告诉我配方名称，我帮您查找后提供编辑链接。" |
| 删除配方 | "⚠️ 删除操作请在**配方管理**页面操作。\n\n👉 [前往配方管理](/formula)" |
| 创建原料 | "📋 创建原料请前往**原料管理**页面操作。\n\n👉 [前往创建原料](/material/add)" |
| 编辑/删除原料 | 同上模式，链接到 `/material` |
| 创建业务员 | "📋 创建业务员请前往**业务员管理**页面操作。\n\n👉 [前往创建业务员](/salesman/add)" |
| 编辑/删除业务员 | 同上模式，链接到 `/salesman` |

**导航链接规则**:
- 有具体ID → 直接链接到编辑页 `/resource/edit/{id}`
- 无ID但有名称 → 先查询获取ID，再返回链接
- 无任何标识 → 链接到列表页 `/resource`

### 3.3 F1-3 SSE流式响应稳定性

**问题与方案**:

| 问题 | 方案 | 预期效果 |
|------|------|----------|
| 心跳15s间隔过长 | 缩短至8s，增加客户端心跳检测 | 减少静默断开 |
| 断开无恢复 | 客户端检测断开后自动重连（最多3次） | 网络波动自动恢复 |
| 流式解析出错 | 增加SSE数据校验，错误chunk跳过不中断 | 避免单chunk错误导致整流失败 |
| 写入确认中断流程 | 移除确认流程，写入意图直接回复指引 | SSE不再中断 |

### 3.4 F1-4 LLM调用容错

**重试策略**:

```
请求 → Provider A (30s超时)
  ├─ 成功 → 返回
  ├─ 超时 → 重试1 (同Provider, 30s)
  │   ├─ 成功 → 返回
  │   ├─ 超时 → 重试2 (同Provider, 30s)
  │   │   ├─ 成功 → 返回
  │   │   └─ 失败 → Provider B (30s)
  │   └─ 错误 → Provider B
  └─ 错误 → Provider B
```

**熔断规则**:
- 同一Provider连续3次失败 → 标记为"熔断"状态，5分钟内跳过
- 熔断恢复: 5分钟后允许1次试探请求，成功则恢复

### 3.5 F1-5 会话过期清理

**清理策略**:

| 数据 | 过期时间 | 清理方式 |
|------|----------|----------|
| agent_sessions | 7天无活动 | 定时任务每天凌晨3点清理 |
| agent_messages | 跟随session | CASCADE删除 |
| agent_pending_confirmations | 1小时 | 定时任务每小时清理 |
| agent_pending_forms | 1小时 | 定时任务每小时清理 |

## 5. 验收标准

| 编号 | 验收条件 | 优先级 |
|------|----------|--------|
| AC-1 | 输入查询类意图，Agent在5s内开始流式返回，结果以Markdown格式展示 | P0 |
| AC-2 | 输入写入类意图，Agent返回操作指引+可点击导航链接 | P0 |
| AC-3 | LLM Provider故障时，自动fallback到备选Provider，用户无感知 | P0 |
| AC-4 | SSE连接断开后，客户端自动重连，不丢失已有消息 | P1 |
| AC-5 | 7天未活动的会话自动清理，SQLite体积不持续增长 | P1 |
| AC-6 | 原有页面布局、路由、悬浮球Agent功能不受影响 | P0 |
