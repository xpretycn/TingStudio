## 实施步骤（按依赖顺序）

### 阶段一：真实变更摘要（数据基础）

#### 步骤 1.1 — 数据库迁移：添加 `changes_json` 字段

- **文件**: `backend/src/scripts/migrations/addChangesJsonToMaterials.ts`（新建）
- **操作**: 为 `materials` 表添加 `changes_json TEXT DEFAULT NULL` 字段
- **要求**:
  - 使用 `query()` 函数（兼容 SQLite + MySQL）
  - 幂等设计：先检查字段是否已存在
  - 同时更新 `init.sql` 中的表定义
- **SQL**: `ALTER TABLE materials ADD COLUMN changes_json TEXT DEFAULT NULL`

#### 步骤 1.2 — 后端：`buildChangesSummary` 函数

- **文件**: `backend/src/services/materialService.ts`（增强）
- **操作**: 新增 `buildChangesSummary(current, newData)` 函数
- **逻辑**:
  1. 定义字段标签映射：`{ name: "名称", unit: "单位", stock: "库存", unit_price: "单价", material_type: "类型", data_source: "数据源" }`
  2. 遍历映射，对比 `current[key]` 与 `newData[camelCaseKey]`
  3. 仅记录有差异的字段，生成 `[{ field, label, old, new }]` 数组
  4. 返回 JSON 字符串
- **营养数据变更**: 也需对比 `material_nutrition.per_100g_json` 中的营养素变化

#### 步骤 1.3 — 后端：增强 `createNewVersion` 生成变更摘要

- **文件**: `backend/src/services/materialService.ts`（增强）
- **操作**: 在 `createNewVersion()` 函数中：
  1. 调用 `buildChangesSummary(current, newData)` 生成变更 JSON
  2. 将 `changes_json` 写入新版本的 INSERT 语句
  3. 同时将旧版本的 `changes_json` 保留（历史记录）

#### 步骤 1.4 — 后端：增强 `getMaterialVersions` 返回结构化变更

- **文件**: `backend/src/services/materialService.ts`（增强）
- **操作**: 在 `getMaterialVersions()` 的版本列表映射中：
  1. 读取每个版本的 `changes_json` 字段
  2. 如果 `changes_json` 存在且非空，解析为结构化数据
  3. 将解析后的变更列表附加到 `changesDetail` 字段
  4. `changesSummary` 逻辑升级：有 `changes_json` 时生成可读摘要，否则保留占位文本

#### 步骤 1.5 — 前端：`MaterialVersion` 接口增强

- **文件**: `frontend/src/api/material.ts`（增强）
- **操作**: 为 `MaterialVersion` 接口新增字段：
  ```typescript
  changesDetail?: Array<{ field: string; label: string; old: any; new: any }>;
  ```

#### 步骤 1.6 — 前端：`MaterialVersions.vue` 变更摘要结构化展示

- **文件**: `frontend/src/views/materials/MaterialVersions.vue`（增强）
- **操作**:
  1. 修改 `formatChanges()` 函数：当 `changesDetail` 存在时，生成结构化展示
  2. 在时间线卡片 `tl-reason` 区域，将纯文本替换为变更列表：
     - 每行显示 `• 字段名：旧值 → 新值`
     - 最多展示 3 条，超出显示 `+N 项变更`
  3. 在版本快照详情面板中，变更摘要区域也展示结构化数据

### 阶段二：版本对比（依赖变更摘要数据）

#### 步骤 2.1 — 后端：`compareVersions` 服务函数

- **文件**: `backend/src/services/materialService.ts`（增强）
- **操作**: 新增 `compareVersions(materialId, versionId1, versionId2)` 函数
- **逻辑**:
  1. 查询两个版本的完整数据（基本信息 + 营养数据）
  2. 对比基本信息字段，生成 diff 列表：`{ field, label, left, right, change, type }`
  3. 对比营养数据，生成 diff 列表
  4. 计算变化百分比：`(right - left) / |left| * 100`
  5. 标记变更类型：`increase` / `decrease` / `unchanged` / `new` / `deleted`
- **返回格式**:
  ```typescript
  {
    left: { versionId, version, name, ... },
    right: { versionId, version, name, ... },
    diff: {
      basic: [...],
      nutrition: [...]
    }
  }
  ```

#### 步骤 2.2 — 后端：`compareVersions` 控制器

- **文件**: `backend/src/controllers/materialController.ts`（增强）
- **操作**: 新增 `compareMaterialVersions(req, res)` 函数
- **逻辑**:
  1. 从 `req.params.id` 获取原料 ID
  2. 从 `req.query.v1` / `req.query.v2` 获取对比版本 ID
  3. 参数校验：v1 和 v2 必填
  4. 调用 `materialService.compareVersions()`
  5. 返回 `success(data)`

#### 步骤 2.3 — 后端：注册对比路由

- **文件**: `backend/src/routes/materials.ts`（增强）
- **操作**: 新增路由 `GET /:id/versions/compare`
- **注意**: 路由注册顺序 — `compare` 必须在 `:versionId` 之前，避免被通配路由捕获

#### 步骤 2.4 — 前端：新增 `compareVersions` API

- **文件**: `frontend/src/api/material.ts`（增强）
- **操作**: 新增方法：
  ```typescript
  compareVersions(id: string, v1: string, v2: string) {
    return http.get<any, CompareResult>(`/materials/${id}/versions/compare`, { params: { v1, v2 } })
  }
  ```
- **新增接口**: `CompareResult`, `CompareDiffItem` 等类型定义

#### 步骤 2.5 — 前端：新增 `MaterialVersionCompare.vue` 页面

- **文件**: `frontend/src/views/materials/MaterialVersionCompare.vue`（新建）
- **设计参考**: `VersionCompare.vue`（配方版本对比）的布局模式
- **UI 结构**:
  1. **页面头部**: 返回按钮 + 面包屑 + 原料名称 + 版本对比标签
  2. **版本选择器**: 左侧版本下拉 + 右侧版本下拉（从版本列表填充）
  3. **对比结果区**:
     - 基本信息对比表格：字段 | v1值 | v2值 | 变更标记
     - 营养成分对比表格：营养素 | v1值 | v2值 | 变更标记
  4. **变更标记样式**:
     - 增大 `↑ +N%` → 绿色
     - 减小 `↓ -N%` → 红色
     - 无变化 `—` → 灰色
     - 新增 `NEW` → 蓝色
     - 删除 `DEL` → 红色删除线
- **样式**: 使用项目设计令牌，遵循 `DESIGN.md` 规范

#### 步骤 2.6 — 前端：注册版本对比路由

- **文件**: `frontend/src/router/index.ts`（增强）
- **操作**: 在 `materials/:id/versions` 路由之后新增：
  ```typescript
  {
    path: "materials/:id/versions/compare",
    name: "MaterialVersionCompare",
    component: () => import("@/views/materials/MaterialVersionCompare.vue"),
    meta: { title: "版本对比", hideHeader: true }
  }
  ```

#### 步骤 2.7 — 前端：`MaterialVersions.vue` 增加"对比"按钮

- **文件**: `frontend/src/views/materials/MaterialVersions.vue`（增强）
- **操作**:
  1. 在页面头部 actions 区域增加"版本对比"按钮
  2. 点击后跳转到 `MaterialVersionCompare` 路由
  3. 在时间线卡片的操作区域增加"与此版本对比"快捷入口
  4. 支持选中两个版本后直接跳转对比（通过 query 参数 `?v1=xxx&v2=xxx`）

## 风险与注意事项

1. **路由注册顺序**: `materials/:id/versions/compare` 必须在 `materials/:id/versions/:versionId` 之前注册，否则 `compare` 会被当作 `:versionId` 捕获
2. **数据库迁移幂等性**: 迁移脚本必须检查字段是否已存在，避免重复执行报错
3. **营养数据对比**: `per_100g_json` 是 JSON 字符串，需要解析后逐字段对比，注意空值处理
4. **版本对比性能**: 两个版本的数据量不大，无需分页，但需注意营养数据解析的性能
5. **快照刷新副作用**: 刷新快照会创建新的配方版本（`versionController.ts` 已实现此逻辑），前端需正确处理刷新后的数据重载
6. **设计令牌**: 所有新增 UI 必须使用 `DESIGN.md` 定义的设计令牌，禁止硬编码颜色/间距
