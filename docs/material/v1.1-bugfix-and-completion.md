## 需要修复的问题

### 🔴 严重（构建阻断 / 功能不可用）

#### Bug 1：路由缺失 — `MaterialVersionCompare` 路由未注册

- **文件**: `frontend/src/router/index.ts`
- **问题**: `MaterialVersionCompare` 路由未注册，点击"版本对比"按钮将导致路由解析失败
- **修复**: 在 `materials/:id/versions` 路由之后（第 70 行后）添加：
  ```typescript
  {
    path: "materials/:id/versions/compare",
    name: "MaterialVersionCompare",
    component: () => import("@/views/materials/MaterialVersionCompare.vue"),
    meta: { title: "版本对比", hideHeader: true },
  },
  ```

#### Bug 2：Import 路径错误 — `MaterialVersions.vue` 引用不存在的模块

- **文件**: `frontend/src/views/materials/MaterialVersions.vue` 第 7 行
- **问题**: `import { formatTimestamp, splitDateTime } from "@/utils/date"` — `@/utils/date` 不存在
- **修复**: 改为 `import { formatTimestamp, splitDateTime } from "@/utils/timeFormat"`

### 🟡 中等（功能不完整）

#### Bug 3：搜索/筛选功能未生效 — 模板使用 `versions` 而非 `filteredVersions`

- **文件**: `frontend/src/views/materials/MaterialVersions.vue`
- **问题**: 
  - 脚本中定义了 `searchKeyword`、`filterType`、`filteredVersions` 计算属性
  - 但模板中 `v-for="(ver, index) in versions"` 使用的是原始 `versions` 数组
  - 模板中没有搜索框和筛选标签的 UI 元素
- **修复**:
  1. 在 `section-head` 区域添加搜索框和筛选标签 UI
  2. 将模板中所有 `versions` 引用替换为 `filteredVersions`（v-for、空状态判断、计数等）

### 🟢 轻微（非阻断，可后续处理）

#### Issue 4：版本计数显示不准确

- **文件**: `frontend/src/views/materials/MaterialVersions.vue` 第 182 行
- **问题**: `{{ versions.length }}` 显示的是全部版本数，而非筛选后的数量
- **修复**: 改为 `{{ filteredVersions.length }}`

## 实施步骤

### 步骤 1：修复路由缺失（Bug 1）

- **文件**: `frontend/src/router/index.ts`
- **操作**: 在 `materials/:id/versions` 路由之后添加 `MaterialVersionCompare` 路由
- **验证**: 确认路由注册在 `:versionId` 通配路由之前（虽然当前 materials 下没有 `:versionId` 路由，但仍需注意顺序）

### 步骤 2：修复 Import 路径（Bug 2）

- **文件**: `frontend/src/views/materials/MaterialVersions.vue`
- **操作**: 将第 7 行 `@/utils/date` 改为 `@/utils/timeFormat`

### 步骤 3：完善搜索/筛选 UI（Bug 3）

- **文件**: `frontend/src/views/materials/MaterialVersions.vue`
- **操作**:
  1. 在 `section-head` 区域添加搜索输入框（`t-input`）和筛选标签（`全部`/`最新`/`历史`）

  4. 将版本计数 `versions.length` 改为 `filteredVersions.length`
  5. 将时间线连接线判断 `index < versions.length - 1` 改为 `index < filteredVersions.length - 1`

  3. `cd frontend && npm run build` — 确认构建通过

## 风险与注意事项

1. **路由注册顺序**: 新增的 `materials/:id/versions/compare` 路由必须在任何 `:param` 通配路由之前
2. **搜索/筛选 UI 风格**: 需与现有 `MaterialVersions.vue` 的设计风格保持一致，使用 TDesign 组件和项目设计令牌
3. **空状态区分**: 搜索无结果时应显示"未找到匹配的版本"而非"暂无版本数据"
