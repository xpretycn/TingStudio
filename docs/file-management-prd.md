# 文件管理功能 PRD 文档

> 生成日期：2026-05-19
> 适用项目：TingStudio 食品配方工作数据管理平台

## 1. Executive Summary

### 1.1 Problem Statement

当前文件管理功能实现完整但定位模糊，用户和管理员权限边界不清晰，部分功能可能冗余。需要明确文件管理的核心定位，避免给用户带来不必要的复杂度。

### 1.2 Proposed Solution

明确文件管理为**"管理员专属的解析原文件留底中心"**，精简功能，强化追溯能力。用户通过解析历史缓存查看结果，无需关心原始文件；管理员通过文件管理进行追溯、差错、留底。

### 1.3 Success Criteria

| KPI | 目标值 |
|-----|--------|
| 文件管理入口可见性 | 仅管理员可见 |
| 原文件可追溯率 | 100%（AI解析成功即自动上传） |
| 操作审计日志完整率 | 100% |
| 存储空间可监控性 | 支持按文件大小统计和告警 |

## 2. User Experience & Functionality

### 2.1 User Personas

| 角色 | 需求 | 使用场景 |
|------|------|---------|
| 系统管理员（admin） | 追溯原文件、差错排查、合规审计 | 查看所有AI解析上传的原文件，下载核对，清理过期文件 |
| 普通用户（formulist） | 无需直接操作文件 | 通过解析历史缓存查看解析结果 |

### 2.2 User Stories

#### Story 1：查看原文件列表

**As a** 系统管理员  
**I want to** 查看所有AI解析上传的原文件列表  
**So that** 我可以进行追溯和审计

**Acceptance Criteria：**
- 列表展示：文件名、上传时间、上传人、关联配方/原料、文件大小、文件状态
- 支持按时间范围、文件类型、关联状态筛选
- 支持按文件名搜索
- 分页展示，默认每页20条

#### Story 2：预览/下载原文件

**As a** 系统管理员  
**I want to** 预览或下载原文件  
**So that** 我可以核对解析结果

**Acceptance Criteria：**
- Excel文件支持在线预览（前500行）
- 图片文件支持缩略图和原图查看
- 所有文件支持下载到本地
- 预览/下载操作记入审计日志

#### Story 3：查看操作审计日志

**As a** 系统管理员  
**I want to** 查看文件的操作记录  
**So that** 我可以进行审计

**Acceptance Criteria：**
- 展示文件的上传、下载、删除、关联等操作记录
- 记录操作人、操作时间、IP地址
- 按时间倒序排列

#### Story 4：清理原文件

**As a** 系统管理员  
**I want to** 删除不再需要的原文件  
**So that** 我可以释放存储空间

**Acceptance Criteria：**
- 支持单文件删除
- 支持批量删除
- 删除前二次确认（使用Popconfirm）
- 删除操作记入审计日志
- 仅管理员可执行删除

### 2.3 Non-Goals（明确不做）

| 功能 | 不做理由 |
|------|---------|
| 用户个人文件管理 | 用户无需上传/管理自己的文件，AI已自动上传 |
| 复杂的文件版本管理 | 当前版本机制已足够 |
| 文件编辑/在线修改 | 只读追溯，不涉及修改 |
| 用户可见的文件管理入口 | 用户通过解析历史缓存查看结果即可 |
| 手动上传入口 | AI解析成功后自动上传，无需手动上传 |

## 7. 总结

### 7.1 核心结论

1. **文件管理有必要保留**，但定位应为**"管理员专属的解析原文件留底中心"**，而非用户功能
2. **当前实现已经相当完善**，主要需要调整的是产品定位和入口可见性，而非大量开发
3. **与解析历史缓存的关系**：解析历史缓存面向用户展示结果，文件管理面向管理员留存原文件，两者互补
