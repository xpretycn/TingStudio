# 原料枚举字段管理 — PRD 产品需求文档

> 日期：2026-05-25 | 版本：v1.0 | 状态：待确认

---

## 1. 背景与目标

### 1.1 背景

当前原料（materials）数据仅包含基础信息（名称、编码、类型、库存、单价）和营养成分数据，缺少对原料**性状、口感、功效**的描述。这些属性是配方师日常工作中的核心参考维度，直接影响配方设计和原料筛选决策。

### 1.2 目标

1. 为原料补充**性状、口感、功效**三个多选枚举字段
2. 提供独立的枚举值管理功能，管理员可动态维护选项
3. 原料表单/列表/详情页完整展示这些字段
4. 历史数据无需强制补充，平滑过渡

---

## 2. 用户角色

| 角色 | 枚举管理 | 原料编辑 |
|------|---------|---------|
| admin | 增删改查枚举值 | 可编辑所有原料的枚举字段 |
| formulist | 不可管理枚举值 | 仅可编辑自己创建的原料的枚举字段 |

---

## 3. 功能需求

### 3.1 枚举值管理（独立页面）

**页面路径**：`/settings/enums`

**权限**：仅 admin 可访问

**功能清单**：

| # | 功能 | 说明 |
|---|------|------|
| F3.1.1 | 按分类查看枚举值 | 三个 Tab：性状 / 口感 / 功效，每个 Tab 展示该分类下的所有枚举值 |
| F3.1.2 | 新增枚举值 | 点击"添加"按钮，输入枚举值名称，保存后立即生效 |
| F3.1.3 | 编辑枚举值 | 点击枚举值可修改名称 |
| F3.1.4 | 删除枚举值 | 删除前需确认；若该枚举值已被原料使用，提示"该选项已被 N 个原料使用，确认删除？" |
| F3.1.5 | 搜索枚举值 | 支持关键字搜索过滤 |

### 3.2 原料表单集成

**页面**：MaterialForm.vue（新建/编辑原料）

| # | 功能 | 说明 |
|---|------|------|
| F3.2.1 | 性状多选 | 下拉多选，选项从枚举接口动态获取 |
| F3.2.2 | 口感多选 | 下拉多选，选项从枚举接口动态获取 |
| F3.2.3 | 功效多选 | 下拉多选，选项从枚举接口动态获取 |
| F3.2.4 | 表单验证 | 三个字段均为选填，不强制填写 |

### 3.3 原料列表展示

**页面**：MaterialList.vue

| # | 功能 | 说明 |
|---|------|------|
| F3.3.1 | 列表新增列 | 表格新增"性状""口感""功效"三列，以标签形式展示 |
| F3.3.2 | 标签折叠 | 当选项超过3个时，显示前3个 + "+N" 折叠提示 |

### 3.4 原料详情展示

**页面**：MaterialDetail.vue

| # | 功能 | 说明 |
|---|------|------|
| F3.4.1 | 详情新增区域 | 在原料概况卡片中新增"性状""口感""功效"展示，以标签形式 |

## 5. 非功能需求

| 维度 | 要求 |
|------|------|
| 性能 | 枚举接口响应 < 200ms，前端下拉选项加载无感知 |
| 兼容性 | 历史原料数据不受影响，新字段允许为空 |
| 安全性 | 枚举管理接口仅 admin 可调用，后端强制校验角色 |
| 可扩展 | 枚举表设计支持未来新增其他分类（如产地、季节等） |

---

## 6. 验收标准

- [ ] 管理员可在 `/settings/enums` 页面对三类枚举值进行增删改查
- [ ] 非管理员无法访问枚举管理页面和接口
- [ ] 原料新建/编辑表单中可多选性状、口感、功效
- [ ] 原料列表页展示三个字段（标签形式，超过3项折叠）
- [ ] 原料详情页展示三个字段
- [ ] 删除已使用的枚举值时给出确认提示
- [ ] 历史原料数据正常展示，新字段为空时不报错
- [ ] 种子数据包含所有初始枚举值
