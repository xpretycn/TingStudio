# PRD：快速配方录入交互流程优化

## 1. Executive Summary

### Problem Statement

当前快速配方录入流程要求用户在进入编辑前手动输入配方名称，且无法复用历史配方数据。配方师在反复调试相似配方时需要重复配置原料和参数，效率低下。同时，QuickFormulaPanel 页面受 Home.vue 顶部导航栏挤压，操作区域有限。

### Proposed Solution

引入「快速配方」概念，在 QuickFormulaPanel 左侧增加可折叠的历史快速配方面板，支持选择/新建/删除快速配方。快速配方仅要求配方名称必填，原料等核心数据可为空。编辑完成后可保存为快速配方，也可通过抽屉补充必填信息后发布为正式配方草稿进入审核中心。

### Success Criteria

1. 配方师从进入快速录入到开始编辑原料的操作步骤 ≤ 2 步
2. 历史快速配方列表加载时间 < 500ms（50 条以内）
3. 左侧面板折叠/展开动画流畅，无卡顿
4. 快速配方发布为正式配方草稿成功率 ≥ 99%
5. QuickFormulaPanel 操作区域高度增加 ≥ 56px（取消 header 后）

## 2. User Experience & Functionality

### 2.1 User Personas

| 角色 | 场景 | 核心诉求 |
|------|------|---------|
| **formulist（配方师）** | 日常配方调试，反复调整原料配比 | 快速进入编辑、复用历史配方、一键发布 |
| **admin（管理员）** | 查看所有配方师的工作进度，直接发布配方 | 全局可见、直接操作 |

### 2.2 User Stories

#### US-1：快速进入配方编辑

> As a 配方师, I want to 从左侧面板选择一个历史快速配方或新建快速配方, so that 我能跳过重复配置直接开始编辑原料。

**Acceptance Criteria:**
- 左侧面板默认展开，显示历史快速配方列表
- 点击列表项即可加载该快速配方数据到工作区
- 点击「新建」按钮弹出输入配方名称的对话框，确认后进入编辑
- 左侧面板可折叠，折叠后仅显示图标按钮

#### US-2：管理历史快速配方

> As a 配方师, I want to 在左侧面板中删除不需要的历史快速配方, so that 列表保持整洁。

**Acceptance Criteria:**
- 每个列表项有删除按钮，点击后弹出确认对话框
- 删除后列表实时更新
- admin 可删除任意快速配方，formulist 仅可删除自己的

#### US-3：保存快速配方

> As a 配方师, I want to 在编辑过程中随时保存当前进度为快速配方, so that 我可以稍后继续编辑而不丢失数据。

**Acceptance Criteria:**
- 保存按钮始终可用，不要求原料/业务员等字段完整
- 保存成功后显示成功提示
- 保存后左侧列表实时更新

#### US-4：发布快速配方

> As a 配方师, I want to 在编辑完成后通过抽屉补充必填信息并发布, so that 快速配方升级为正式配方草稿进入审核中心。

**Acceptance Criteria:**
- 点击「发布」按钮弹出右侧抽屉
- 抽屉中必填字段：业务员、配方描述
- 抽屉中可选字段：制备方法
- 发布成功后原快速配方标记为「已发布」，不可再编辑
- 发布成功后自动跳转到工作台或配方详情页

#### US-5：复用已发布快速配方

> As a 配方师, I want to 从已发布的快速配方创建新的快速配方, so that 我能基于已有配方快速开始新的调试。

**Acceptance Criteria:**
- 已发布的快速配方在列表中显示「已发布」标签
- 点击已发布的快速配方，弹出确认「基于此配方新建」
- 新建的快速配方继承原配方的原料和参数数据

#### US-6：全屏操作区域

> As a 配方师, I want to 在快速录入页面获得更大的操作区域, so that 我能更舒适地编辑配方。

**Acceptance Criteria:**
- QuickFormulaPanel 页面隐藏 Home.vue 的顶部 header
- 页面自带毛玻璃效果的顶部操作栏
- 操作区域高度增加约 56px

### 2.3 Non-Goals

- 不做快速配方的搜索/筛选功能（后续迭代）
- 不做快速配方的排序功能（后续迭代）
- 不做批量删除快速配方（后续迭代）
- 不做快速配方的分享/协作功能
- 不修改现有配方审核流程的核心逻辑

## 3. Technical Specifications

### 3.1 Architecture Overview

```
┌─────────────────────────────────────────────────────────────┐
│                   QuickFormulaPanel.vue                      │
│  ┌──────────────┐  ┌──────────────────────────────────────┐ │
│  │  左侧面板     │  │           右侧工作区                  │ │
│  │ (可折叠)      │  │                                      │ │
│  │              │  │  ┌──────────┐  ┌──────────────────┐  │ │
│  │ [折叠按钮]   │  │  │ Formula  │  │  MaterialPool    │  │ │
│  │              │  │  │ Editor   │  │                  │  │ │
│  │ [新建按钮]   │  │  │          │  │                  │  │ │
│  │              │  │  └──────────┘  └──────────────────┘  │ │
│  │ 配方列表:    │  │                                      │ │
│  │ - 配方A 🟢   │  │  ┌──────────────────────────────────┐│ │
│  │ - 配方B 🔵   │  │  │      FormulaDashboard            ││ │
│  │ - 配方C ✓    │  │  └──────────────────────────────────┘│ │
│  │              │  │                                      │ │
│  └──────────────┘  └──────────────────────────────────────┘ │
│                                                              │
│  ┌─────────────────────────────────────────────────────────┐ │
│  │              发布抽屉 (t-drawer, 右侧弹出)               │ │
│  │  必填：业务员、描述   可选：制备方法                       │ │
│  └─────────────────────────────────────────────────────────┘ │
└─────────────────────────────────────────────────────────────┘
```

### 3.2 Data Flow

```
用户操作 → QuickFormulaPanel → QuickFormulaStore → QuickFormulaApi → /api/quick-formulas → quickFormulaController → quick_formulas 表
                                                                                    │
发布时 → /api/quick-formulas/:id/publish → 创建 formulas 记录 + formula_versions 记录 → 审核中心
                                                                                    │
                              → 更新 quick_formulas.status = 'published'
```

### 3.3 Integration Points

| 集成点 | 说明 |
|--------|------|
| 现有配方创建接口 | 发布时复用 `formulaController.createFormula` 的核心逻辑 |
| 审核中心 | 发布后创建 `formula_versions` 记录，status = `draft`（formulist）或 `published`（admin） |
| 业务员接口 | 发布抽屉中需要调用 `/api/salesmen` 获取业务员列表 |
| 原料接口 | 编辑时复用现有 `/api/materials` 获取原料数据 |
| 路由系统 | 修改 `/formulas/quick` 路由 meta，增加 `hideHeader: true` |

### 3.4 Security & Privacy

- 快速配方 API 全部需要 `authMiddleware` 认证
- 数据隔离：formulist 仅可见/操作自己创建的快速配方
- 发布时验证业务员 ID 有效性
- 配方名称唯一性校验（同用户下不可重复）

## 4. Risks & Roadmap

### 4.1 Phased Rollout

| 阶段 | 内容 | 状态 |
|------|------|------|
| **MVP** | 快速配方数据表 + CRUD API + 左侧可折叠面板 + 保存/发布 | 本次实现 |
| **v1.1** | 快速配方搜索/筛选、排序 | 后续迭代 |
| **v1.2** | 批量操作、模板化复用 | 后续迭代 |

### 4.2 Technical Risks

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 发布时数据转换复杂 | 快速配方字段可能不完整 | 前端抽屉强制补充必填字段，后端二次校验 |
| 左侧面板与工作区布局冲突 | 窄屏下操作区域不足 | 面板可折叠，折叠后仅 52px 宽 |
| 快速配方名称重复 | 多用户可能创建同名配方 | 同用户下名称唯一性校验 |
| 发布后原快速配方状态管理 | 误操作发布 | 已发布状态不可逆，但可基于它创建新快速配方 |
