# 智能工具页面布局与功能分析 PRD

## 1. Executive Summary

**问题陈述**：智能工具页面当前布局存在信息层级冗余、核心操作路径被遮挡、部分功能模块价值密度低等问题，导致用户进入页面后需要滚动才能触及核心功能，且仪表盘卡片与操作动态区占据了大量空间但提供的信息在其他区域已有体现。

**建议方案**：精简页面结构，移除冗余的仪表盘和操作动态区，将页面重心聚焦到核心工具功能上；同时补充已开发但未集成的智能搜索功能，以及缺失的效率增强特性。

**成功指标**：
- 核心功能（填单/导入）首屏可见，无需滚动
- 页面交互步骤减少 ≥ 30%（移除冗余中间层）
- 用户从进入页面到开始操作 ≤ 2 次点击

## 3. 冗余设计识别

### 🔴 冗余 1：仪表盘统计卡片（建议移除）

**当前 4 张卡片逐一分析：**

| 卡片 | 信息价值 | 冗余原因 |
|------|---------|---------|
| **可用模型** | 低 | 模型数量和选择状态已在工具栏模型选择器中完整展示，卡片仅重复显示数字 |
| **填单状态** | 低 | 填单的"已完成/已终止/解析中/待解析"状态在 SmartFormTab 内部已有完整的进度面板和阶段提示，卡片只是摘要 |
| **导入状态** | 低 | 同上，SmartImportTab 内部已有完整的解析状态展示 |
| **Token 用量** | 低 | 这是运营级数据，对日常操作无指导意义；且数据来源不稳定（有 usageStats 时显示月度，否则显示当次累计） |

**核心问题**：
- 4 张卡片占据首屏约 1/3 空间，但均为只读摘要信息，不可交互、不可点击
- 用户进入页面的核心目标是使用工具（填单/导入），而非查看统计
- 卡片数据与工具栏/标签页内容高度重复
- 卡片的 hover 动画、终止脉冲动画等视觉效果增加了代码复杂度但无功能价值

**建议**：整体移除 dashboard-grid 区域，将模型选择器和标题直接作为页面顶部。

---

### 🔴 冗余 2：操作动态时间线（建议移除或大幅精简）

**当前实现问题**：

1. **数据不持久**：activityHistory 是 ref([]) 内存变量，页面刷新即丢失，无法作为真正的操作历史
2. **信息重复**：时间线记录的内容（"智能配方解析完成"、"智能原料导入完成"、"解析异常"）在 SmartFormTab/SmartImportTab 内部已有更详细的状态展示
3. **watch 重复监听**：SmartTools 通过 watch(aiStore.parseResult) 添加活动记录，但子组件自身也监听同一状态来更新 UI，逻辑重复
4. **分页过度设计**：最多 50 条记录、每页 4 条、带翻页控件——对于临时内存数据来说过于复杂

**建议**：移除整个 activity-section，操作反馈由各标签页组件自行处理（当前已如此）。

---

### 🟡 冗余 3：智能工具中心引导卡（建议移除）

**当前实现**：
- 绿色渐变卡片，显示引导文案 + "前往 AI 助手"按钮
- 文案根据模型配置状态动态变化（3 种状态）
- 底部展示可用模型数量（又一次重复仪表盘信息）

**问题**：
- 这是一个导航入口，不是工具功能，放在工具页面底部作为 1/3 区域的卡片过于奢侈
- "前往 AI 助手"可以通过顶部导航栏实现，无需占用主内容区
- 引导文案对已熟悉系统的用户无价值

**建议**：移除此卡片。如需引导新用户，可在空状态时显示引导提示。

---

### 🟡 冗余 4：2 个 Tab 的可折叠侧边栏（建议简化）

**当前实现**：
- 170px 宽的可折叠侧边栏，含折叠按钮、2 个 tab 项
- 折叠状态持久化到 localStorage
- 折叠后 56px，仅显示图标

**问题**：
- 仅有 2 个标签时，垂直侧边栏 + 折叠功能的 ROI 极低
- 折叠后图标模式节省的空间有限（114px），但增加了交互复杂度
- 左侧栏占用的水平空间减少了核心内容区的可用宽度

**建议**：改为水平 Tab 栏（与工具栏合并），释放内容区宽度。当标签数量 ≥ 4 时再考虑侧边栏。

---

### 🟡 冗余 5：工具栏标题区（建议精简）

**当前实现**：
- 左侧：h3 标题 "智能工具" + p 副标题 "AI 驱动的配方解析与原料导入"
- 右侧：模型选择器

**问题**：
- 用户已通过导航菜单进入"智能工具"页面，标题重复
- 副标题对已熟悉用户无增量价值

**建议**：将标题精简为一行，与 Tab 栏合并。如：智能工具 | [智能填单] [智能导入] | 模型: [选择器]

---

## 4. 缺失功能识别

### 🟢 缺失 1：智能搜索标签页（已开发未集成）

**发现**：项目中存在 SmartSearchTab.vue，支持自然语言转 SQL 查询，但未在 SmartTools.vue 中注册。

**功能概览**：
- 自然语言输入 → AI 生成 SQL → 执行查询 → 表格展示结果
- 快捷标签（"含黄芪的配方"、"按业务员统计"等）
- 查询类型标签（简单查询/跨表查询/聚合分析）
- CSV 导出
- 搜索历史（最多 10 条）

**建议**：作为第三个标签页集成到智能工具页面，标签名"智能查询"。

---

### 🟢 缺失 2：操作历史持久化

**当前问题**：所有操作记录仅存于内存，刷新即丢失。

**建议**：
- 将操作历史持久化到后端（新增 ai_operations 表）
- 记录字段：操作类型、输入文件、AI 模型、Token 消耗、耗时、结果摘要、创建时间
- 在页面中提供"历史记录"入口（可放在工具栏右侧）

---

### 🟢 缺失 3：模型选择的全局一致性

**当前问题**：
- SmartTools 页面有独立的模型选择器
- AI 助手页面（AIDashboard.vue）也有独立的模型选择
- 两处的选择状态可能不一致

**建议**：模型选择应通过 aiStore 全局统一管理，任何页面切换模型后，其他页面自动同步。

---

### 🟢 缺失 4：批量操作的撤销/回滚

**当前问题**：
- SmartImportTab 的批量录入一旦提交无法撤销
- SmartFormTab 创建配方后需手动删除

**建议**：
- 批量导入前生成预览快照
- 提供"撤销上次导入"功能（限时，如 5 分钟内）

---

### 🟢 缺失 5：文件解析模板/预设

**当前问题**：每次解析都是全新开始，无法复用之前的解析配置。

**建议**：
- 保存常用解析配置为模板（如"配方文件模板"、"营养数据模板"）
- 模板包含：默认模型、字段映射规则、校验规则
- 支持从历史成功解析中创建模板

---

## 5. 优化后的页面布局建议

```
优化后布局
┌──────────────────────────────────────────────────┐
│  工具栏: [智能填单] [智能导入] [智能查询]  模型:[选择器] │  ← 一行搞定
├──────────────────────────────────────────────────┤
│                                                  │
│  核心内容区（全宽）                                 │  ← 首屏即达
│  SmartFormTab / SmartImportTab / SmartSearchTab  │
│                                                  │
│                                                  │
└──────────────────────────────────────────────────┘
```

**关键变化**：
1. ❌ 移除仪表盘卡片区 → 核心功能首屏可见
2. ❌ 移除操作动态区 → 减少页面长度
3. ❌ 移除引导卡 → 导航通过顶部菜单实现
4. ✅ 侧边栏 → 水平 Tab 栏 → 释放内容区宽度
5. ✅ 新增智能查询标签页 → 补充已有功能
6. ✅ 工具栏精简为一行 → 标题+Tab+模型选择器

## 6. 风险与实施建议

### 风险

| 风险 | 影响 | 缓解措施 |
|------|------|---------|
| 移除仪表盘后 Token 用量无处查看 | 低 | 可在模型选择器旁添加小型用量指示器，或移至个人设置页 |
| 移除操作动态后用户无法回顾操作 | 中 | 需同步实现操作历史持久化功能 |
| SmartSearchTab 未集成可能有未完成功能 | 中 | 需先验证 SmartSearchTab 的完成度和稳定性 |

### 分阶段实施

**Phase 1（快速优化）**：
- 移除仪表盘区域
- 移除操作动态和引导卡区域
- 将侧边栏改为水平 Tab 栏
- 精简工具栏标题区

**Phase 2（功能补充）**：
- 集成 SmartSearchTab 为第三个标签页
- 实现操作历史持久化

**Phase 3（体验增强）**：
- 解析模板/预设功能
- 批量操作撤销
- 模型选择全局一致性
