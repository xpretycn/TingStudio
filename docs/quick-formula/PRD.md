# PRD：快速配方录入功能

## 1. Executive Summary

### Problem Statement

当前配方录入存在两大痛点：AI 助手录入依赖现成配方文件无法自由发挥；手动录入页面（FormulaForm）功能繁杂、无关因素过多，操作效率低下。配方师需要一个"所见即所得"的快速录入入口。

### Proposed Solution

在 Dashboard 页面集成**快速配方录入功能**，采用上下分栏布局：上方实时仪表盘展示营养与成本计算，下方左右双栏——左栏配方编辑区、右栏动态原料池。原料池以"鱼"式动态游动展示，hover 暂停并展示详情，点击添加到编辑区。提交后生成草稿配方，走正常审批流程。支持配方模板的创建与复用。

### Success Criteria

1. 配方录入操作步骤减少 50% 以上（对比现有 FormulaForm 流程）
2. 营养数据与成本实时计算延迟 < 100ms（编辑即算）
3. 原料池加载完成时间 < 500ms（100 条原料场景）
4. 快速录入的配方草稿与现有配方管理流程 100% 兼容

## 2. User Experience & Functionality

### 2.1 User Personas

| 角色                    | 场景                           | 核心诉求                       |
| ----------------------- | ------------------------------ | ------------------------------ |
| **配方师（formulist）** | 日常配方开发，频繁调整原料配比 | 快速选料、实时看数据、一键提交 |
| **管理员（admin）**     | 审核配方、创建模板、管理原料   | 高效审核、模板标准化、全局管控 |

### 2.2 User Stories

#### US-1：快速进入录入模式（含准入规则）

> As a 配方师, I want to 在 Dashboard 页面一键展开快速录入面板, so that 无需跳转页面即可开始录入配方。

**Acceptance Criteria**:

- Dashboard 页面有明显的"快速录入"入口按钮
- 点击后**必须先填写"本次配方名称"**（必填字段），验证通过后才能进入快速配方编辑界面
- 进入编辑界面后，在页面顶部适当位置展示本次编辑的**配方名称**和**状态标签**（新建/草稿）
- 可随时收起录入面板回到 Dashboard

#### US-2：实时营养与成本仪表盘

> As a 配方师, I want to 在编辑配方时实时查看营养数据和成本信息, so that 我可以边调边看，无需提交后才知道结果。

**Acceptance Criteria**:

- 仪表盘展示：含量比(%)、能量(kJ)、蛋白质(g)、脂肪(g)、碳水(g)、钠(mg)
- 仪表盘展示：原料成本(元)、成本小计(元)、最终报价(元)
- 编辑区任何变更（原料增删、用量修改、系数调整）均在 100ms 内更新仪表盘
- 含量比超过 100% 时以红色警告标识
- 营养数据遵循 0 界限归零规则（能量≤17kJ 归零等）

#### US-3：配方编辑区

> As a 配方师, I want to 在左侧编辑区调整配方参数和原料, so that 我可以精确控制配方组成。

**Acceptance Criteria**:

- 可调整：主料系数(0.15-0.25)、辅料系数(0.5-1.5)、成品重量(g)
- 可对每个原料：修改用量、选择版本、删除
- 可设置：包材费用、其他费用、利润率(%)
- 原料列表按主料/辅料分组展示，带类型标签
- 原料用量修改后实时更新仪表盘营养数据和成本

#### US-4：动态原料池

> As a 配方师, I want to 在右侧原料池中浏览和选取原料, so that 我可以直观地发现和添加需要的原料。

**Acceptance Criteria**:

- 原料以名称+类型标签（主料/辅料）展示在原料池中
- 原料视觉上像鱼一样在池内动态游动（CSS/Canvas 动画）
- 鼠标 hover 时：原料停止游动，弹出信息浮层（版本、营养数据、成本）
- 浮层内有"添加"按钮，点击后原料从池中移除并出现在左侧编辑区
- 已添加到编辑区的原料在原料池中不再展示
- 原料池顶部有搜索框、类型筛选（全部/主料/辅料）、清除筛选

#### US-5：一键提交配方草稿（无弹窗）

> As a 配方师, I want to 编辑完成后一键提交配方为草稿, so that 配方进入正常审批流程，无需额外填写信息。

**Acceptance Criteria**:

- 点击"提交"按钮**直接提交**，不弹出任何对话框
- 系统自动使用准入阶段填写的"本次配方名称"作为配方名称
- "业务员"字段默认填充当前用户关联的业务员信息（快速创建场景下的默认值）
- 提交前校验：配方名称不为空、至少包含 1 个原料、成品重量 > 0、含量比校验通过
- 校验不通过时在页面内联展示错误提示（非弹窗）
- 提交后配方状态为 `draft`，与正常配方草稿一致
- formulist 提交后需走审批流程，admin 可直接发布

#### US-6：数据状态保持与草稿恢复

> As a 配方师, I want to 我的编辑数据在离开时被保存、下次进入时可以恢复, so that 我不会因为意外离开而丢失工作进度。

**Acceptance Criteria**:

- 用户尝试离开快速录入页面时，系统检测当前编辑状态
- 若存在未保存的修改，触发保存提醒，引导用户将当前内容保存为草稿
- 草稿数据保存在 `localStorage`，包含：配方名称、系数、成品重量、原料列表、费用参数
- 用户下次重新进入快速录入模式时，系统自动检测是否存在草稿数据
- 存在草稿时，提供"从草稿恢复"选项，用户可选择恢复或重新开始
- 恢复后编辑区恢复到上次编辑状态，配方名称和所有参数还原
- 成功提交配方后自动清除对应的草稿数据

#### US-7：配方模板

> As a 配方师, I want to 将常用配方保存为模板, so that 下次可以快速复用。

**Acceptance Criteria**:

- 编辑区有"保存为模板"按钮
- 保存时需填写模板名称（必填）和模板描述（选填）
- 模板保存：主料/辅料系数、成品重量、原料列表（含用量）、包材/其他费用、利润率
- 从模板创建时，加载模板数据到编辑区，用户可修改后提交
- 模板列表展示在录入面板中，支持搜索和删除
- admin 创建的模板全员可见，formulist 创建的模板仅自己可见

### 2.3 Non-Goals

- **不**做 AI 智能推荐原料
- **不**做配方对比功能（已有独立页面）
- **不**做批量导入配方
- **不**做原料池的拖拽排序（仅点击添加）
- **不**做配方的直接发布（必须走草稿→审批流程）

## 3. Technical Specifications

### 3.1 Architecture Overview

```
Dashboard.vue
├── QuickFormulaPanel.vue (新增 - 快速录入面板容器)
│   ├── QuickFormulaEntry.vue (新增 - 准入页面：填写配方名称)
│   ├── FormulaDashboard.vue (新增 - 实时仪表盘)
│   ├── FormulaWorkspace.vue (新增 - 工作区容器)
│   │   ├── FormulaEditor.vue (新增 - 配方编辑区)
│   │   └── MaterialPool.vue (新增 - 动态原料池)
│   │       └── MaterialFish.vue (新增 - 原料鱼动画组件)
│   └── TemplateManager.vue (新增 - 模板管理)
```

### 3.2 Integration Points

| 集成点                          | 说明                                    |
| ------------------------------- | --------------------------------------- |
| `useFormulaStore`               | 复用 `createFormula()` 创建配方         |
| `useMaterialStore`              | 复用 `fetchAllForSelect()` 获取原料列表 |
| `useAuthStore`                  | 获取用户角色和权限                      |
| `/api/formulas`                 | 创建配方 API                            |
| `/api/formulas/validate-ratio`  | 含量比实时校验                          |
| `/api/formula-templates` (新增) | 模板 CRUD API                           |
| `formulas` 表                   | 配方数据存储                            |
| `formula_templates` 表 (新增)   | 模板数据存储                            |

### 3.3 Security & Privacy

- 模板 API 需 `authMiddleware` 认证
- formulist 仅可操作自己创建的模板
- admin 可管理所有模板
- 配方创建遵循现有数据隔离规则

## 4. Risks & Roadmap

### 4.1 Phased Rollout

| 阶段     | 功能                                               | 优先级 |
| -------- | -------------------------------------------------- | ------ |
| **MVP**  | F1-F6：准入+仪表盘+编辑区+原料池+一键提交+状态保持 | P0     |
| **v1.1** | F7：配方模板                                       | P1     |
| **v1.2** | 原料池动画优化、移动端适配                         | P2     |

### 4.2 Technical Risks

| 风险                         | 影响       | 缓解措施                                               |
| ---------------------------- | ---------- | ------------------------------------------------------ |
| 原料池动画性能（大量原料时） | 卡顿       | 限制可视区域内渲染数量，使用 CSS transform 代替 Canvas |
| 实时计算频繁触发             | CPU 占用   | 使用 debounce 300ms                                    |
| 原料池与编辑区状态同步       | 数据不一致 | 统一 Store 管理                                        |
| 模板与配方版本耦合           | 模板过时   | 模板仅保存结构，加载时重新查最新原料数据               |
