## 1. Executive Summary

### 1.1 Problem Statement

配方编辑页面（FormulaForm.vue）当前存在**三个独立的原料相关表格**：

- **原料配比表**：核心编辑区，管理原料名称、用量、单位
- **配方报价表**：成本计算区，展示原料单价、金额、小计
- **解析结果原料列表**：AI 解析区，展示匹配状态、原始名称

三个表格结构相似但分散在页面不同区域，存在以下问题：

- **功能重复**：添加/删除/排序操作在三处重复实现
- **数据冗余**：同一原料的用量在多处显示，修改时需同步
- **维护困难**：修改一个功能需在三处代码中同步更新
- **用户困惑**：操作逻辑不连贯，上下文切换成本高

### 1.2 Proposed Solution

将三个表格**整合为一个统一的「原料管理表格」**，通过列的动态显示/隐藏来满足不同场景的需求：

- **编辑模式**：显示用量、单位、成本相关信息
- **AI 解析预览模式**：显示匹配状态、原始名称、可信度
- **报价模式**：在原有基础上增加成本相关列

### 1.3 Success Criteria

| KPI            | 目标              | 测量方式                      |
| -------------- | ----------------- | ----------------------------- |
| **代码复用率** | 减少 60% 重复代码 | 统计 `materials` 相关函数数量 |
| **操作效率**   | 减少 30% 操作步骤 | 用户测试对比                  |
| **代码行数**   | 减少 400+ 行      | 重构前后代码量对比            |
| **功能一致性** | 100% 操作行为一致 | 操作逻辑审查                  |
| **向后兼容**   | 0 个现有功能退化  | 回归测试通过率                |

## 2. User Experience & Functionality

### 2.1 User Personas

| 用户角色       | 角色描述                   | 核心需求                           |
| -------------- | -------------------------- | ---------------------------------- |
| **配方管理员** | 创建和管理配方的核心用户   | 快速添加/编辑原料、实时查看成本    |
| **AI 操作员**  | 通过智能填单导入配方的用户 | 清晰看到匹配状态、快速修正未匹配项 |
| **业务员**     | 查看配方报价的用户         | 一目了然的成本构成和总价           |
| **审核员**     | 审核配方合规性的用户       | 快速定位异常数据和历史变更         |

### 2.2 User Stories

#### Story 1: 手动录入原料

> **As a** 配方管理员
> **I want to** 在统一的原料表格中一次性添加所有原料，并直接填写用量
> **So that** 不需要在多个表格之间切换

**Acceptance Criteria:**

- [ ] 表格内可直接编辑原料名称（自动联想搜索）
- [ ] 表格内可直接编辑用量（数字输入，自动保留2位小数）
- [ ] 添加行按钮在表格底部，点击即新增一行
- [ ] 删除行按钮在每行末尾，悬停时显示
- [ ] 上移/下移按钮可调整原料顺序

#### Story 2: AI 解析结果预览与回填

> **As a** AI 操作员
> **I want to** 在同一表格中看到 AI 解析的原料列表及匹配状态
> **So that** 一目了然地识别未匹配原料并快速修正

**Acceptance Criteria:**

- [ ] AI 解析后，结果直接展示在原料表格中（而非独立区域）
- [ ] 未匹配原料高亮显示，右侧显示"修正"按钮
- [ ] 点击修正按钮弹出原料选择器进行匹配
- [ ] 已匹配/未匹配状态用颜色区分（绿色=已匹配，黄色=未匹配）
- [ ] 确认回填后，表格自动切换为编辑模式

#### Story 3: 成本实时计算与单价调整

> **As a** 配方管理员
> **I want to** 在原料表格中直接看到每种原料的单价，并可灵活调整
> **So that** 实时掌握配方成本结构，支持灵活定价并可随时还原

**Acceptance Criteria:**

- [ ] 原料表格中显示"单价"列（从原料数据库带出，作为默认值）
- [ ] 原料表格中显示"金额"列（用量 × 单价，自动计算）
- [ ] 单价单元格可点击编辑，修改后显示"调"标记（橙色标签）
- [ ] 单价调整后，单元格背景变为淡橙色，表示已被修改
- [ ] 每行单价单元格右侧显示"撤销"按钮（返回箭头图标），点击可还原为数据库原价
- [ ] 撤销按钮仅在单价被调整后显示（原价时不显示）
- [ ] 表格底部显示"原料小计"汇总行
- [ ] 报价相关列默认展开，可折叠隐藏
- [ ] 总成本实时汇总显示在表格下方
- [ ] 支持批量调整单价（选中多行后统一设置）

**单价状态示意：**

```
┌─────────────────────────────────────────────────────────────┐
│  原料名称      │  用量  │  单位  │  单价（元/kg） │ 金额（元） │
├─────────────────────────────────────────────────────────────┤
│  原料A        │  100   │   g    │  120.00      │  12.00    │
│  原料B        │  50    │   g    │  80.00 [调]↩│   4.00    │  ← 已调整，显示"调"标签和撤销按钮
│  原料C        │  200   │   g    │  200.00      │  40.00    │
├─────────────────────────────────────────────────────────────┤
│  原料小计: 56.00 元                                              │
└─────────────────────────────────────────────────────────────┘
```

**交互细节：**

- 单价修改：点击单元格 → 进入编辑状态 → 输入新值 → 回车确认
- 修改确认后：自动显示"调"橙色标签 + 撤销按钮
- 撤销操作：点击撤销按钮 → 恢复原价 → 标签和按钮消失
- 撤销支持 `Ctrl+Z` 快捷键（全局撤销）

#### Story 4: 多原料批量操作

> **As a** 配方管理员
> **I want to** 批量选择多个原料后统一操作（删除/调整用量）
> **So that** 提高大量原料时的操作效率

**Acceptance Criteria:**

- [ ] 每行左侧有复选框，支持多选
- [ ] 批量选择后，工具栏出现批量操作按钮
- [ ] 支持"批量删除"、"批量上移"、"批量设置单位"
- [ ] 已选数量在工具栏实时显示

### 2.3 Non-Goals

以下功能**不在本次整合范围内**：

- ❌ 配方报价表的完整报价单模板导出（保留现有导出功能，不扩展）
- ❌ AI 解析的完整流程改造（仅整合结果展示，不改变解析逻辑）
- ❌ 原料数据库管理（原料的增删改查独立于配方模块）
- ❌ 配方版本对比功能（属于配方历史模块）

### 3.3 Component Design

#### 核心组件: `UnifiedMaterialTable.vue`

```
frontend/src/components/formula/
└── UnifiedMaterialTable.vue    # 统一的原料管理表格组件
```

**Props:**

| Prop         | 类型                                | 必填 | 说明                         |
| ------------ | ----------------------------------- | ---- | ---------------------------- |
| `mode`       | `'edit' \| 'ai_preview' \| 'quote'` | 是   | 表格显示模式                 |
| `materials`  | `MaterialItem[]`                    | 是   | 原料列表数据                 |
| `showCost`   | `boolean`                           | 否   | 是否显示成本列（默认 false） |
| `editable`   | `boolean`                           | 否   | 是否可编辑（默认 true）      |
| `selectable` | `boolean`                           | 否   | 是否支持多选（默认 false）   |

**Events:**

| Event              | 参数                                    | 说明           |
| ------------------ | --------------------------------------- | -------------- |
| `update:materials` | `MaterialItem[]`                        | 数据变更时触发 |
| `add`              | `void`                                  | 添加新行       |
| `remove`           | `index: number`                         | 删除指定行     |
| `reorder`          | `{ from: number; to: number }`          | 行顺序变更     |
| `match`            | `{ index: number; materialId: string }` | 原料匹配修正   |
| `batch-action`     | `{ action: string; indices: number[] }` | 批量操作       |

**Emits 数据结构:**

```typescript
interface MaterialItem {
  id?: string; // 行唯一ID（用于React key和操作）
  materialId?: string; // 匹配的系统原料ID
  materialName: string; // 原料名称（显示用）
  quantity: number; // 用量
  unit: string; // 单位
  ratio?: number; // 含量比（用量 / 总用量 × 100%，自动计算）
  // 成本相关
  unitPrice?: number; // 当前单价（元/kg，可手动调整）
  originalUnitPrice?: number; // 数据库原价（用于撤销还原）
  amount?: number; // 金额（用量 × unitPrice，自动计算）
  isPriceAdjusted?: boolean; // 单价是否已被调整（用于判断是否显示"调"标记）
  // AI 相关
  originalName?: string; // AI 解析原始名称
  matched?: boolean; // 是否已匹配
  confidence?: number; // 匹配可信度
  // 报价相关
  costRatio?: number; // 成本占比
}
```

### 3.4 表格列配置

**设计原则：所有列默认全部显示，用量单位固定为 g（克），无需单位列。**

| 列名          | 宽度  | 可排序 | 默认显示 | 适用模式   | 说明                         |
| ------------- | ----- | ------ | -------- | ---------- | ---------------------------- |
| 序号          | 50px  | ❌     | ✅       | 全部       |                              |
| 复选框        | 40px  | ❌     | 多选时   | 全部       |                              |
| 原料名称      | 200px | ✅     | ✅       | 全部       | 支持点击编辑                 |
| 用量(g)       | 120px | ❌     | ✅       | 全部       | 支持点击编辑，单位固定为 g   |
| 含量比（%）   | 100px | ❌     | ✅       | 全部       | 实时计算：用量/总用量×100%   |
| 单价（元/kg） | 100px | ❌     | ✅       | 全部       | 可编辑，修改后显示"调"标签   |
| 金额（元）    | 100px | ❌     | ✅       | 全部       | 自动计算（用量 × 单价）      |
| 小计（元）    | 100px | ❌     | ✅       | 全部       | 表格底部汇总行               |
| 原始名称      | 150px | ❌     | AI预览时 | ai_preview | AI 解析原始名称              |
| 匹配状态      | 80px  | ❌     | AI预览时 | ai_preview | 绿=已匹配，黄=未匹配         |
| 操作          | 100px | ❌     | ✅       | 全部       | 含撤销按钮（单价调整后显示） |

**标准列顺序：**

```
序号 → 复选框 → 原料名称 → 用量(g) → 含量比(%) → 单价 → 金额 → 操作
                                                           ↑
                              AI预览时：原始名称/匹配状态 插入在金额后
```

**用量单位说明：**

- 所有原料用量单位统一固定为 **g（克）**
- 无需单位列选择器，简化操作
- 若未来需要支持多单位，可扩展

**含量比列实时计算规则：**

- 含量比 = 该原料用量 / 所有原料总用量 × 100%
- 保留 2 位小数（如 `28.57%`）
- 总用量为 0 时显示 `--`
- 当用户修改任意原料用量时，所有行的含量比自动重新计算

```vue
// AI 解析完成时，切换模式
watch(
  () => aiStore.parseResult,
  result => {
    if (result) {
```

#### AI 解析集成

AI 解析结果直接写入 `formData.materials`，无需独立的解析结果区域：

```typescript
// AI 解析完成 → 数据直接进入统一表格
const handleParseComplete = (result: ParseResult) => {
  formData.materials = result.materials.map((m, i) => ({
    id: `ai-row-${i}-${Date.now()}`,
    materialName: m.matched ? m.name : m.originalName,
    materialId: m.materialId,
    quantity: m.quantity,
    unit: m.unit || "g",
    matched: m.matched,
    originalName: m.originalName,
    confidence: m.confidence,
  }));
  currentTableMode.value = "ai_preview"; // 自动切换到预览模式
};
```

## 3.8 配方编辑页面整体布局优化

### 3.8.1 优化背景

当前配方编辑页面存在以下问题：

1. **结构分散**：三个独立表格分布在页面不同位置，用户需要在多个区域之间来回切换
2. **操作断裂**：添加原料在配比表，查看成本在报价表，修正未匹配在解析结果区
3. **信息割裂**：同一原料的数据分散在多个表格，修改时需同步更新多处
4. **响应式差**：大屏幕时三表格横向排列造成视觉混乱，小屏幕时堆叠导致页面过长

### 3.8.2 视觉层次重构

| 区域       | 优化前                  | 优化后                       | 优化效果                     |
| ---------- | ----------------------- | ---------------------------- | ---------------------------- |
| **标题区** | 分散在各表格上方        | 统一为页面级标题 + 功能描述  | 减少视觉噪音，明确页面焦点   |
| **操作区** | 各表格独立工具栏（3处） | 统一顶部工具栏（1处）        | 降低操作复杂度，减少重复按钮 |
| **内容区** | 三表格垂直堆叠          | 单表格 + 动态列              | 节省 40% 垂直空间            |
| **校验区** | 独立卡片悬浮在表格下方  | 整合到表格下方与汇总信息并列 | 与数据紧密关联，减少页面跳转 |
| **提交区** | 分散在页面底部          | 固定在底部，响应式位置       | 提高可访问性                 |

```
步骤 2: 切换到配方报价表查看成本
        ↓
步骤 3: 发现成本异常，返回原料配比表修改用量
        ↓
步骤 4: 再次切换到报价表确认
        ↓
步骤 5: 前往含量比校验卡片查看校验结果
        ↓
步骤 6: 发现校验不通过，修改原料
        ↓
完成：需要 5+ 次区域切换
```

#### 新流程（统一操作）

```
步骤 1: 在统一表格添加原料
        ↓
步骤 2: 点击"显示成本"按钮，实时查看成本
        ↓
步骤 3: 直接在当前行修改用量/单价，成本实时更新
        ↓
步骤 4: 表格下方实时显示含量比校验结果
        ↓
完成：所有操作在同一区域完成，0 次切换
```

## 5. Risks & Mitigation

| 风险                    | 等级 | 缓解策略                                             |
