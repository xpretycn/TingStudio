## 简介

营养分析控制器是TingStudio系统中的核心模块，负责处理营养成分计算、标准对照和合规性检查功能。该控制器实现了完整的营养数据分析流程，包括原料营养数据管理、配方营养汇总计算、营养标准配置和合规性验证等功能。

系统采用前后端分离架构，后端使用Node.js + Express框架，数据库采用SQLite，前端使用Vue.js构建用户界面。营养分析控制器通过RESTful API提供服务，支持多种营养分析场景和标准配置。

### 配方营养计算引擎

#### 计算算法实现

配方营养计算采用加权平均算法，基于原料的用量和营养密度：

```mermaid
flowchart TD
Start([开始计算]) --> LoadFormula["加载配方数据"]
LoadFormula --> InitVars["初始化变量<br/>totalWeight=0<br/>totalNutrition={}"]
InitVars --> LoopMaterials["遍历每个原料"]
LoopMaterials --> GetNutrition["获取原料营养数据"]
GetNutrition --> CheckAlt["检查备选查找"]
CheckAlt --> ParseData["解析并标准化营养数据"]
ParseData --> CalcContribution["计算营养贡献<br/>contribution = per100g × (quantity/100)"]
CalcContribution --> UpdateTotal["更新总营养值"]
UpdateTotal --> UpdateWeight["更新总重量"]
UpdateWeight --> AddBreakdown["添加到明细"]
AddBreakdown --> NextMaterial{"还有原料?"}
NextMaterial --> |是| LoopMaterials
NextMaterial --> |否| CalcPercentage["计算占比"]
CalcPercentage --> CalcPer100g["计算每100g营养值"]
CalcPer100g --> SaveSummary["保存计算结果"]
SaveSummary --> End([结束])
```

#### 关键计算公式

1. **营养贡献计算**:
   ```
   contribution[field] = per100g[field] × (quantity / 100)
   ```

2. **总营养值累积**:
   ```
   totalNutrition[field] += contribution[field]
   ```

3. **配方占比计算**:
   ```
   percentage = (weightContribution / totalWeight) × 100
   ```

4. **每100g营养值计算**:
   ```
   per100g[field] = (totalNutrition[field] / totalWeight) × 100
   ```

## 性能考虑

### 数据库优化策略

1. **索引优化**
   - `material_nutrition(material_id)` - 主要查询条件
   - `formula_nutrition_summaries(formula_id)` - 汇总查询
   - `nutrition_profiles(category)` - 标准查询
   - `nutrition_analysis_reports(formula_id)` - 报告查询

2. **批量查询优化**
   - 原料营养数据批量获取，减少数据库往返
   - 含量比系数批量查询，避免逐条查询

3. **缓存策略**
   - 营养标准数据可缓存到内存中
   - 常用查询结果可考虑短期缓存

### 计算性能优化

1. **算法复杂度**
   - 配方计算：O(n)，n为原料数量
   - 合规性检查：O(m×n)，m为标准字段数，n为配方字段数

2. **内存使用优化**
   - 使用流式处理避免大对象创建
   - 及时释放临时变量

3. **并发处理**
   - 支持多配方并行计算
   - 数据库连接池管理
