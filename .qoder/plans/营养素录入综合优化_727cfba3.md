## Task 1：移除折叠，改为全展开平铺分组

**现状**：使用 `t-collapse` + `t-collapse-panel` 实现 4 组折叠面板，用户需手动展开每组。

**目标**：去掉折叠交互，4 个分组（基础营养/矿物质/维生素/其他）同时平铺展示，用分组标题+分割线区分。

**具体改动**：
- 删除 `t-collapse` / `t-collapse-panel` 包裹
- 删除 `collapseExpanded` reactive、`expandAllGroups`/`collapseAllGroups`/`handleCollapseChange` 函数
- 删除"展开""收起"两个按钮
- 每组渲染为 `<div class="nutrition-group-section">` + 组头 `<div class="ngroup-header">`（图标+名称+X项）
- 样式：组与组之间用 `border-bottom: 1px solid var(--color-border-light)` 分隔，最后一组无底线

## Task 2：重构字段布局为紧凑表格行模式

**现状**：`nutrition-grid` 用 2 列 grid 布局，每字段是 `label` + `input` 纵向堆叠。

**目标**：改为横向紧凑行，每个字段占一行：`[字段名 90px] [输入框 110px] [单位 40px]`，支持 Tab 键依次跳转。

**具体改动**：

模板改为：
```html
<div class="nutrition-table-group">
  <div class="ntg-header">
    <t-icon :name="group.icon" size="16" />
    <span>{{ group.title }}</span>
    <t-tag size="small" variant="light" shape="round">{{ group.fields.length }}项</t-tag>
  </div>
  <div class="ntg-rows">
    <div v-for="field in visibleFields(group)" :key="field.key" class="ntg-row" :class="{ 'is-zero': isFieldZero(field.key) }">
      <span class="ntg-label">{{ field.label }}</span>
      <t-input-number v-model="nutritionData[field.key]" :min="0"
        :decimal-places="field.decimals" theme="normal"
        :placeholder="field.placeholder"
        class="ntg-input"
        @keydown.tab="handleTab" />
      <span class="ntg-unit">{{ field.unit }}</span>
      <!-- 能量行特殊处理：显示计算值，不可手动输入 -->
    </div>
  </div>
</div>
```

样式：
```scss

  font-size: 13px;

  width: 40px;
  flex-shrink: 0;
  font-size: 12px;
```

## Task 3：进度指示器 + 智能零值过滤

**目标**：
- 顶部显示进度条 + 文字："已填 X/26 项"
- 增加"隐藏零值"开关，录入后自动隐藏值为 0 的字段（视觉降噪）

**具体改动**：

新增状态：
```typescript
const hideZeroValues = ref(false)

const filledCount = computed(() =>
  nutritionFields.filter(f => f.key !== 'energy' && nutritionData[f.key] > 0).length
)
const totalCount = computed(() =>
  nutritionFields.filter(f => f.key !== 'energy').length
)

function visibleFields(group: NutritionGroup) {
  const fields = group.fields.filter(f => f.key !== 'energy')
  if (!hideZeroValues.value) return fields
  return fields.filter(f => nutritionData[f.key] > 0)
}

function isFieldZero(key: string) {
  return !nutritionData[key] || nutritionData[key] === 0
}
```

进度条模板（放在折叠工具栏位置）：
```html
<div class="nutrition-progress">
  <div class="np-bar-wrap">
    <div class="np-bar" :style="{ width: progressPercent + '%' }"></div>
  </div>
  <span class="np-text">已填 {{ filledCount }}/{{ totalCount }} 项</span>
  <label class="np-toggle-label">
    <t-switch v-model="hideZeroValues" size="small" />
    <span>隐藏零值</span>
  </label>
</div>
```

## Task 4：种子库自动提示气泡

**目标**：用户在"原料名称"输入框填写完毕后（debounce 600ms），自动调用种子库 API 检查是否存在匹配数据；若有，在营养区顶部浮现一条提示气泡。

**具体改动**：

新增状态与 watcher：
```typescript
const seedAvailable = ref(false)
const seedMatchScore = ref(0)
let seedCheckTimer: ReturnType<typeof setTimeout> | null = null

watch(() => formData.name, (newName) => {
  clearTimeout(seedCheckTimer)
  seedAvailable.value = false
  if (!newName || newName.length < 2) return
  seedCheckTimer = setTimeout(async () => {
    try {
      const res = await nutritionApi.checkSeedAvailability(newName)
      if (res?.found) {
        seedAvailable.value = true
        seedMatchScore.value = res.matchScore ?? 0
      }
    } catch { /* 忽略 */ }
  }, 600)
})
```

> 注意：`nutritionApi.checkSeedAvailability` 需确认后端是否已有此轻量接口；若无，需新增后端接口 `GET /api/nutrition/check-seed?name=xxx`，只返回 `{ found: boolean, matchScore: number }`，不返回完整营养数据。

气泡模板（在 `nutrition-toolbar` 上方）：
```html
<Transition name="fade-slide">
  <div v-if="seedAvailable && !hasNutrition" class="seed-hint-banner">
    <t-icon name="root-list" />
    <span>检测到种子库有该原料的营养数据（匹配度 {{ seedMatchScore }}%）</span>
    <t-button theme="primary" variant="text" size="small" @click="seedEnrichVisible = true">
      一键填充
    </t-button>
    <t-icon name="close" class="seed-hint-close" @click="seedAvailable = false" />
  </div>
</Transition>
```

样式：横幅条，浅绿色背景，左边图标，右侧关闭按钮。

## Task 5：后端新增轻量种子检测接口（若不存在）

文件：`backend/src/controllers/nutritionController.ts` + `backend/src/routes/nutrition.ts`

新增接口 `GET /api/nutrition/check-seed?name=xxx`：
- 只做模糊名称匹配，返回 `{ found: boolean, matchScore: number }`
- 不返回完整 per100g 数据，响应极轻量
- 用于前端实时检测是否有种子数据可填充
