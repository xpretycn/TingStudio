# 需求拆解

按企业研发规范拆解需求，输出结构化、可执行、可验收的任务清单。

## 输出内容

1. **页面/模块清单** — 列出所有涉及的页面和模块
2. **组件划分** — 每个页面拆分为哪些组件，层级关系
3. **接口与数据结构** — 需要的 API 接口和数据模型
4. **开发顺序** — 按依赖关系排列开发优先级

## 约束

- 只输出可执行、可验收的结构化任务
- 不输出废话和泛泛而谈的建议
- 每个任务有明确的验收标准
