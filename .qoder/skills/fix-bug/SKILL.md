# 修复 Bug

系统化定位并修复问题，确保不引入新问题。

## 执行步骤

1. **定位根因** — 分析报错信息和相关代码，找到真正的原因
2. **给出修复代码** — 提供可直接替换的完整修复代码
3. **解释修改原因** — 说明为什么要这样改
4. **回归验证** — 确认修复不影响已有功能

## 约束

- 只改根因，不连带改无关代码
- 修复后说明可能的影响范围
- 如涉及多模块，按依赖顺序逐个修复
