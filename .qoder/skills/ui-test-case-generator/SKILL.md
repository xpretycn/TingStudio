# UI 测试用例生成器

扫描前端页面的所有可操作区域，生成结构化测试用例文档，覆盖正向流程、异常流程、边界条件、UI 变化和联动功能。

## 触发词

生成测试用例、测试用例文档、页面测试覆盖、测试覆盖

## 执行流程

```
确定目标页面 → 扫描交互元素 → 分析业务逻辑 → 生成测试用例 → 输出文档
```

### 阶段 1：确定目标页面

#### 用户指定了页面

直接使用指定页面，支持以下格式：
- 文件路径：`frontend/src/views/formula/FormulaForm.vue`
- 页面名称：`配方编辑`、`FormulaForm`
- 相对路径：`formula/FormulaForm.vue`

#### 用户未指定页面

采用**分层扫描**策略：索引层秒级列出清单，深度层按需扫描内容。

**索引层（自动，<1s）**：

1. 扫描 `frontend/src/views/` 和 `frontend/src/components/` 的文件名和目录结构
2. 对每个 `.vue` 文件，快速统计 `<template>` 中 `t-` 前缀组件的数量（不读取业务逻辑）
3. 按目录分组，输出选择清单：

```
扫描到 23 个页面（仅统计交互组件数量，未深度分析）：

【配方管理】3 个页面
  1. FormulaList.vue    — 8 个交互组件（表格/按钮/搜索）
  2. FormulaForm.vue    — 15 个交互组件（表单/上传/提交）
  3. FormulaVersion.vue — 6 个交互组件（标签页/对比）

【原料管理】2 个页面
  4. MaterialList.vue   — 7 个交互组件
  5. MaterialForm.vue   — 12 个交互组件

【AI 助手】1 个页面
  6. AgentChat.vue      — 10 个交互组件

...

建议：交互组件 ≥10 的页面优先生成测试用例。
请输入编号（多选用逗号，all 全部，推荐 2,5）：
```

4. 用户选择模式：

| 输入 | 行为 |
|------|------|
| 单个编号 | 只生成该页面 |
| 多个编号（逗号分隔） | 依次生成，每个页面独立文档 |
| `all` | 批量生成所有页面，每个页面独立文档 |
| `页面级` / `组件级` | 按分类批量生成 |
| 页面名称关键词 | 模糊匹配，列出匹配项再确认 |

**深度层（选定后触发，2-5s/页）**：

用户选定页面后，才读取 Vue + Store + API 完整内容，提取交互元素和业务逻辑。

**批量模式优化**：

| 策略 | 说明 |
|------|------|
| 并行扫描 | 多个页面用 Task agent 并行深度扫描 |
| 增量缓存 | 已生成过的页面记录文件 hash，未变更则跳过，提示"无变更，跳过" |
| 优先级排序 | 交互组件 ≥10 的页面先生成（覆盖面大） |
| 进度提示 | "正在生成 3/23：FormulaForm.vue..." |

5. 上下文推断（智能模式）：如果对话上下文有明确线索，自动推断并确认：
   - 刚修改了某 .vue 文件 → 推断为该页面
   - 讨论了某个业务功能 → 推断为相关页面
   - 推断后仍需确认："检测到您最近在编辑 XX 页面，是否为该页面生成测试用例？"

### 阶段 2：扫描交互元素

读取目标 `.vue` 文件，识别所有 TDesign 交互组件：

| 组件类型 | 匹配规则 | 可操作行为 |
|---------|---------|-----------|
| 按钮 | `t-button` | 点击、禁用态、loading 态 |
| 输入框 | `t-input`, `t-textarea` | 输入、清空、失焦、最大长度 |
| 下拉框 | `t-select` | 选择、搜索、清空、多选 |
| 单选 | `t-radio-group` | 切换选项 |
| 多选 | `t-checkbox-group` | 勾选/取消、全选 |
| 日期 | `t-date-picker` | 选日期、清空、范围选择 |
| 上传 | `t-upload` | 上传、删除、格式校验、大小限制 |
| 展开/折叠 | `t-collapse` | 展开、收起 |
| 弹窗确认 | `t-popconfirm` | 确认、取消 |
| 开关 | `t-switch` | 开启/关闭 |
| 标签页 | `t-tabs` | 切换 tab |
| 表格操作 | `t-table` + 操作列 | 行内按钮、排序、筛选 |
| 分页 | `t-pagination` | 翻页、跳转、改每页条数 |
| 对话框 | `t-dialog` | 打开、关闭、确认（仅遗留场景） |
| 消息提示 | `t-message` | 触发条件、显示内容 |

对每个交互元素提取：

```
元素编号 | 元素名称 | 组件类型 | 所在区域 | 绑定事件 | 关联状态 | 前置条件 | 依赖元素
```

同时扫描关联文件：
- 同目录的 `composables/`、`components/` 子组件
- 对应的 Pinia Store（提取业务逻辑和状态变更）
- 对应的 API 调用（提取接口参数和响应）

### 阶段 3：分析业务逻辑

从 Store 和 API 层提取：
- 状态流转（如 draft → pending_review → published）
- 字段联动（选择 A 后 B 的选项变化）
- 校验规则（必填、格式、范围、正则）
- 权限控制（不同角色看到不同按钮/操作）
- 数据转换（提交前格式化、展示时解析）
- 计算逻辑（营养成分计算、利润率、成品重量等前端计算公式）
- 并发场景（多人同时编辑同一数据）
- 路由依赖（页面是否依赖 URL 参数、刷新后状态恢复）

### 阶段 4：生成测试用例

对每个交互元素，按 7 个维度生成用例：

| 维度 | 用例类型 | 说明 |
|------|---------|------|
| **正向流程** | 正常操作 | 标准用户操作路径，验证功能正确性 |
| **异常流程** | 操作失败 | 网络错误、接口报错、权限不足、并发冲突 |
| **边界条件** | 极值/临界 | 空值、最大长度、0、负数、特殊字符、SQL注入 |
| **UI 变化** | 视觉反馈 | loading 态、禁用态、错误高亮、成功提示、动画 |
| **联动功能** | 级联影响 | 操作 A 后 B 的变化、状态同步、数据刷新 |
| **权限角色** | 角色差异 | 不同角色下的操作可见性、可执行性、数据隔离 |
| **状态流转** | 状态约束 | 不同数据状态下操作的允许/禁止、状态回退 |

#### 特殊场景用例（按条件生成）

以下场景不是每个元素都有，仅在检测到相关代码时生成：

| 场景 | 前缀 | 触发条件 | 用例内容 |
|------|------|---------|---------|
| **计算准确性** | X-C | 检测到前端计算逻辑（营养成分、利润率、成品重量） | 验证计算结果与预期值一致，含 ratioFactor=0.18 等特殊系数 |
| **并发冲突** | X-CC | 检测到编辑/更新操作 | 两人同时编辑同一数据，验证是否数据覆盖或提示冲突 |
| **浏览器路由** | X-R | 检测到 URL 参数依赖或表单编辑页 | 刷新后数据保持、后退不丢失、直接访问 URL 状态正确 |
| **搜索筛选组合** | X-F | 检测到列表页含搜索+排序+分页 | 多条件组合筛选结果正确、翻页后筛选条件保持、清空筛选重置 |
| **版本化** | X-V | 检测到 version/previous_version_id/is_latest 字段 | 版本对比差异高亮、版本回退 is_latest 指向、多版本共存查询、版本链断裂 |
| **AI 功能** | X-AI | 检测到 AI 解析/Agent 聊天相关代码 | 解析结果关联转化、AI 降级/熔断切换、多轮对话上下文保持、AI 响应超时反馈 |
| **导出下载** | X-EX | 检测到导出/下载/分享相关代码 | 导出文件内容正确性、大数据量导出、模板切换、分享链接过期、失败重试 |
| **文件解析** | X-FP | 检测到文件上传+AI 解析相关代码 | 相同文件缓存命中、解析失败降级、解析结果关联完整性、文件格式校验 |
| **主题切换** | X-T | 检测到品牌色/主题切换相关代码 | 品牌色切换后组件颜色跟随、亮暗模式对比度、主题持久化、组件主题适配 |
| **登录会话** | X-AU | 检测到登录/Token/权限校验相关代码 | Token 过期跳转、并发登录互踢、权限变更即时生效 |
| **异步任务** | X-AS | 检测到异步导出/AI 解析/长对话 | 任务进度追踪、任务取消清理、任务超时终止、完成后通知 |
| **数据一致性** | X-DC | 检测到多表联动操作 | 多表原子性（部分失败回滚）、删除级联处理、引用完整性检查、缓存与数据库一致 |

每个用例包含：

| 字段 | 说明 |
|------|------|
| 用例ID | 元素编号 + 维度缩写 + 序号（如 E01-P01、E01-E01、E01-B01） |
| 用例名称 | 简明描述测试场景 |
| 前置条件 | 执行前必须满足的状态 |
| 操作步骤 | 编号步骤列表 |
| 预期结果 | 编号预期，与步骤对应 |
| 实际结果 | 空白，供测试执行时填写 |

## 注意事项

- **TDesign 组件优先**：项目使用 TDesign 组件库，扫描时以 `t-` 前缀组件为主
- **popconfirm 替代 dialog**：二次确认操作统一使用 `t-popconfirm`，不再使用 `t-dialog`（遵循 AGENTS.md 约定）
- **时间展示**：验证时间字段是否正确展示为本地时区格式，不应出现 ISO 8601 UTC 格式（遵循 AGENTS.md 约定）
- **品牌色切换**：涉及颜色/主题的 UI 变化用例，需验证在不同品牌色下的表现（遵循 DESIGN.md）
- **批量生成**：选择 `all` 时，每个页面独立生成文档，不合并
- **增量更新**：如果目标文档已存在，保留手动修改的用例，只更新新增/变更的用例
- **文档ID 透传**：文档ID 是下游 test-case-runner 和 bug-fixer 的唯一追溯标识，不可省略
- **文件hash 校验**：下游技能启动时通过对比 hash 判断用例文档是否变更，hash 不一致时警告
