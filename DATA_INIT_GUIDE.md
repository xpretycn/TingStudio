# TingStudio 示例数据初始化使用说明

本工具用于为TingStudio应用生成示例测试数据。

## 功能特点

- 生成20个测试用户
- 每个用户包含20条客户数据
- 每个用户包含20条原料数据  
- 每个用户包含20条配方数据
- 数据关联关系完整正确
- 支持清除所有现有数据

## 使用方法

### 方法一: 在浏览器控制台中初始化(推荐)

1. 启动应用: `npm run dev`
2. 在浏览器中打开应用
3. 按F12打开浏览器开发者工具
4. 切换到"Console"(控制台)标签
5. 输入以下命令并回车:

```javascript
initTingStudioData()
```

6. 等待数据生成完成,会看到类似输出:
```

- 用户: 20 条
```

### 方法二: 使用命令行脚本

运行以下命令:
```bash
npm run init:sample-data
```

注意: 此方法主要用于Node.js环境,不会直接修改浏览器的LocalStorage。

## 其他控制台命令

### 验证数据完整性
```javascript
validateTingStudioData()
```

### 清除所有数据
```javascript
clearTingStudioData()
```

## 自定义数据量

如果需要生成不同数量的数据,可以修改`src/utils/initData.ts`中的参数:

```javascript
// 格式: initSampleData(userCount, customerPerUser, materialPerUser, formulaPerUser)
initSampleData(10, 15, 20, 25) // 10个用户,每个用户15个客户、20个原料、25个配方
```

## 数据说明

### 用户数据
- 用户名格式: user001 ~ user020
- 每个用户都有独立的客户、原料和配方数据

### 客户数据
- 包含中文姓名
- 包含联系人、电话、邮箱、地址
- 数据真实感强

### 原料数据
- 编码格式: MTL{userId}-{编号}
- 包含名称、编码、单位、库存
- 单位包括: kg, g, ml, L, 个, 包, 瓶, 盒

### 配方数据
- 关联该用户的客户和原料
- 每个配方包含2-5种原料
- 包含配方描述
- 类型包括: 保湿型、滋养型、修复型等

## 注意事项

1. ⚠️ **数据会覆盖**: 初始化会清除所有现有数据,请谨慎使用
2. **数据隔离**: 每个用户只能看到自己创建的数据
3. **登录测试**: 可以使用生成的任意用户名登录,密码为该用户名(例如user001的密码是user001)
4. **性能考虑**: 生成大量数据可能需要几秒钟时间

## 常见问题

### Q: 初始化后登录时密码是什么?
A: 密码与用户名相同,例如用户名为user001,密码就是user001

### Q: 如何恢复默认状态?
A: 运行 `clearTingStudioData()` 清除所有数据,然后刷新页面

### Q: 数据存储在哪里?
A: 数据存储在浏览器的LocalStorage中,清除浏览器数据会同时清除应用数据

### Q: 可以在不同用户间共享数据吗?
A: 不可以,每个用户的数据是隔离的,这符合实际应用场景

## 技术实现

- **数据生成**: `src/utils/mockData.ts` - 提供各种随机数据生成函数
- **浏览器初始化**: `src/utils/initData.ts` - 浏览器环境数据初始化
- **命令行脚本**: `scripts/initSampleData.ts` - Node.js环境数据初始化
- **全局命令**: 在`main.ts`中导入初始化模块,自动注册控制台命令
