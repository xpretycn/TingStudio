# 临时文件管理规则

## 1. 核心规则

- 所有临时测试代码文件**必须**放在项目根目录下的 `test/` 文件夹中统一管理。
- 所有截图文件**必须**放在项目根目录下的 `test/screenshots/` 文件夹中统一管理。
- **禁止**在项目根目录或其他非指定目录下散落临时测试文件和截图文件。

## 2. 目录结构

```
项目根目录/
├── test/                  ← 所有临时文件放这里
│   ├── screenshots/       ← 所有截图文件放这里
│   │   ├── *.png
│   │   ├── *.jpg
│   │   └── *.webp
│   ├── *.test.js
│   ├── *.test.ts
│   ├── *.spec.js
│   ├── *.spec.ts
│   └── ...其他临时测试脚本
├── src/
└── ...
```

## 3. 适用范围

- 临时验证代码片段的测试文件
- 快速原型验证脚本
- 调试用的临时脚本
- 一次性功能测试脚本
- Playwright/浏览器自动化截图
- UI 调试截图
- 错误现场截图
- 任何临时产生的图片文件

## 4. 命名规范

### 测试文件
- 测试文件命名格式：`*.test.{js,ts,py}` 或 `*.spec.{js,ts,py}`
- 调试脚本命名格式：`debug-*.{js,ts,py}`
- 临时验证脚本命名格式：`try-*.{js,ts,py}`

### 截图文件
- 截图命名格式：`{描述}-{日期}.png`，例如：`homepage-20260514.png`
- 调试截图格式：`debug-{描述}.png`，例如：`debug-login-form.png`
- 错误截图格式：`error-{描述}.png`，例如：`error-modal-overflow.png`

## 5. 禁止行为

- 禁止在 `src/` 目录下创建临时测试文件
- 禁止在项目根目录直接放置测试脚本（如 `test-xxx.js`）
- 禁止在组件/模块同级目录放置临时测试文件
- 禁止在项目根目录直接放置截图文件（如 `screenshot.png`）
- 禁止在 `src/` 或其他业务目录下放置截图文件
