# 快速录入配方 — "保存配方"按钮 测试用例集

> **测试对象**：`QuickFormulaPanel.vue` → `handleSave()` → `quickFormulaListStore.saveQuickFormula()` → `PUT /api/quick-formulas/:id`
>
> **测试环境**：Vitest + Playwright E2E
>
> **版本**：v2.0.0 | 编写日期：2026-06-05

---

## 一、功能概述

### 保存链路

```
用户点击"保存"按钮
  → QuickFormulaPanel.handleSave(id)
    → 构建 payload: { name, ratioFactor, supplementRatioFactor, finishedWeight,
                       packagingPrice, otherPrice, profitMargin, materials }
      → quickFormulaListStore.saveQuickFormula(id, data)
        → PUT /api/quick-formulas/:id (后端 validateBody 校验 + 业务校验)
          → 返回 { success: true } → MessagePlugin.success("保存成功") → 刷新列表
          → 返回 { success: false } → catch 分支 → return false
```

### 前端校验规则（quickFormulaStore.validate()）

| 字段 | 规则 | 错误提示 |
|------|------|---------|
| 配方名称 | 非空（trim 后） | "配方名称不能为空" |
| 原料列表 | 至少 1 种 | "至少需要添加一种原料" |
| 成品重量 | > 0 | "成品重量必须大于 0" |

### 后端校验规则（validateBody + 控制器）

| 字段 | 类型 | 必填 | 约束 | 错误码 |
|------|------|------|------|--------|
| name | string | 创建必填 / 更新选填 | 1-100 字符 | VALIDATION_ERROR(400) |
| ratioFactor | number | 否 | 0.15-0.25 | VALIDATION_ERROR(400) |
| supplementRatioFactor | number | 否 | 0.5-1.5 | VALIDATION_ERROR(400) |
| finishedWeight | number | 否 | 必须 number 类型 | VALIDATION_ERROR(400) |
| materials | array | 否 | minLength: 1 | VALIDATION_ERROR(400) |
| packagingPrice | number | 否 | 必须 number 类型 | VALIDATION_ERROR(400) |
| otherPrice | number | 否 | 必须 number 类型 | VALIDATION_ERROR(400) |
| profitMargin | number | 否 | 必须 number 类型 | VALIDATION_ERROR(400) |

### 后端业务规则

| 规则 | 条件 | 响应 |
|------|------|------|
| 配方不存在 | id 无效 | 404 NOT_FOUND |
| 无权修改 | formulist 修改他人配方 | 403 FORBIDDEN |
| 仅草稿可更新 | status !== "draft" | 400 VALIDATION_ERROR |
| 名称唯一性 | 新名称与已有重复 | 409 DUPLICATE_ENTRY |
| 已发布不可修改 | isCurrentPublished = true | 前端禁用按钮 |
