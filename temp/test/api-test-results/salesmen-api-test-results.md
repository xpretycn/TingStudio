## 失败用例详情

### D04-P01b 更新业务员信息（不传code字段） — 500（应为200）

- **严重程度**：High
- **预期状态码**：200
- **实际状态码**：500
- **预期响应**：更新成功，仅更新传入的字段
- **实际响应**：`{ success: false, message: "更新业务员失败", error: "NOT NULL constraint failed: salesmen.code" }`
- **修复建议**：在 `salesmanController.ts` 的 `updateSalesman` 函数中，当 `code` 字段未传入时，应保留原有值而非将其设为 null。当前实现将未传入的字段设为 null，导致违反数据库 NOT NULL 约束。应在构建 UPDATE 语句时，仅更新传入的字段，或确保 code 等必填字段在未传入时使用原值。
