## 失败用例详情

### U03-P01 — 禁用formulist用户返回 400
- **严重程度**: Medium
- **预期状态码**: 200
- **实际状态码**: 400
- **预期响应**: `{success:true, data:null, message:"用户状态更新成功"}`
- **实际响应**: 400 错误（可能是"不能禁用管理员账号"）
- **分析**: 测试中使用的formulist用户ID可能实际角色为admin（因为U02-P01测试中先将用户角色改为formulist，但U02-P02又改回了admin），导致禁用时触发了"不能禁用管理员"规则
- **修复建议**: 测试数据准备问题。需要确保禁用操作的目标用户确实是非admin角色。建议在禁用测试前先确认目标用户角色，或使用专门的测试用户。
