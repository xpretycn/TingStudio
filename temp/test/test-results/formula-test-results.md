## 测试局限性说明

1. **受控组件问题**：TDesign 的 Input 组件使用 clear() 无法正确清空值，需用 fill("")
2. **行点击交互**：配方列表行点击跳转详情的交互方式不明确，需要前端添加 data-testid
3. **FormulaForm 定位器**：表单页的输入框 placeholder 不包含"名称"关键词，导致定位失败
4. **FormulaVersions 依赖**：版本页需要从列表页获取配方 ID，但列表行内无链接元素
5. **动态内容**：部分页面依赖特定业务数据，无数据时用例被跳过

## 改进建议

1. **添加 data-testid**：为关键交互元素添加 `data-testid` 属性，提高自动化稳定性
2. **修复行点击跳转**：为配方列表行添加明确的点击跳转逻辑
3. **修复快速录入按钮**：检查 quick-formula-btn 的路由跳转是否正确
4. **表单输入框 placeholder**：统一 placeholder 文案，便于自动化定位
5. **测试数据准备**：创建标准测试数据集，确保版本管理等功能有数据可用
