# 业务员详情页测试用例

## 页面信息
| 项目 | 值 |
|------|------|
| 页面名称 | 业务员详情 |
| 路由 | /salesmen/:id |
| 文件 | frontend/src/views/salesmen/SalesmanDetail.vue |

## 交互元素清单

| 编号 | 元素名称 | 选择器 | 行为 | 业务说明 |
|------|---------|--------|------|---------|
| E01 | 返回按钮（左上角） | `.header-back-btn` | @click="handleBack" → router.push({ path: '/salesmen', query: route.query }) | 返回列表页，保留查询参数 |
| E02 | 面包屑链接 | `.breadcrumb-link` | @click="handleBack" → 同 E01 | 点击"业务员管理"返回列表 |
| E03 | 返回按钮（右上角） | `.header-action-btn.secondary` | @click="handleBack" → 同 E01 | 右上角"返回"按钮 |
| E04 | 编辑按钮 | `.header-action-btn`（非 secondary） | @click="router.push(`/salesmen/${route.params.id}/edit`)" | 跳转编辑页面 |
| E05 | 业务员概况卡片 | `.info-card`（第一个） | 纯展示 | 显示姓名、工号、部门、电话、邮箱、状态、配方数量、创建/更新时间，空值显示 '--' |
| E06 | 状态标签 | `.status-tag` / `.t-tag` | 纯展示 | active 显示"活跃"(success)，inactive 显示"停用"(default) |
| E07 | 变更记录时间线 | `.timeline-list` | 纯展示 | createdAt !== updatedAt 时显示"最近更新"+"初始创建"两个节点；createdAt === updatedAt 时只显示"初始录入" |
| E08 | 关联配方列表 | `.formula-item` | @click="router.push(`/formulas/${item.id}`)" | 点击配方项跳转配方详情 |
| E09 | 配方列表分页 | `.pagination-btn`（配方区域内） | goToFormulaPage(page) → loadFormulas() | 分页浏览关联配方，每页 10 条 |
| E10 | 空配方状态 | `.formula-empty` / `.t-empty` | 显示"该业务员暂无关联配方"和"创建配方"按钮 | 无配方时显示空状态，点击"创建配方"跳转 /formulas?new=1 |
| E11 | 配方加载中 | `.formula-loading` / `.t-loading` | formulasLoading 为 true 时显示 | 加载配方数据时显示 loading |
| E12 | 页面骨架屏 | `PageSkeleton` | salesman 为 null 时显示 | 数据加载中显示骨架屏 |
| E13 | 标题头像 | `.title-avatar` | 显示姓名首字母大写 | 展示业务员头像占位 |
