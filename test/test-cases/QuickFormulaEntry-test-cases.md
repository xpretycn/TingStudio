## 一、页面概览

快速配方入口页为侧边栏列表布局，用于管理快速配方。页面包含：
- **侧边栏头部**：标题"快速配方" + 模板管理按钮 + 折叠按钮
- **新建配方按钮**：点击后出现内联输入框，输入名称后按 Enter 创建
- **草稿横幅**（条件渲染）：当 localStorage 存在未完成草稿时显示，包含草稿名称提示、恢复按钮、丢弃按钮
- **配方列表**：显示所有快速配方项，每项包含名称、状态标签、操作按钮（保存模板、重命名、删除）
- **折叠/展开**：侧边栏可折叠为 0 宽度，折叠后仅显示触发卡片

### 关键业务逻辑
- 点击"新建配方"按钮 → 出现内联输入框 → 输入名称按 Enter 或 blur → 调用 `store.createQuickFormula(name)` 创建
- 输入为空或纯空格时按 Enter/blur → 取消创建
- 双击配方名称 → 进入重命名模式 → 修改后按 Enter/blur 保存
- 点击配方项 → emit `select` 事件，父组件加载配方数据
- 已发布配方点击 → emit `select-published` 事件，提示"已发布的配方请在配方管理中查看详情"
- 删除配方 → t-popconfirm 二次确认后调用 `store.deleteQuickFormula(id)`
- 草稿恢复 → 调用 `quickFormulaStore.restoreDraft(draft)` + emit `restore-draft`
- 草稿丢弃 → 调用 `quickFormulaStore.clearDraft()` + 本地 draft 置 null
- 编辑中点击侧边栏外部 → 自动折叠侧边栏
