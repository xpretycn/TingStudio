## 一、页面概览

配方编辑器是快速配方模块的核心编辑面板，位于三栏布局中间区域，包含以下三个区域：
- **参数设置区**：主料系数、辅料系数、成品重量三个参数输入，影响含量比和营养计算
- **原料列表区**：主料/辅料分组显示，支持用量/单价编辑、恢复基价、删除原料，空态提示
- **费用与操作区**：包材费用、其他费用、利润率输入，保存/发布按钮，校验错误列表

### 关键业务逻辑
- 主料系数范围 0.15~0.25（步长0.01），辅料系数范围 0.5~1.5（步长0.01），成品重量最小1（步长100）
- 含量比 = (用量 / 成品重量) × 系数，主料用 ratioFactor，辅料用 supplementRatioFactor
- 小计 = (用量 / 1000) × 单价，单价为null时小计显示"—"
- 恢复基价有闪烁动画（CSS restore-flash 0.5s + JS 600ms定时器清除）
- 删除原料需 t-popconfirm 二次确认
- 保存需检查 quickFormulaListStore.selectedId，未选中则提示"请先创建配方"并触发 request-create 事件
- 发布前调用 store.validate() 校验（名称非空、至少1种原料、成品重量>0），错误显示在错误列表
- 发布按钮在未选中配方或配方已发布时 disabled
