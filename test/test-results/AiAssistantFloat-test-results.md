## 跳过用例详情

### 主要跳过原因说明

1. **SSE流式对话（20条）**: AI助手的核心功能依赖后端SSE流，需要配置AI API Key。当前环境未配置API Key，healthStatus=error，无法触发SSE对话。

2. **拖拽交互（15条）**: 气泡和抽屉的拖拽功能需要精确的鼠标事件模拟（mousedown/mousemove/mouseup序列），自动化测试不稳定。

3. **内部状态模拟（30条）**: 很多用例需要模拟特定内部状态（如fillFeedback非空、badgeCount=5、healthStatus=loading等），直接修改Pinia store state不一定触发Vue响应式更新。

4. **视觉/动画验证（40条）**: CSS动画、过渡效果、悬停效果等需要视觉判断，自动化验证困难。

5. **AI功能依赖（25条）**: 所有涉及AI回复、回填、意图识别的用例都需要AI API可用。

6. **消息操作（12条）**: 复制、删除、重试等操作需要先有消息存在，而消息需要AI对话产生。

## 测试建议

1. **AI API配置**: 建议在测试环境中配置AI API Key（如DeepSeek），以便执行SSE流式对话、回填、意图识别等核心功能测试。

2. **拖拽测试**: 建议使用Playwright的page.mouse API进行精确的拖拽测试，而非通过JavaScript事件模拟。

3. **内部状态测试**: 建议通过Vue DevTools或Pinia插件在开发环境中手动验证内部状态相关的用例。

4. **视觉回归测试**: 建议使用截图对比工具（如Playwright的screenshot comparison）验证动画和视觉效果。
