## 失败用例详情

### A07-E02 文件名路径遍历
- **严重程度**: Low
- **预期状态码**: 500（"无效的备份文件名格式"）
- **实际状态码**: 404
- **预期响应**: `{success:false, error:{message:"无效的备份文件名格式", code:"INTERNAL_ERROR"}}`
- **实际响应**: 404 Not Found
- **分析**: 路径遍历字符 `../../` 被 Express 路由解析为路径导航，导致路由不匹配返回404，而非被控制器校验拒绝返回500。虽然安全效果相同（未泄露文件），但错误码不符合预期。
- **修复建议**: 无需修复。404表示路由不匹配，实际安全效果等价。若需统一错误码，可在路由前添加中间件校验文件名格式。

### A08-E04 恢复文件名路径遍历
- **严重程度**: Low
- **预期状态码**: 500（"无效的备份文件名格式"）
- **实际状态码**: 404
- **分析**: 同A07-E02，路径遍历字符被路由层解析导致404。
- **修复建议**: 同A07-E02。

### A09-E02 删除路径遍历
- **严重程度**: Low
- **预期状态码**: 500（"无效的备份文件名格式"）
- **实际状态码**: 404
- **分析**: 同A07-E02，路径遍历字符被路由层解析导致404。
- **修复建议**: 同A07-E02。

### A12-P01 执行verify-data脚本返回status=failed
- **严重程度**: Medium
- **预期结果**: data.status="completed"
- **实际结果**: data.status="failed"
- **分析**: verify-data脚本执行失败，可能是脚本内部逻辑依赖的数据条件不满足（如缺少某些表数据）。
- **修复建议**: 检查verify-data脚本逻辑，确认失败原因。可能是测试环境数据不完整。

### A06-I01 连续创建备份文件名相同
- **严重程度**: Low
- **预期结果**: 两次创建的fileName不同（时间戳不同）
- **实际结果**: 两次创建返回相同的fileName `tingstudio_backup_2026-06-07T22-36-25.json`
- **分析**: 连续快速创建备份时，时间戳精度不足导致文件名相同。第二次请求可能覆盖了第一次的备份文件。
- **修复建议**: 在文件名中增加毫秒精度或随机后缀，确保唯一性。
