## 失败用例详情

### C01-P01 ~ C01-DI01 — GET /api/files 返回 500
- **严重程度**: Critical
- **预期状态码**: 200
- **实际状态码**: 500
- **预期响应**: `{success:true, data:{list:[...],total:N,stats:{...}}}`
- **实际响应**: 500 Internal Server Error
- **修复建议**: 检查 fileController.getFileList 中的查询逻辑，uploaded_files 表可能缺少必要字段或查询语句有误。可能是数据库表结构与代码不匹配。

### C06-P01 — POST /api/files/fix-garbled 返回 500
- **严重程度**: High
- **预期状态码**: 200
- **实际状态码**: 500
- **修复建议**: 检查 fix-garbled 接口的数据库查询，可能依赖 uploaded_files 表的正常访问。

### C07-E01 ~ C16-E02 — 文件不存在场景返回 500 而非 404
- **严重程度**: High
- **预期状态码**: 404
- **实际状态码**: 500
- **修复建议**: fileController 中各接口在文件不存在时应先检查文件是否存在，返回 404 而非让查询异常导致 500。当前实现可能未正确处理"文件不存在"的业务异常。

### X-MD-02 — GET /api/files/upload 返回 500
- **严重程度**: Low
- **预期状态码**: 404
- **实际状态码**: 500
- **修复建议**: 路由匹配异常，GET /api/files/upload 可能被 :fileId 参数路由捕获，导致查询 ID="upload" 的文件失败返回 500。
