## 测试环境问题

### 问题1: 工作台页面重定向

- **现象**: 访问 `/dashboard` 时，页面被重定向到 `/smart-tools` 或 `/formulas`
- **原因**: `Home.vue` 的 `onMounted` 钩子中，`fetchPreferences()` 获取的 `defaultHomePage` 不为 `/dashboard`，触发 `router.replace(prefs.defaultHomePage)`
- **解决**: 通过 API `PUT /api/auth/preferences` 将 `defaultHomePage` 修改为 `/dashboard`
- **影响**: 此问题不影响生产环境（用户可自行设置默认首页），但影响自动化测试

### 问题2: MCP click 与脚本 click 差异

- **现象**: 通过 `evaluate_script` 中的 `element.click()` 点击按钮有时不触发 Vue 事件
- **原因**: Vue 事件绑定在虚拟 DOM 上，原生 click 可能不触发 Vue 事件处理器
- **解决**: 使用 Chrome DevTools MCP 的 `click` 工具，通过 accessibility tree 定位元素并点击
